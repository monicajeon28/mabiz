═══════════════════════════════════════════════════════════════════════════════

📊 분석 결과
───────────────────────────────────────────────────────────────────────────────
✅ 문제 식별: 3가지 주요 데이터 일관성 위험
   1. Contact 생성 후 Funnel 트리거 분리 → 불일치 가능
   2. GroupMember 생성 후 memberCount 증가 무시 → 영구 부정확
   3. SMS 발송 + Funnel이 Promise.allSettled로 병렬 → 실패 무시

✅ 솔루션 선택: Prisma $transaction
   - 즉시 적용 (3-5줄 수정 × 4파일)
   - ACID 보장 (강한 일관성)
   - 기존 호환성 100% (tx 파라미터 선택사항)
   - 성능 오버헤드 ~100ms (허용 범위)

═══════════════════════════════════════════════════════════════════════════════

🎯 구현 대상 (4개 파일)
───────────────────────────────────────────────────────────────────────────────

Phase A: Funnel 트리거 함수 tx overload 추가
  [1] src/lib/funnel-trigger.ts
      • Import: import { Prisma } from "@prisma/client";
      • 함수 시그니처: txOrPrisma: TxOrPrisma = prisma 파라미터 추가
      • 내부: 모든 prisma.* → txOrPrisma.* 변경 (~5줄)

  [2] src/lib/funnel-sms-trigger.ts
      • 동일 패턴으로 tx overload 추가 (~5줄)

Phase B: Contact 생성 엔드포인트 트랜잭션화
  [3] src/app/api/contacts/route.ts (POST, Line 276-404)
      • 구조: Contact 생성 → 렌즈 감지 → Funnel 트리거 (tx 전달)
      • 에러 처리: P2025, P2002 명시적 핸들링
      • SMS는 트랜잭션 외부 (최종 일관성)
      • 변경: ~60줄

Phase C: GroupJoin 엔드포인트 트랜잭션화
  [4] src/app/api/public/group-join/route.ts (POST, Line 81-146)
      • 구조: Contact upsert → Member 체크 → Member upsert → memberCount 증가
      • memberCount: 신규만 증가, 재입장은 유지
      • 에러 처리: 유니크/참조무결성 위반 핸들링
      • FunnelSms는 트랜잭션 외부
      • 변경: ~60줄
