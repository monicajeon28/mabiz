## 핵심 요구사항

### 사용자 페르소나
| 페르소나 | 특성 | 필요한 기능 |
|---------|------|-----------|
| **초보자** | 코딩 모름, UI로 만들고 싶음 | 블록 드래그앤드롭, 명확한 설명, 실시간 미리보기 |
| **중급자** | 기본 구조는 알지만 커스텀 원함 | 세부 설정(색상/폰트/여백), A/B 테스트, CSS override |
| **고급자** | HTML/CSS 직접 수정 원함 | Raw HTML 편집 모드 전환, 컴포넌트 재사용, API 연동 |

### MVP 기능 체크리스트
```
✅ 블록 드래그앤드롭 추가/제거
✅ 블록 순서 변경 (위/아래 이동)
✅ 라이브 프리뷰 (WYSIWYG)
✅ 블록별 설정 패널 (동적)
✅ 반응형 미리보기 (PC/태블릿/모바일)
✅ 설정 변경 → 즉시 캔버스 반영
✅ 블록 복제 / 삭제
✅ 실행 취소 / 다시 실행
✅ 저장 / 로드
❌ (Phase 2) A/B 테스트 버전 관리
❌ (Phase 2) 블록 커스텀 CSS / 고급 설정
❌ (Phase 3) AI 카피 생성 통합
```

### 드래그 앤 드롭 UX

#### 1️⃣ 블록 목록에서 캔버스로 드래그

```
마우스 다운 (좌측 블록)
  ↓
드래그 중 (캔버스 하이라이트)
  ↓
마우스 업 (캔버스)
  ↓
새 블록 추가 + 자동 ID 생성
  ↓
설정 패널에 새 블록 필드 표시
  ↓
라이브 프리뷰 자동 업데이트
```

#### 2️⃣ 캔버스 내 블록 순서 변경

```
상황: 블록 2를 블록 1 위로 이동하고 싶음

옵션 A - 드래그앤드롭:
  블록 2의 GripVertical 아이콘 잡기
  ↓
  블록 1 위로 드래그
  ↓
  마우스 업
  ↓
  순서 변경 + 저장

옵션 B - 버튼:
  블록 2의 [↑] 버튼 클릭
  ↓
  블록 2 ↔ 블록 1 위치 교환
```

### 블록 선택 및 포커스

```
클릭한 블록:
- 테두리 강조 (2px solid #3b82f6)
- 음영 추가 (box-shadow)
- 설정 패널에 해당 필드 표시
- 키보드 포커스 가능 (Tab)

다른 블록 클릭:
- 이전 포커스 제거
- 새로운 블록 강조
- 설정 패널 자동 스크롤
```

| **textarea** | 다중행 입력 | 설명, 본문 |

### 설정 변경 실시간 반영

```
입력창에 글자 입력
  ↓
onChange 이벤트 발생 (debounce 300ms)
  ↓
상태(state) 업데이트
  ↓
캔버스 라이브 프리뷰 즉시 변경
  ↓
"변경 사항 저장되었습니다" 토스트 (2초)
```

### 저장 & 초기화 버튼

```
[저장]:
  - 설정 변경사항 → DB 저장
  - 이전 설정과 비교
  - 변경 없으면 스킵
  - 성공 메시지 표시

[초기화]:
  - 해당 블록 기본값으로 복원
  - 확인 다이얼로그 표시
  - "확인하시겠어요?" → [예] [아니오]
```

## 블록 타입 정의

### 1. Hero 섹션
```
목적: 큰 제목 + 부제목 + 배경 + CTA 버튼
필드:
├─ 제목 (text)
├─ 부제목 (textarea)
├─ 배경 이미지 (image)
├─ 배경 색상 (color)
├─ 오버레이 투명도 (range 0-1)
├─ 버튼 텍스트 (text)
├─ 버튼 링크 (text)
├─ 버튼 색상 (color)
├─ 높이 (select: 300px/400px/500px/custom)
└─ 텍스트 정렬 (select: left/center/right)

렌더링:
<section class="hero" style="...">
  <h1>{{ title }}</h1>
  <p>{{ subtitle }}</p>
  <button onclick="...">{{ buttonText }}</button>
</section>
```

### 2. 제목 (H1-H6)
```
필드:
├─ 텍스트 (text)
├─ 크기 (select: H1/H2/H3/H4/H5/H6)
├─ 색상 (color)
├─ 정렬 (select: left/center/right)
└─ 여백 (object: top/bottom)

렌더링:
<h{{ level }}>{{ text }}</h{{ level }}>
```

### 3. 단락 (Paragraph)
```
필드:
├─ 텍스트 (textarea)
├─ 텍스트 크기 (range: 12-32px)
```

## UI/UX 원칙

### 1. WYSIWYG (What You See Is What You Get)

```
설정 패널에서 한 글자 입력 → 캔버스에 즉시 반영
"사용자가 지정한 것만" 보여주기
추가 장식/기능 절대 금지

❌ 나쁜 예:
┌─────────────────┐
│ Hero 섹션       │
│ [큰 제목]       │
│ [부제목]        │
│ ★ 자동 추가됨   │ ← 사용자가 지정 안 함!
└─────────────────┘

✅ 좋은 예:
┌─────────────────┐
│ Hero 섹션       │
│ [큰 제목]       │
│ [부제목]        │
│ (아무것도 없음) │ ← 사용자 설정만!
└─────────────────┘
```

### 2. Predictability (예측 가능)

```
입력값 → 출력값 (1:1 대응)

제목: "파일럿 모집"
  ↓ (캔버스)
  ↓
제목: "파일럿 모집" (똑같음)

색상: #FF0000
  ↓ (캔버스)
  ↓
색상: #FF0000 (똑같음)

부제목: "✅ 3월부터 시작" (마크다운 금지!)
  ↓ (캔버스)
  ↓
부제목: "✅ 3월부터 시작" (그대로 표시)
```

### 3. Discoverability (발견 용이)

```
🔍 검색:
  입력: "버튼"
  결과: CTA Button, Form Submit, ... (관련 블록만 표시)

(?) 도움말:
  각 필드마다 설명 + 팁 + 피하기 항목

기본값:
  모든 블록은 의미 있는 기본값으로 시작
  (예: Hero → "제목을 입력하세요" placeholder)
```

### 4. Feedback (피드백)

```
사용자 액션 → 시각적 피드백

블록 추가:
  ✅ 토스트: "블록이 추가되었습니다"
  ✅ 자동 스크롤: 새 블록으로 이동
  ✅ 자동 선택: 설정 패널 열림

설정 저장:
  ✅ 토스트: "저장되었습니다" (2초)
  ✅ 버튼 비활성화 (저장 중) → 활성화

오류 발생:
  ❌ 빨간 테두리: 잘못된 필드
  ❌ 에러 메시지: "이미지 크기가 너무 큽니다 (Max 2MB)"
  ❌ 제안: "최적화해드릴까요?" → [자동 압축]
```

### 5. Efficiency (효율성)

```
키보드 단축키:
  Ctrl+Z: 실행 취소
  Ctrl+Y: 다시 실행
  Ctrl+S: 저장
  Delete: 선택 블록 삭제 (포커스 시)

빠른 복제:
  선택 블록 우클릭 → [복제] 메뉴
  또는 [복제] 버튼 클릭

다중 선택:
  Ctrl+클릭으로 여러 블록 선택
  → 일괄 삭제/복제/이동

드래그 앤드롭:
  팔레트에서 캔버스로 1초 이내 추가
  블록 내에서 순서 변경 (Shift+위/아래도 가능)
```

### 데이터 구조

```typescript
// Block 상태 (메모리)
interface Block {
  id: string;           // UUID
  type: BlockType;      // 'hero' | 'heading' | ...
  order: number;        // 정렬순서
  settings: Record<string, any>;  // 블록별 설정
  isSelected: boolean;  // UI 상태 (메모리만)
}

type BlockType = 
  | 'hero'
  | 'heading'
  | 'paragraph'
  | 'image'
  | 'button'
  | 'list'
  | 'faq'
  | 'stats'
  | 'spacer'
  | 'divider'
  | ...;

// Editor 상태 (메모리)
interface EditorState {
  blocks: Block[];
  selectedBlockId: string | null;
  history: {
    past: Block[][];
    present: Block[];
    future: Block[][];
  };
  unsaved: boolean;
}

// DB 저장 형식 (마이그레이션: htmlContent → 구조화된 JSON)
// 현재:
{
  "htmlContent": "<html>...</html>"  // 단일 문자열
}

// 마이그레이션 후:
{
  "blocks": [
    {
      "id": "block_1",
      "type": "hero",
      "order": 0,
      "settings": {
        "title": "제목",
        "subtitle": "부제목",
        "backgroundImage": "...",
        "buttonText": "신청하기",
        "buttonUrl": "/..."
      }
    },
    ...
  ]
}

// 또는 HTML 캐시도 유지 (성능):
{
  "blocks": [...],
  "htmlCache": "<html>...</html>"  // 미리 컴파일된 HTML
}
```

### API 엔드포인트

```typescript
// 기존 (유지)
GET  /api/landing-pages
GET  /api/landing-pages/[id]
POST /api/landing-pages
PUT  /api/landing-pages/[id]
DELETE /api/landing-pages/[id]

// 신규 (블록 편집)
POST   /api/landing-pages/[id]/blocks
  { type: 'hero', settings: {...} }
  → { block: Block }

PUT    /api/landing-pages/[id]/blocks/[blockId]
  { settings: {...} }
  → { block: Block }

DELETE /api/landing-pages/[id]/blocks/[blockId]
  → { success: true }

PUT    /api/landing-pages/[id]/blocks/reorder
  { blockIds: ['block_1', 'block_3', 'block_2'] }
  → { blocks: Block[] }

// 이미지 업로드 (기존 재사용)
POST /api/landing-pages/images
  multipart/form-data: { file }
  → { url: string, assetId: string }
```

### 성능 최적화

```
1. Debounce 설정 변경 (300ms)
   입력 중에는 저장 안 함
   입력 멈춘 후 300ms 후 자동 저장

2. 가상 스크롤 (100+ 블록)
   캔버스에 보이는 영역만 렌더링
   (react-window 고려)

3. 이미지 최적화
   업로드 시 자동 압축 (prepareImageForUpload)
   WebP 포맷 제안

4. 메모이제이션
   각 블록 컴포넌트: React.memo()
   설정 패널: useMemo() 활용

5. 상태 관리
   Zustand + Immer
   선택 블록만 재렌더링
   전체 블록 배열 재렌더링 최소화
```
