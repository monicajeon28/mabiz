2. P2 변화의 핵심 UX 영향도
================================================================================

2.1 기술 변화

이전 (P0-5):
  페이지 로드 -> /api/auth/me 호출 -> authChecked 대기 -> 데이터 로드

P2 이후:
  페이지 로드 -> Layout 권한 검증 -> 데이터 즉시 로드 (authChecked 제거)

2.2 영향받는 7개 페이지

  1. /pnr/[id]                        -> 200ms 개선 (⭐ 1,000회/일)
  2. /admin/partner-applications      -> 150ms 개선 (100회/일)
  3. /admin/affiliate-sales-by-partner -> 150ms 개선 (50회/일)
  4. /team/affiliate                  -> 300ms 개선 (200회/일, 3회 호출)
  5. /payments                        -> 150ms 개선 (100회/일)
  6-7. 기타 대시보드                   -> 50-100ms 개선

2.3 정량적 지표

  데이터베이스 쿼리:
  - 현재: 1,550회/일 (월 46,500회)
  - P2 후: 0회/일 (완전 제거)
  - 절감: 월 46,500회 쿼리 제거

  페이지 로딩:
  - 현재 평균: 883ms (LCP)
  - P2 목표: 683ms (LCP)
  - 개선율: 22.6%

  비용 절감 (연간):
  - DB 쿼리 비용: $2,400
  - 서버 CPU 비용: $1,800
  - 네트워크 비용: $250
  - 합계: $4,450/년

3. 사용자 경험 검증 전략
================================================================================

3.1 무조건 같아야 할 것 (기능적 동일성)

  OK 로그인 상태 감지
     - 로그인 사용자 접근 -> 페이지 로드 (동일)
     - 로그아웃 사용자 접근 -> /sign-in 리다이렉트 (동일)

  OK 역할별 접근 제어 (더 엄격함)
     - GLOBAL_ADMIN만 /admin/* 접근
     - OWNER+만 /team/* 접근
     - 현재: 클라이언트 검증 -> P2: 서버 검증 (더 안전)

  OK 에러 메시지 (동일)
     - "전화번호 미일치" 메시지
     - "권한이 없습니다" 메시지
     - "서버 오류가 발생했습니다" 메시지

  OK 폼 제출 및 데이터 변경
     - 신청서 승인 -> 목록 즉시 갱신
     - 객실 배정 제출 -> 토스트 메시지
     - 필터 변경 -> 데이터 재로드

3.2 개선 가능한 것 (성능)

  OK 페이지 로딩 시간
     - 현재: 600-1,200ms
     - 목표: 450-950ms
     - 허용 범위: ±10%

  OK API 응답 시간
     - 현재: 150-250ms
     - 목표: 100-200ms
     - 기준: authChecked 대기 시간 제거

  OK 서버 에러율
     - 현재: ~0.5%
     - 목표: < 0.3%
     - 감시: 401/403 에러 로그

3.3 허용 불가능한 변화

  X 권한 없는 정보 노출
     - 비로그인 사용자 -> 정산금 정보 노출 (보안 실패)

  X 로그인 상태 미감지
     - 로그아웃 후 페이지 새로고침 -> 낡은 데이터 표시

  X UI 깜박임 (Flash)
     - 페이지 로드 -> 권한 없음 -> 리다이렉트

  X 예상 못한 리다이렉트
     - 사용자가 작업 중 -> 갑자기 홈으로 리다이렉트

4. 다중 탭 시나리오 (최고 위험도)
================================================================================

4.1 시나리오 1: 세션 만료 (다중 탭)

  탭 A (로그아웃):
    1. GLOBAL_ADMIN 상태로 /admin/partner-applications 표시
    2. "로그아웃" 클릭 -> 서버 세션 무효화

  탭 B (새로고침):
    1. 새로고침 (F5)

  [현재 방식]
    - /api/auth/me 호출 -> 401 응답
    - authChecked = false -> 빈 페이지 표시
    - 500ms 후 리다이렉트
    - UX: 로딩 후 리다이렉트 (깜박임 가능)

  [P2 방식]
    - Layout 레벨에서 권한 검증
    - 권한 없음 -> 즉시 /sign-in 리다이렉트
    - 100ms 내 완료
    - UX: 즉시 리다이렉트 (깜박임 없음) OK

4.2 시나리오 2: 권한 변경 (다중 탭)

  주의: P2 후에도 권한 변경은 캐시 문제로 최대 50초 지연
  해결책: next-auth 세션 갱신 메커니즘 필요

4.3 시나리오 3: 다중 조직 계정

  위험도: 매우 높음
  문제: P2에서 조직 전환 시 /api/auth/me 호출 제거
        -> 조직 ID 헤더가 오래된 조직 기준일 수 있음
  해결책: 조직 전환 시 세션 갱신 필수

7. 롤백 기준 (즉시 롤백 필요)
================================================================================

CRITICAL (배포 < 1시간 후 즉시 롤백)

  1. 비로그인 사용자가 민감 정보 접근 가능
     (예: 정산금 정보 노출)

  2. 로그아웃 후 페이지 새로고침 -> 낡은 데이터 표시
     (Hydration 오류)

  3. 401/403 에러 > 10% 증가
     (시스템 오작동 신호)

  4. LCP > 3초 (심각한 성능 악화)

  5. 무한 리다이렉트 루프
