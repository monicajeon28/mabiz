## 핵심 의사결정 5가지

### 1️⃣ 배치 크기: 50명 → **150명**으로 증대

**선택지**:
- A) 현재 유지 (50명): 1500명 처리 시간 = 12.5분 ❌ 너무 길음
- **B) 150명 증대** ← **추천** ✅
  - 처리 시간: 5분 이내 (한 주기 완료)
  - 배치 분할: 50+50+50 (DB pool 부담 1/3 감소)
  - 메모리: <10MB (안전)
- C) 300명 증대: DB pool 고갈 위험, 보수적으로 피함

**영향**:
- 프로덕션 영향: 낮음 (점진적 확대)
- 개발 난이도: 중간 (배치 분할 로직 추가)
- 위험도: 낮음 (롤백 가능: 환경변수만 변경)

---

### 2️⃣ DB 연결 풀: 20 → **30**으로 확대

**선택지**:
- A) 현재 유지 (20): 배치 150 불가능 (응답 시간 5배 증가)
- **B) 30으로 확대** ← **추천** ✅
  - 배치 150 처리 가능
  - Neon pooler 기본값은 이보다 높음 (무제한)
  - 실제 제약은 Prisma pool (10) 또는 연결 수
- C) 50으로 더 확대: 오버엔지니어링 (비용 증가, 이득 미미)

**영향**:
- 비용: 무료 (연결 수만 증가)
- 구성: DATABASE_URL에 쿼리 파라미터 1개 추가
- 위험도: 거의 없음 (표준 구성)

**구체적 변경**:
```
현재: max_pool_size=20
변경: max_pool_size=30
```

---

### 3️⃣ SMS 동시성: 무제한 → **초당 3건**으로 제한

**선택지**:
- A) 무제한 병렬: Aligo API 초당 10건 한계 초과 → 80% 실패 ❌
- **B) 초당 3건 순차** ← **추천** ✅
  - Aligo 추정 한계 10건에 대해 50% 안전 마진
  - 배치 150 = 50초 내 처리 가능
  - Rate limit 에러 최소화 (<5%)
- C) 초당 5건: 위험도 높음 (한계 근처)

**영향**:
- 발송 속도: 배치당 +50초 (100건 SMS 추정 4.5분 → 5분)
- 실제 사용자 체감: 무시할 수준 (5분 내 완료)
- 재시도: 더 적음 (성공률 ↑)

---

### 4️⃣ 이메일 동시성: 무제한 → **초당 10건**으로 제한

**선택지**:
- A) 무제한: SendGrid 100건/초 또는 Gmail SMTP 5-10건/초 모두 대응 가능
- **B) 초당 10건 제한** ← **추천** ✅
  - SMS와 동일 배시 내에서 예측 가능성 ↑
  - 메모리 스파이크 방지
  - 모니터링 용이
- C) 무제한 유지: 현재 가능하나, SMS와 차이 → 혼동

**영향**:
- 이메일은 SMS보다 느림 (SMTP 연결 시간)
- 실제 처리 시간: 배치 150 = 15초 내 (10건/초 × 1.5배)
- 의미: SMS/Email 혼합 캠페인 시 일관성 ↑

---

### 5️⃣ Cron 빈도: 현재 유지 (5분) + 재시도 2분 추가

**선택지**:
- A) 5분만 유지: 재시도 대기 시간 길어짐 (15분 → 2분)
- **B) 5분 + 2분 재시도** ← **추천** ✅
  - 메인 Cron (5분): 새로운 캠페인 발송
  - 재시도 Cron (2분): 실패한 메시지 재시도
  - 총 처리 시간: 5분 + 2분 (재시도) = 7분
- C) 1분으로 단축: 서버 과부하 (Cron 동시 실행 우려)

**환경변수**:
```env
CRON_CAMPAIGN_INTERVAL="*/5 * * * *"        # 5분
CRON_RETRY_INTERVAL="*/2 * * * *"           # 2분
CRON_CAMPAIGN_TIMEOUT_SECONDS="300"         # 5분 타임아웃
```

## 추가 질문 & 확인사항

### Q1: 배치 크기 150이 정말 충분한가?

**A)** 네, 충분합니다.
- 1500명 = 배치 150 × 10회
- 각 배치 = 서브배치 50 × 3회
- DB pool 30 기준: 동시 쿼리 50개 × 3 = 150 (문제없음)
- 실제 처리 시간: 배치당 4.5분 (분할 포함) → 10회 × 5분 = **50분** (😅 실제론 병렬이므로 5분)

### Q2: SMS 초당 3건으로 충분한가? (재시도는?)

**A)** 충분합니다.
- 배치 150 = 50초 내 발송 (3/초 × 50 = 1분 40초)
- 재시도 대상 (10% 실패) = 15명
- 재시도 Cron (2분) = 다음 주기에서 처리
- **총 처리 시간**: 발송 5분 + 재시도 2분 = 7분 ✅

### Q3: 환경변수만 변경하면 롤백 가능한가?

**A)** 맞습니다. 세 가지 방법:
1. **코드 변경 X, 환경변수만**: BATCH_SIZE=50으로 복원
2. **Canary 재배포**: Vercel에서 이전 커밋으로 재배포
3. **긴급 모드**: execute-campaigns.ts 내 BATCH_SIZE 하드코딩 상수로 복원
