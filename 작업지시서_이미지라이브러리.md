## 핵심 설계 결정

### 워터마크 시점: 사전 생성 (업로드/동기화 시)
- 요청시점 처리 → Vercel 메모리/타임아웃 위험 → **업로드 시 1회 처리**
- 원본(PNG) + 처리본(WebP+워터마크)을 Drive에 분리 저장
- 클라이언트에는 처리본 URL만 노출, 원본 driveFileId는 서버 내부 격리

### 이미지 서빙: Drive 직접 URL (처리본만)
- processed/ 폴더의 WebP → `anyone: reader` 공개 설정 → lh3 URL로 직접 서빙
- 원본(originals/)은 비공개 유지 → `/api/images/[id]/download` 프록시로만 접근

### 공유 폴더: 환경변수 관리
- `IMAGE_LIBRARY_SHARED_FOLDER_ID=1NKe7U-VKROfuJL6LvhP4KT1Gp28bpK1-`
- 코드에 하드코딩 금지

**파일**: `src/lib/image-processor.ts` (신규)


## 주의사항
1. processed/ 폴더 파일만 `anyone:reader` 공개 — 원본은 절대 공개 금지
2. 클라이언트 응답에서 driveFileId 제거 — 원본 URL 격리
3. SVG 업로드 차단 (XSS 위험)
4. sharp는 Vercel Node.js runtime에서 동작 (Edge runtime 불가)
5. maxDuration=60 설정 (Pro 플랜 기준, Hobby는 10s)
6. 워터마크 로고는 SVG로 유지 (해상도 독립적)
