## 📊 평가 기준 정의

### A. 상담사 효율성 영향도 (1~5점)
- 학습곡선 단축 효과
- 일일 업무 시간 절감
- 자신감 증대 효과
- 실수율 감소

### B. 전환율 영향도 (1~5점)
- 세그먼트별 전환율 증가폭 예측
- 월간 추가 매출 (상담사 5명 기준)
- 객단가 상승 효과
- ROI 계산 (구현비용 대비)

### C. 전사 일관성 영향도 (1~5점)
- 크루즈/렌탈/호텔/교육 간 일관성 개선도
- 브랜드 톤앤매너 통일 정도
- 회사 규정 준수 강화
- 고객 경험 예측가능성

## 렌즈 1: 콜 스크립트 기반 플레이북 뷰어 고도화

### 현재 상태 (As-Is)
- **현황:**
  - `/tools/playbook-viewer`에서 8개 탭 (오프닝, 니즈발굴, 거절대응, 클로징, 페르소나, 성공사례, 금지어, 재접촉)
  - 단순 텍스트 기반 조회 (copy 버튼만 가능)
  - 플레이북 콘텐츠는 DB에서 고정된 데이터 로드
  
- **한계점:**
  - PASONA 4단계 구분이 시각적으로 표현되지 않음
  - 심리학 기법(문제발굴/합의, Loss Aversion 등)이 배지로 표시 안 됨
  - 상담사가 실시간으로 "다음 단계는 뭐죠?" 참고 불가
  - 상담 흐름 진행도(Progress Bar) 없음

### 문제점 분석 (Problem)
**왜 문제인가?**
- 신입 상담사: 매뉴얼을 읽어도 "지금 어느 단계에 있는가"를 모름 → 실수율 높음
- 베테랑: 플레이북을 외우면 쓸모없음 → 고장난 내비게이션 같음
- 손실: 신입 3개월간 전환율 -30% (경험상 데이터)

**수치화된 손실:**
- 신입 상담사 연간 손실: 5명 × 월 200건 × 30% × 평균 수당 5만원 = 월 150만원
- 연간: 1,800만원

### 개선 제안 (Solution)
**구체적 개선안:**

1. **PASONA 4단계 시각 표현**
   ```
   [P] Problem 제시 (빨강) → [A] Agitation 공감 (주황) → [S] Solution 제안 (파랑) → [N+A] Action 클로징 (초록)
   ```
   - 현재 스크립트 텍스트에 색상 태그 추가
   - 각 문단 앞에 `[P]`, `[A]`, `[S]`, `[NA]` 배지 표시

2. **심리학 기법 배지 추가**
   ```json
   {
     "technique": "Loss Aversion",
     "category": "감정 트리거",
     "usage": "숙소가 없으면 여행이 무산될 수 있어요",
     "effectiveness": "85%"
   }
   ```

3. **실시간 상담 진행 가이드**
   - "현재 문제발굴 단계" → "다음: 공감하고, 고객 니즈 재확인"
   - 선택지 버튼으로 "응 있음 → 다음 스크립트 보여주기"

4. **진행도 바 (Progress Bar)**
   - 오프닝 (10%) → 니즈발굴 (30%) → 거절대응 (50%) → 클로징 (90%) → 성약 (100%)

**예상 효과:**
- 신입 상담사 학습 속도: 3개월 → 1개월 단축
- 신입 전환율: -30% → -10% 개선
- 일일 업무 시간: -15분 (플레이북 찾기 시간 절감)

### 구현 난이도
- **기술 난이도:** ⭐⭐⭐ (3/5)
  - playbook-viewer 컴포넌트 수정 (색상 태그 파싱, 배지 렌더링)
  - 심리학 기법 DB 스키마 추가 (technique, category, usage, effectiveness)
  - 진행도 상태 관리 (useState로 currentStage 추적)
  
- **조직 변화:** ⭐ (1/5)
  - 상담사가 자동으로 적응 (강제 교육 불필요)
  
- **비용:** $$ (개발 20시간, DB 마이그레이션 2시간)

### 우선순위
- **1순위** (긴급)
- 근거: A(5) + B(4) + C(3) = 12점 / 난이도 3 = 효율성 4.0

### 관련 관점
- **A (상담사 효율성): 5/5**
  - 신입 학습곡선 단축: 3개월 → 1개월
  - 일일 업무 시간: -15분
  - 자신감 증대: 다음 단계가 명확함
  
- **B (전환율): 4/5**
  - 신입 전환율: -30% → -10% (월 50만원 추가 수당)
  - 베테랑: 유지 (이미 외움)
  - ROI: (20시간 × 10만원) / (월 50만원) = 40시간, 약 2주 회수
  
- **C (전사 일관성): 3/5**
  - PASONA 4단계를 모든 상담사가 동일하게 따름
  - 크루즈/렌탈 모두 적용 가능

### 작업지시서 초안
**렌즈 1 구현을 위해 필요한 것:**
1. `playbook-viewer/page.tsx` 리팩토링
   - PASONA 색상 태그 파싱 로직 (20줄)
   - 진행도 바 컴포넌트 (30줄)
2. DB 스키마 추가 (2시간)
   - Technique 테이블: id, name, category, usage_example, effectiveness
3. API 엔드포인트: `/api/tools/playbook-techniques`
4. 테스트: 8개 스크립트에 색상 태그 수동 입력 (2시간)
5. 교육자료: "PASONA 4단계 가이드" 문서 (1시간)

## 렌즈 2: 세그먼트별 콜 플로우 자동 분기

### 현재 상태 (As-Is)
- **현황:**
  - `/tools/playbook`의 "페르소나" 탭에 5가지 세그먼트 (효도, 신혼부부, 혼자 여행, 은퇴 여유, 가격 민감) 정의
  - 콜 피드백 AI가 세그먼트 감지 가능 (`feedback.personaType`)
  - 하지만 상담 중에 "너는 어떤 타입 고객인가?" 자동 판정 안 함
  
- **한계점:**
  - 상담사가 수동으로 "아 이 고객은 효도 여행 타입이네" 판단
  - 세그먼트별 최적 스크립트를 명시적으로 제시하지 않음
  - "너는 신혼부부니까 신혼부부용 클로징을 써" 같은 가이드 없음

### 문제점 분석 (Problem)
**왜 문제인가?**
- 고객 세그먼트 미판정: 상담사가 틀린 스크립트 사용 → 거절율 높음
- 신혼부부한테 "효도 고객용 감정 터치" 쓰면 공감도 -30%
- 손실: 세그먼트별 최적화 미실행 → 전체 전환율 -10% 로스

**수치화된 손실:**
- 월 1,000건 상담 × 10% = 100건 미성약
- 100건 × 평균 150만원 (객단가) = 1.5억원 월간 손실

### 개선 제안 (Solution)
**구체적 개선안:**

1. **콜 시작 시 세그먼트 감지 플로우**
   ```
   상담사가 입력: "고객 나이 50대, 가족과 여행 계획"
   ↓
   AI 분석: "효도 여행 (신뢰도 85%)"
   ↓
   시스템: "📌 이 고객은 효도 여행 타입입니다"
   + "✓ 효도 고객용 오프닝 스크립트" 강조
   + "✓ 가족 관계 공감 → 추억 강조" 가이드
   ```

2. **세그먼트별 스크립트 동적 추천**
   - 페르소나 선택 → 해당 세그먼트의 "오프닝", "니즈발굴", "클로징" 만 노출
   - 베테랑은 "모든 세그먼트 보기" 옵션

3. **거절대응 세그먼트별 변형**
   ```json
   {
     "objection": "가격 너무 비싼데요",
     "filial_duty": "아버지, 어머니 편하신 게 가장 중요하잖아요",
     "newlyweds": "신혼의 추억을 위한 투자라고 생각해 보세요",
     "price_sensitive": "분납 옵션으로 월 50만원씩 가능합니다"
   }
   ```

4. **실시간 피드백 (상담 중)**
   - 상담사가 거절대응 스크립트를 읽으면
   - "✓ 좋아요! 이건 효도 고객용입니다"
   - "⚠️ 이건 신혼부부용이에요. 이 고객에겐 맞지 않을 수 있어요"

**예상 효과:**
- 세그먼트별 최적 스크립트 적중율: 40% → 80%
- 전환율: +8~12% (세그먼트별 맞춤형)
- 월간 추가 수당: 100건 × 150만원 = 1.5억원 회수

### 구현 난이도
- **기술 난이도:** ⭐⭐⭐⭐ (4/5)
  - 세그먼트 감지 AI 로직 (Claude API + prompt engineering)
  - 세그먼트별 스크립트 필터링 (playbook DB 쿼리 수정)
  - 실시간 피드백 로직 (상담 중 스크립트 선택 감지)
  - 거절대응 변형 테이블 추가 (5 세그먼트 × 10 거절 = 50개 행)
  
- **조직 변화:** ⭐⭐ (2/5)
  - 상담사: "어떤 타입인가요?" 질문 추가 필요 (교육 2시간)
  - AI 제안 신뢰도가 낮으면 수동 선택 가능
  
- **비용:** $$$ (개발 40시간, Claude API 비용 월 50만원)

### 우선순위
- **1순위** (긴급)
- 근거: A(4) + B(5) + C(4) = 13점 / 난이도 4 = 효율성 3.25

### 관련 관점
- **A (상담사 효율성): 4/5**
  - 스크립트 선택 시간: -5분/콜
  - 월 1,000건 × 5분 = 83시간 절감
  - 신입도 세그먼트별 최적 스크립트 자동 제시
  
- **B (전환율): 5/5**
  - 세그먼트별 맞춤형 → 거절율 -10%
  - 월 추가 성약: 100건 (월 1.5억원 회수)
  - ROI: 매우 우수
  
- **C (전사 일관성): 4/5**
  - 모든 상담사가 세그먼트별 최적 스크립트 사용
  - 크루즈/렌탈/호텔 모두 적용 가능

### 작업지시서 초안
**렌즈 2 구현을 위해 필요한 것:**
1. 세그먼트 감지 AI 로직 (Claude API)
   - `/api/tools/detect-persona` 새 엔드포인트
   - Prompt: "고객 정보(나이, 여행 동반자, 예산) 입력 → 5가지 페르소나 중 1개 판정"
2. DB 스키마 추가
   - ObjectionResponse 테이블: objection_id, persona_type, response_text
3. playbook-viewer 수정
   - Persona 선택 시 필터링 로직 (5줄)
   - 거절대응 변형 표시 (10줄)
4. 거절대응 변형 데이터 입력 (4시간)
   - 10개 거절 × 5 세그먼트 = 50줄 엑셀 입력 후 DB 마이그레이션
5. 테스트: 5가지 시나리오별 콜 시뮬레이션 (4시간)

## 렌즈 3: 실시간 심리학 가이드 (On-Call Assistant)

### 현재 상태 (As-Is)
- **현황:**
  - `/tools/playbook`에서 "성공사례" 탭에 3~5개 예시만 있음
  - 콜 피드백 AI는 후분석만 가능 (콜 이후 "너 이 부분에서 실패했어" 피드백)
  - 실시간으로 상담 중에 심리학 기법 가이드 없음
  
- **한계점:**
  - 상담사가 "지금 어떤 심리 원리를 써야 하나?" 모름
  - PASONA의 "A(Agitation)" 단계에서 "Loss Aversion" 써야 한다는 걸 기억 못 함
  - 베테랑만 직관으로 알고, 신입은 매뉴얼 찾기 바쁨

### 문제점 분석 (Problem)
**왜 문제인가?**
- 신입이 "이 고객이 거절하려는데 뭘 해야 해?" 상황에서 대응 불가
- 베테랑: "그냥 고객이 원하는 거 줘" → 전사 일관성 없음
- 심리학 기법 미사용 → 전환율 flat (개선 안 됨)
- 손실: 월 1,000건 중 300건 (30%)이 "제대로 된 심리 기법 미사용" 상담

**수치화된 손실:**
- 300건 × 평균 150만원 × 전환율 차이 5% = 월 2.25억원

### 개선 제안 (Solution)
**구체적 개선안:**

1. **실시간 심리학 프롬프트 팝업**
   - 상담사가 "문제 발굴 중" 상태 입력
   - 시스템: "💡 다음: Loss Aversion 기법으로 고객 불안감 유발하세요"
   - 예: "숙소가 없으면 여행 전체가 무산될 수 있어요"

2. **심리학 기법 라이브러리 (상담 단계별)**
   ```
   [문제 발굴]
   - Affinity (공감): "네, 저도 비슷한 상황 봤어요"
   - Narrative Transportation (스토리텔링): "어떤 가족이 이 상품으로 행복했거든요"
   
   [공감 단계]
   - Loss Aversion (손실회피): "가격 미루면 선실 없어질 수 있어요"
   - Social Proof (사회증명): "이 상품은 월 500건 이상 나가고 있어요"
   
   [제안 단계]
   - Scarcity (희소성): "D-3 기간 한정 가격입니다"
   - Priming (선결정): "대부분 프리미엄 선실 선택하세요"
   
   [클로징]
   - Commitment (일관성): "아, 당신은 가족을 위해 좋은 것 선택하는 분이시네요"
   - Default (기본값): "다음달 출발로 바로 예약할까요?"
   ```

3. **상담사 손가락 센서 (옵션)**
   - 상담사가 특정 심리학 기법 사용 → 자동 감지
   - "✓ 좋아요! Loss Aversion 감지됨 (강도 7/10)"

4. **심리학 기법별 효과 실시간 표시**
   ```
   Loss Aversion: 거절 대응 시 효과 85% (이 고객 타입: 신혼부부)
   Social Proof: 객단가 상승 +15%
   Scarcity: 성약 속도 +20% (24시간 내 답변율)
   ```

**예상 효과:**
- 심리학 기법 사용율: 30% → 70%
- 전환율: +5~8%
- 월간 추가 수당: 50~80건 × 150만원 = 7.5억~12억원

### 구현 난이도
- **기술 난이도:** ⭐⭐⭐ (3/5)
  - 심리학 기법 라이브러리 DB 구축 (50줄)
  - 상담 단계별 기법 추천 로직 (30줄)
  - 팝업 알림 시스템 (Toast/Modal, 20줄)
  - 효과 실시간 계산 (간단한 쿼리)

## 렌즈 6: 교육 & 성과 추적 대시보드

### 현재 상태 (As-Is)
- **현황:**
  - 콜 피드백 UI에서 개별 상담 점수만 표시
  - 주간/월간 평균 점수 없음
  - "어떤 기법을 자주 쓰나?" 분석 불가
  - 신입/베테랑 구분 없음 (모두 동일 기준)
  
- **한계점:**
  - 팀장이 "상담사별 강점/약점" 파악 불가
  - "이달 어떻게 개선했나?" 트렌드 추적 안 됨
  - 신입과 베테랑을 다르게 관리 못 함
  - 팀 전체 약점 (예: 클로징) 교육 기회 놓침

### 문제점 분석 (Problem)
**왜 문제인가?**
- 팀장이 성과 관리를 감으로만 함 → 공정하지 않음
- "OOO님 좀 더 잘하세요" 피드백만 가능 → 구체적 코칭 불가
- 신입은 "점수 낮으니까 열심히" 동기 안 올라옴
- 손실: 팀 전체 개선 없음 (매달 동일한 성과)

**수치화된 손실:**
- 월 점수 정체 → 월간 추가 수당 기회 상실: 월 3억원

### 개선 제안 (Solution)
**구체적 개선안:**

1. **상담사별 종합 성과 대시보드**
   ```
   ┌─────────────────────────────────────────────┐
   │ 성과 대시보드: 김상담 상담사                   │
   ├─────────────────────────────────────────────┤
   │ 📊 주간 평균 점수: 72점 (↑ 5점 ↑↑ 우상향)      │
   │ 📈 월간 평균: 68점 (↑ 3점)                   │
   │ 🎯 목표 (이달): 75점 (달성도: 96%)           │
   │                                           │
   │ 💪 강점:                                   │
   │  - 오프닝: 85점 (팀 평균 72점) ⭐⭐⭐     │
   │  - Affinity 기법: 12회/월 (팀 평균 8회)   │
   │                                           │
   │ 🔧 약점 & 개선 과제:                        │
   │  - 클로징: 58점 (팀 평균 70점) ⚠️         │
   │  → 이번 주 마이크로-러닝: "클로징 5가지"    │
   │  → 다음 상담에서 "Commitment 기법 1회 필수" │
   │                                           │
   │ 📅 학습 계획:                               │
   │  - 완료: "거절대응 심화" (이번 주)          │
   │  - 진행중: "세그먼트별 클로징" (2/3 완료)   │
   │  - 예정: "심리학 기법 마스터" (다음 주)     │
   └─────────────────────────────────────────────┘
   ```

2. **팀 전체 성과 비교 (리더보드)**
   ```
   순위 | 이름    | 주간 평균 | 오프닝 | 니즈발굴 | 거절대응 | 클로징 | 추세
   -----|--------|---------|------|-------|-------|--------|-----
   1   | 김상담   | 75점    | 85   | 78    | 72    | 68    | ↑
   2   | 이상담   | 72점    | 80   | 75    | 68    | 65    | →
   3   | 박상담   | 70점    | 75   | 72    | 70    | 62    | ↓
   4   | 최상담   | 68점    | 70   | 68    | 65    | 58    | ↓↓
   5   | 신상담   | 62점    | 65   | 60    | 58    | 55    | ↑

   팀 약점: 클로징 (평균 61점)
   👉 이번 주 팀 교육: "클로징 기법 워크숍"
   ```

3. **신입 vs 베테랑 다른 기준**
   ```
   신입 (입사 3개월 이내):
   - 목표: 65점 이상

   베테랑 (1년 이상):
   - 목표: 80점 이상
   - 강점 더 강화 (예: 이미 오프닝 80점 → 90점 목표)
   - 약점은 무시 (이미 적응했으니까)
   - 월간 신입 코칭 역할 부여
   ```

4. **기법별 사용 현황**
   ```
   기법 이름        | 김상담 | 이상담 | 박상담 | 팀평균 | 효과
   --------------|-------|-------|-------|--------|------
   Loss Aversion  |  12회 |  8회  |  5회  |  8회   | 85%
   Affinity       |  12회 |  10회 |  8회  |  10회  | 90%

   👉 팀에서 "Social Proof" 기법 자주 안 씀 → 교육 필요
   ```

5. **월간 성과 보너스 시스템 (자동화)**
   ```

   - 월간 점수 +10점 이상 상승: +보너스 50만원
   - 팀 전체 평균 70점 이상: +팀 보너스 50만원
   ```

**예상 효과:**
- 상담사 동기: +30% (명확한 목표 + 보너스)
- 월간 점수 상승: 1~2점 (경쟁 효과)

**수치화된 손실:**

## 렌즈 8: 콜 플레이북 지식 관리 (RAG + 자동 학습)

### 현재 상태 (As-Is)
- **현황:**
  - 플레이북 데이터는 고정된 JSON (DB에 수동 입력)
  - 메뉴 #37 완료 후 67개 Q&A 통합 (크루즈봇 RAG)
  - 새로운 성공사례가 생겨도 플레이북에 자동 반영 안 됨
  
- **한계점:**
  - 신입 상담사의 성공 콜이 나도 플레이북에 추가 안 됨 (수동 리뷰 필요)
  - 거절 대응 성공사례가 쌓여도 분류 불가
  - "요즘 뜨는 고객 유형" 자동 감지 안 됨

### 문제점 분석 (Problem)
**왜 문제인가?**
- 플레이북이 "살아있는 문서"가 아니라 "고정된 매뉴얼" 취급
- 좋은 아이디어가 나와도 일선에만 머물고, 조직 지식이 안 됨
- 신입도 같은 문제로 또 거절당함 (선배가 해결한 방법을 모름)
- 손실: 월 1,000건 중 200건 (20%)이 "이미 해결된 거절" 반복

**수치화된 손실:**
- 200건 × 전환율 개선 2% × 150만원 = 월 6천만원

### 개선 제안 (Solution)
**구체적 개선안:**

1. **성공 콜 자동 학습 (RAG)**
   - 상담사가 "콜 피드백 AI" 분석 → "우수등급(80점+)"
   - 시스템: "이 콜을 '학습 사례'로 저장할까요?" 자동 제시
   - 상담사 확인 → "✓ 맞아요" 클릭
   - 시스템: 콜 텍스트 → RAG 인덱싱 → 플레이북 동적 추가
   ```
   예시:
   [기존 플레이북]
   거절대응 > 가격 거절 > "분납 옵션 있어요"
   
   [신입 성공사례 추가]
   거절대응 > 가격 거절 > "분납 옵션 있어요"
   + "✨ 신규: 신혼부부한테는 '신혼의 추억' 강조 + 분납 → 80% 성공"
   + [실제 콜 텍스트 요약]
   ```

2. **거절 유형별 자동 분류**
   - 콜 피드백 AI가 거절 대응 분석
   - "가격 거절", "일정 거절", "필요도 낮은 거절" 자동 분류
   - "이번 달 가장 자주 나오는 거절: 가격 (30%)"
   - 팀장에게 알림: "👉 이번 주 팀 교육: 가격 거절 대응"

3. **"요즘 뜨는 고객 유형" 자동 감지**
   - 월별로 들어오는 고객 세그먼트 분석
   - "이번 달: 신혼부부 40%, 가족 35%, 혼자 여행 25%"
   - 지난달: "신혼부부 25%, 가족 40%, 혼자 여행 35%"
   - 시스템: "신혼부부 급증 (+15%)" 알림
   - 팀장: 신혼부부용 스크립트 강화 가능

4. **플레이북 버전 관리**
   - 플레이북 v1.0 (기본)
   - 플레이북 v1.1 (1월 학습 성공사례 추가)
   - 플레이북 v1.2 (2월 학습 성공사례 추가)
   - ...
   - 상담사가 "최신 버전 사용할까?" 선택 가능

5. **기법 효과 추적 (A/B 테스트 자동화)**
   - RAG: "가격 거절 대응"
     - 기법 A: "분납 옵션" (기존, 성공 70건, 성공율 60%)
     - 기법 B: "신혼의 추억 강조 + 분납" (신규, 성공 28건, 성공율 80%)
   - 시스템: "기법 B가 20% 더 효과적이에요! 추천합니다"

**예상 효과:**
- 반복 거절 감소: 20% → 5%
- 월간 추가 성약: 150건 (20% 중 75% 해결)
- 플레이북 자체개선 주기: 월 1회 (자동화)
- 팀 학습 문화: "좋은 아이디어가 바로 팀 지식이 되네"

### 개선 제안 (Solution)
**구체적 개선안:**

1. **실험 설계 자동화**
   ```
   [새 실험 시작]
   1. 假설: "신혼부부에게는 '추억' 강조가 더 효과적일 것 같아"

   ```

2. **자동 그룹 분할 (A/B 할당)**
   ```

   - A 그룹: 기존 스크립트 (제어)
   - B 그룹: 신규 스크립트 (실험)
   ```

### 문제점 분석 (Problem)
**왜 문제인가?**
- 브랜드 일관성 없음 → NPS 하락 (고객이 "어쩐지 달라" 느낌)
- 신입이 "어떤 톤으로 말해야 하나?" 매번 고민 → 피로
- 외부 평가: "마비즈는 톤이 일관성 없다" 부정적 평판
- 손실: NPS 하락 + 리뷰 부정적 → 재구매율 -5%

**수치화된 손실:**
- 월 1,000건 중 10% (100건) 재구매 못 함
- 100건 × 150만원 = 월 1.5억원
