## 🎯 핵심 메시지

**"당신이 원하는 배, 지금 자리가 줄어들고 있어요. 3주 후엔 다 찰 거 같아요."**

L6 고객의 심리:
```
"언제 가는 게 좋을까요?"
→ 의사결정 미루기
→ "지금 해도 되지, 3주 후에 해도 되지" (긴급성 없음)

우리의 재정의:
"지금이 마지막 기회예요.
당신이 원하는 배, 3주 후에는 못 타실 거예요."
→ Loss Aversion (손실 회피) 극대화
```

## 📊 시각화 #2: 가격 변동 (지금 vs 나중)

### 배별 가격 변동 추이

```
지중해 7박 (로얄 캐리비안)

가격 변동:
┌─────────────────────────────────────┐
│ 오늘 (자리 8개)                      │
│ 오션뷰: $1,200 ← 지금 이 가격      │
│ 발코니: $1,500                       │
├─────────────────────────────────────┤
│ 1주 후 (자리 4개)                    │
│ 오션뷰: $1,350 ↑ +$150             │
│ 발코니: $1,650 ↑ +$150             │
├─────────────────────────────────────┤
│ 2주 후 (자리 2개)                    │
│ 오션뷰: $1,500 ↑ +$300 (총)        │
│ 발코니: $1,800 ↑ +$300 (총)        │
├─────────────────────────────────────┤
│ 3주 후                               │
│ (매진 - 다른 배로 변경)              │
│ 동남아 7박: $2,000 ↑ +$800         │
└─────────────────────────────────────┘

당신의 손실:
✅ 오늘 예약: $1,200 (저가)
⚠️ 1주 후: $1,350 (차이 $150)
❌ 2주 후: $1,500 (차이 $300)
❌ 3주 후: 다른 배 $2,000 (차이 $800)

→ "3주를 미루면, $800을 더 쓰는 거예요"
```

## 📊 시각화 #4: 손실 금액 계산

### 시간 경과에 따른 누적 손실

```
당신이 '타이밍을 미루면' 잃는 것:

오늘 예약 vs 3주 후 예약:

┌─────────────────────────────────────┐
│ 배선택                              │
│ 지중해 vs 지중해 (못 타므로 동남아 변경)
├─────────────────────────────────────┤
│ 가격 변동                            │
│ $1,200 vs $2,000 = +$800           │
├─────────────────────────────────────┤
│ 선실 변동                            │
│ 오션뷰 vs 인테리어 = -$300가치     │
├─────────────────────────────────────┤
│ 총 손실                              │
│ $800(가격) + $300(가치) = $1,100   │
└─────────────────────────────────────┘

→ "3주를 미루면, $1,100을 더 쓰는 거예요.
   당신의 휴가는 배도 다르고, 비용도 더 커요."
```

## 📊 CRM 자동 추적 (L6 고객용)

### ContactLensL6Data 테이블 생성 필요

```sql
CREATE TABLE ContactLensL6Data (
  contactId UUID,
  preferredShip VARCHAR(100),  -- "Royal Caribbean지중해"
  seatAvailability INT,         -- 8 → 6 → 4 → 2 → 0
  priceAtDay1 DECIMAL(10,2),   -- $1,200

);

├─ 매 3시간마다 자리 수 업데이트
```
