## 1. 핫DB 설계 정본 (콜 심리 반영)

- **버튼 = 콜 검증 신호**: 매 A/B에 명시적 `intentDelta`+`signalTag`(가격/신뢰/가족/건강/기항지/시간/연기/경쟁사/환불 = 콜 9대 반론). "바로 상담·신청·올해 안에"=+, "둘러보기"=-.
- **ⓑ는 콜드 아님 = 고관여 신중형**: 가격·믿음·직접질문 클릭은 *팩트 확인 후 사는 최우량*. 별도 태그·별도 큐.
- **자격검증(BANT) 버튼**(신청 직전, 큰 단일선택, 입력창 금지): **시기**[올해안에/내년봄/천천히] + **동행**[부부/효도/친구/혼자]. '올해 안에'=heat++. 예산은 직접 안 묻고 33,000원 클릭으로 대용.
- **니즈증폭**: ①미래 페이싱(가고 싶은 장면 먼저) → 이후 걱정이 손실회피로 전환 ②후기 타이밍 전진(문제 직후 동년배 후기) ③건강·가족 반론 선제(50대 숨은 최대반론).
- **반론 누적 태그**: 마지막 1개(현 구조) 말고 **여정 전체 누적** → 판매원이 "가격+건강 둘 다 걸림" 파악 = 공략 설계도.
- **DB 진입**: 신청 시 BotConversation upsert(intentScore+이름+전화+시기/동행+누적반론) → 기존 `notifyAgentHotLead`(핫리드 SMS)+`/api/bot/leads`(heat 정렬) 합류.

## 2. Phase 구현 (아키텍트 권장 순서 · 스키마 무관 먼저 → 스키마 순차)

### Phase A — 귀속 통일 + 크로스조직 가드 (P0 매출, 스키마 무관) [선행]
- `resolveAttribution`에 page.organizationId 전달 → 타 조직 last_touch/shortlink 폴백 차단(페이지 조직 내 에이전트만).
- `isSelfAttribution` chat에 배선(에이전트 자기링크 테스트 자기귀속 방지).
- register가 `resolveAttribution({ref, visitToken})` 서버재조회로 귀속 확정. register SMS(createdByUserId)와 핸드오프 SMS를 **단일 경로(notifyAgentHotLead)**로 통일(이중발송/오너 불일치 제거).
- 파일: bot-attribution.ts, bot/chat/route.ts, landing-pages/[id]/register/route.ts.

### Phase B — 버튼 플로우 서버화 + heat 신뢰화 (D1, 스키마 무관)
- GUIDE 그래프를 **서버 코드 상수 `FLOW_V1`**(버전드)로 이동(클라 번들 카피 방지=해자). 클라는 `현재 노드 id + A/B 선택`만 전송 → 서버가 엣지 유효성·`intentDelta`·`signalTag` 결정. 누적 heat = HMAC 서명 쿠키(기존 signAttributionToken 패턴).
- 버튼마다 intentDelta/signalTag 부여(D1 해소). ⓑ=고관여 태그.
- 커스터마이징 불가: 그래프는 코드 상수, botConfig는 productCatalogIds·persona·featured·hookText·flowVariant(코드정의 중 선택)만. 무한루프/고아노드 빌드타임 차단.
- 파일: 신규 src/lib/bot-flow.ts(FLOW_V1+heat), BotLandingClient.tsx(분기 렌더), 신규 api/bot/flow(노드 진행·heat 적립).

### Phase C — 정적 이미지 ASCII 복사 + 코드 맵 (저위험, 병렬 가능)
- 선택 13장을 ASCII 안정명으로 복사: `/public/bot/guide-01-opening.webp` … `guide-13-review.webp`. 코드 맵(BOT_IMAGES 스타일). 한글·공백·괄호 경로 Vercel 서빙 리스크 회피.
- 가이드=정적 author asset / 인라인 채팅=Drive 프록시 (2종 분리·문서화).

### Phase D — register → BotConversation upsert (핫DB 진입, **스키마1**) [Phase A·B 후]
- register는 Contact/그룹/퍼널 SoR 유지. flow==='bot-gate' 성공 직후 `visitorId+landingPageId` **멱등 upsert**로 BotConversation(이름·전화·시기/동행·누적반론·서버검증 heat) 생성, 귀속 동결.
- 2-tier 승격: 익명 버튼클릭=경량(convo 미생성), 채팅 오픈 or 리드제출 시 convo. 콜드 버튼-only 호출 안 함.
- **스키마1**: BotConversation `@@unique([visitorId,landingPageId])` + `source String` + `guideHeat Int?` + (선택)`qualifiers Json?`·`objectionTags Json?`. 운영 수동 migrate 스크립트(scripts/apply-*.mjs).

### Phase E — 컴플라이언스 (D2, P0 법적 — 배포 게이트)
- **정적 카피 린트**: bot-guardrail의 BANNED_PHRASE + RECRUIT_GUARANTEE 사전을 빌드/저장 시점에 버튼·랜딩 하드코딩 문구에 적용("가드가 막는 말은 버튼에도 못 쓴다" 기계화).
- "100% 환불보장"→"청약철회 7일내 100% 환불(조건·기간 명시)". 최저가·무조건 전수 제거.
- 게이트 **동의 분리**: (필수)수집·이용 5종 고지 + (선택)광고수신 별도 체크박스 + (선택)소속 지사/판매원 제3자 제공. 끼워넣기 금지. 처리방침·수신거부 자가링크.
- 33,000원 골드 = 전자상거래법 13·17조 고지(가격·청약철회·사업자정보), 무료/유료 시각 분리.
- **세일즈 게이트 수익/부업/파트너 언급 0**(다단계·소비자원 패턴 차단). checkRecruitOutputGuard 사전도 세일즈 버튼 정적 린트에 포함.
- 후기·카톡: 대가성 시 광고표기 + 타인 PII 마스킹 + "일부 각색". 긴박성 카운터=실 DB값 바인딩만.

### Phase F — 지정 공유(지사/점장) + 개인링크 (**스키마2**) [세 파일 원자적]
- CrmLandingShare에 `sharedToUserId String?` + `@@index` + **부분 unique 인덱스**(NULL distinct 문제 → 센티넬/부분 unique). `CREATE UNIQUE INDEX CONCURRENTLY`.
- **세 곳 동시 수정**(격리 붕괴 방지): ①share POST(타깃이 활성 OrganizationMember 검증·그 org로 sharedToOrgId 채움·열거공격 차단) ②목록 GET(`sharedToUserId===ctx.userId` OR org-share OR global, 타인 지정분 격리) ③clone-shared(동일 OR 유효성).
- 권한경계: 지사 OWNER→자기 org 점장(intra-org) / 본사 GLOBAL_ADMIN→지사(org-간). 타 지사 점장 공유는 GLOBAL_ADMIN만.
- 관리자 UI: "공유 대상(지사/점장) 선택" → 받은 사람이 복제(B4 복제경로 기존).

### Phase G — 시뮬검수 버튼 모드 (검증, 배포 게이트)
- scripts/bot-eval에 버튼 플로우 모드: (1)정적 카피 린트(BANNED+고지의무 체크리스트) (2)페르소나 경로 시뮬(가격민감/의심/동반자결정/즉시구매/정보만, 화면별 통과·이탈·핫DB제출율·게이트도달 클릭수) (3)게이트 PASS(카피위반0 AND 제출율≥기준 AND 특정화면 이탈 30%↑ 집중 없음).
- 환류 2분류: **카피(콘텐츠) 문제**→문구수정 / **플로우(구조) 문제**→화면순서·분기수정.

## 3. 전환율 지표 (단일 22% 폐기 — 분리 정의)
① 방문→폼제출(CVR, 콜드 정직밴드 8~15%) ② 폼→유효통화연결 ③ 통화→골드결제(리드의 20~40%) ④ 골드→크루즈. **heat 가중**(핫 intent≥68 vs 콜드 결제율 분리). 운영설계는 보수(방문→리드 10%, 리드→골드 25%)로.
