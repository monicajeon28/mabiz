## 📋 Section 1: 토론 - 왜 20렌즈가 아니라 10렌즈인가?

### 문제 진단: 현행 20렌즈의 구조적 결함

#### 문제 1. "생애 단계" 기준은 판매 현장과 안 맞는다

**현행 구조**:
```
Group A: 신혼부부 (#1-5) → 시간부족, 효율성 강조
Group B: 40대가족 (#6-10) → 자녀경험, 마지막기회 강조
Group C: 중년부부 (#11-15) → 정체성 회복, 부부시간 강조
Group D: 고령층 (#16-20) → 건강 관리, 손주기억 강조
```

**실제 판매 현장**:
- 신혼부부 #4(저소득) 전화 vs 신혼부부 #2(프리랜서) 전화 → **고통점 완전히 다름**
  - #4: "비싼데? 정말 150만원인가?" (가격 오해)
  - #2: "일정이 나올 때까지는 미뤄야 하는데, 그때도 가능해?" (타이밍 미결)
  
- 40대가족 #8(3자녀) 전화 vs 40대가족 #6(자녀초등) 전화
  - #8: "3명이면 비용이..." (가격 오해)
  - #6: "준비가 복잡하지 않을까?" (준비 부담)

**결론**: 같은 그룹 내에서 다른 콜 스크립트가 필요하다. 생애 단계별 분류는 의미 없다.

---

#### 문제 2. "인플루 광고 고객"과 "콜드 고객"의 저항이 다른데, 같은 스크립트로 처리한다

**시나리오 A: 광고 본 고객**
- "월 33,000원 크루즈" 광고 보고 전화
- 실제 상품비 설명("150-300만원") 들으면 → "아, 광고가 사기네요?"
- **현행 대응**: "가격 비교표로 가성비 설득" (Phase 3)
- **문제**: Phase 1-2에서 이미 끊음. 광고 불신이 해소되지 않음

**시나리오 B: 첫 크루즈 고객**
- "크루즈가 뭐예요?" 상태로 전화
- 일반 여행과의 차이를 이해하지 못함 (배=이동수단으로만 인식)
- **현행 대응**: "일반여행 vs 크루즈" 비교표 (Phase 2-3)
- **문제**: 텍스트 설명으로는 차별성이 전달 안 됨. 멀티미디어 자료 필요

**결론**: 고객 source별로 Phase 0의 오프닝부터 다르게 설계되어야 한다.

---

#### 문제 3. "부재중 DB"(부재중고객)가 20렌즈에 독립 렌즈로 없다

**현행**: "기존 고객"을 특별히 다루는 렌즈 없음. 모두 신규 고객 기준 스크립트.

**실제 부재중 고객의 오해**:
- "지난번 갔다왔는데, 왜 계속 멤버십을 유지해야 돼요?"
- "스팟으로 필요할 때마다 하면 되지 않나요?"
- "할인이 뭐가 좋아요?"

**현행 대응**: 콘텍스트 파일에 개념만 있고, 실제 스크립트 없음.

**결론**: "이미 경험 있는 고객"의 저항은 신규 고객과 완전히 다르다.

---

### 핵심 통찰: "누가 전화 받는가" vs "왜 전화를 끊는가"

| 기준 | 현행 20렌즈 | 신규 10렌즈 |
|-----|---------|---------|
| 분류 기준 | "누가": 생애 단계 | "왜": 구매 저항 |
| 콜 스크립트 수정 | 그룹 당 1개 → 전체 일관성 | 저항 당 1개 → 표적 전환 |
| Phase 0 오프닝 | "신혼부부 맞죠?" → 같음 | "뭐가 궁금해요?" → 다름 |
| 장점 | 관리 단순 | 판매 효율 ↑ |
| 약점 | **전환율 낮음** | 관리 복잡 |

## 🎯 Section 2: 10렌즈 정의 및 PASONA/SPIN 매핑

### 10렌즈 완전 정의

#### L1. 가격 오해형 - "크루즈는 비싸다"

**고객 프로필**:
- 신혼 #4(저소득), 40대 #8(3자녀), 중년 #14(재정불안) + **인플루 광고 본 고객**
- 특징: "월 33,000원이라고 했는데 왜 150만원이?" 충격
- Pain Point: 광고 불신 + 가격 접근성

**PASONA 매핑**:
```
Problem   → "광고는 월 33만원이라고 했는데..."
Affinity  → "많은 고객들이 처음에 같은 질문을 해요"
Solution  → "올인클루시브라서 일반 여행(210만원)보다 저렴합니다"
Offer     → "분할 6개월, 월 27만원 + 크루즈 280만원 → 월 47만원"
Narrowing → "가입은 월 33,000원, 상품은 따로. 언제든 멈춰도 됨"
Action    → "지금 가입하면 이번달 10% 할인 + 48시간 우선예약"
```

**SPIN 적용**:
```
Situation: "혹시 크루즈가 비싸다고 생각하세요?" (상황 파악)
Problem:  "그럼 일반 여행이랑 비용을 어떻게 비교해보셨어요?" (문제 명확화)
Implication: "혹시 비용이 문제라면, 올인클루시브를 알면 생각이 달라지지 않을까요?" (함축성)
Need-Payoff: "크루즈가 일반 여행보다 저렴하다면 가입 고려하실 거죠?" (해결가치)
```

**콜 스크립트 구조**:
- Phase 0-1: 광고 불신 해소 (5분)
  - "월 33,000원은 멤버십 가격. 크루즈 탈 때는 별도" 먼저 명확화
  - 일반 여행 비용 자동 계산기 제시 (220만원 = 비행기 80+호텔 80+식사 40+가이드 20)
  
- Phase 2: 가성비 극대화 (3분)
  - "올인클루시브로 150-280만원 → 일반 여행 210-300만원보다 저렴"
  - "평생 할인" 개념 (5년 누적 절약 계산)
  
- Phase 3: 선택지 제시 (2분)
  - "상품만 먼저 사고 멤버는 나중에"
  - "분할 6개월 플랜"
  
- Phase 4: 이의처리
  - Q: "정말 비용이 다 포함돼요?"
  - A: "음식, 객실, 쇼, 기항지 입장료 다 포함. 추가는 주류, 카지노, 스파뿐"

**SMS 3일 시퀀스**:
```
Day 0: "월 33,000원 = 멤버, 크루즈비 별도" 명확화 (오해 해소)
Day 1: 일반여행 vs 크루즈 비용 비교표 (가성비 증명)
Day 2: 고객 후기 (누가 말해도 안 들으면, 고객이 말해주기)
Day 3: 할인 계산 + 긴급성 (선실 한정)
```

**예상 전환율**: 25% → 42% (+68%, 현 평균 34.5%에서 +7% 상향)

#### L2. 준비 부담형 - "준비가 복잡할 것 같다"

**고객 프로필**:
- 신혼 #1(맞벌이), 신혼 #3(임원), 40대 #6(초등자녀), 중년 #12(자영업)
- Pain Point: 시간 부족 + 여행 준비 경험 부담

**PASONA 매핑**:
```

Affinity  → "맞아요, 호텔/식당/일정 찾는데 8시간 걸려요"
Solution  → "크루즈는 배가 다 준비해줍니다"
Offer     → "준비 0시간 → 짐만 싸면 끝"
Narrowing → "비행기 예약? 환전? 숙박 예약? 다 끝. 배만 타세요"

- Phase 2: "일반여행(8시간) vs 크루즈(0시간)" 극대화

**고객 프로필**:
- 첫 크루즈 모든 고객, 특히 #2(프리랜서 자유도 강조), #6(초등자녀와 시간), #16(은퇴예정)

**PASONA 매핑**:
```

Affinity  → "많은 분들이 그렇게 생각해요. 배를 이동수단으로만 알아요"
Solution  → "배 자체가 리조트입니다. 목적지가 아니라 경험의 무대예요"
Offer     → "선셋 갑판, 커뮤니티, 기항지 포함, 밤하늘 아래"
Narrowing → "3박 4일 중 배 위에서 2박을 지낸다는 뜻"

**콜 스크립트**: (신규 작성 필수)
- Phase 0-1: "크루즈를 배로만 아셨나요?" (오해 명확화)
- Phase 2: "배 = 리조트" 개념 전환
  - "호텔(배의 객실) + 레스토랑(배의 식당) + 쇼(배의 극장) + 수영장 + 스파 + 카지노"
  - "목적지가 바뀌면서 창밖 경험도 자동으로 바뀜"
  
- Phase 3: "실제 고객 후기"
  - "첫 크루즈: '아, 호텔이 움직이네요!'"
  - "배 위에서의 밤하늘 경험이 다릅니다"

**SMS 3일 시퀀스**:
```
Day 0: "배 = 리조트" 개념 사진 (갑판, 객실, 레스토랑)
Day 1: 고객 후기 영상 (첫 크루즈의 놀라움)
Day 2: 기항지와 배의 차이 설명 (배가 리조트, 기항지가 모험)
Day 3: 첫 크루즈 고객 특별 패키지 + 긴급성
```
