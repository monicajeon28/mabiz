# 판매관리 페이지 역할별 권한 설계 (RBAC)
**작성일**: 2026-06-18  
**상태**: 설계 완료  
**난이도**: 초등학생 수준 설명

---

## 🎯 핵심 원칙

판매관리 페이지는 **"마무리 확인작업"** 입니다.
- CRM에서 이미 계산된 수당을 보고 확인하는 곳
- 수정/삭제 권한은 없음 (조회만 가능)
- 역할별로 "누가 뭘 볼 수 있는지" 만 다름

---

## 📋 역할 3가지

### 1️⃣ 관리자 (GLOBAL_ADMIN)

**이름**: "전체 보는 사람"  
**권한 핵심**: 전체 회사 판매건을 모두 봄 (개인정보 가려짐)

#### ✅ 볼 수 있는 것
```
┌─────────────────────────────────────────┐
│ 판매관리 대시보드 (관리자 전용)         │
├─────────────────────────────────────────┤
│                                         │
│ 📊 전체 이번 달 매출: ₩12,345,678       │
│    (결제 완료 123건)                    │
│                                         │
│ 👥 대리점별 매출 분석                   │
│    ├─ 서울지점: ₩5,000,000             │
│    ├─ 부산지점: ₩4,500,000             │
│    └─ 대구지점: ₩2,845,678             │
│                                         │
│ 🔗 내 개인 링크 매출: ₩145,678          │
│    (내가 직접 만든 랜딩페이지)          │
│                                         │
│ 📋 최근 결제 내역 (이름+전화 완전 보임) │
│    ├─ 김철수 / 010-1234-5678          │
│    ├─ 이영희 / 010-9876-5432          │
│    └─ 박만수 / 010-5555-5555          │
│                                         │
└─────────────────────────────────────────┘
```

#### 🔒 마스킹 규칙
- **이름**: 완전하게 본다 (김철수)
- **전화번호**: 완전하게 본다 (010-1234-5678)
- **개인정보 노출**: 가능 (완전 신뢰)

#### 📹 기록되는 것 (감시)
```
[감사 로그]
시간: 2026-06-18 14:30:00
관리자: 모니카 (admin1)
행동: "전체 판매관리 페이지 조회"
본 데이터: 전체 회사 결제 123건
기록 위치: 데이터베이스 감사로그
```

---

### 2️⃣ 대리점장 (AGENT)

**이름**: "팀 보는 사람"  
**권한 핵심**: 자기 팀 판매건과 자기 판매건만 봄

#### ✅ 볼 수 있는 것
```
┌─────────────────────────────────────────┐
│ 판매관리 (팀원만 볼 수 있음)            │
├─────────────────────────────────────────┤
│                                         │
│ 📊 우리 팀 이번 달 매출: ₩5,000,000     │
│    (결제 완료 45건)                     │
│                                         │
│ 👥 팀원 판매건 (5명 팀원)               │
│    ├─ 이순신 판매원: ₩2,000,000        │
│    ├─ 강감찬 판매원: ₩1,800,000        │
│    ├─ 세종대왕: ₩800,000               │
│    ├─ 김영삼: ₩200,000                 │
│    └─ 나: ₩200,000 (내 판매건)         │
│                                         │
│ 📋 최근 결제 내역 (이름+전화 마스킹)    │
│    ├─ 김* / 010-1234-****              │
│    ├─ 이* / 010-9876-****              │
│    └─ 박* / 010-5555-****              │
│                                         │
│ ⚠️ 다른 팀 매출은 보이지 않음           │
│    (부산지점, 대구지점 볼 수 없음)      │
│                                         │
└─────────────────────────────────────────┘
```

#### 🔒 마스킹 규칙
- **이름**: 첫 글자만 보임 (김*)
- **전화번호**: 뒤 8자리 가림 (010-1234-****)
- **개인정보 보호**: 강함 (신뢰 적음)

#### 🚫 할 수 있는 것
- ✅ 팀원 판매건 **조회만** 가능
- ✅ 불일치 발견하면 "알림" 가능 (관리자에게 보고)
- ❌ 판매건 수정 불가능
- ❌ 삭제 불가능
- ❌ 다른 팀 보기 불가능

#### 📹 기록되는 것 (감시)
```
[감시 로그] — "팀장이 팀원 판매건 검증했음"을 기록
시간: 2026-06-18 14:30:00
팀장: 임꺽정 (부산지점장)
행동: "팀 판매관리 페이지 조회"
본 데이터: 부산지점 팀원 45건만 조회
기록 위치: 팀장 행동 로그
```

### 3️⃣ 판매원 (FREE_SALES)

**이름**: "내 것만 보는 사람"  
**권한 핵심**: 자기 판매건만 봄 (다른 사람 판매건 안 봄)

#### ✅ 볼 수 있는 것
```
┌─────────────────────────────────────────┐
│ 나의 판매건 (개인용)                    │
├─────────────────────────────────────────┤
│                                         │
│ 📊 내 이번 달 매출: ₩200,000            │
│    (결제 완료 2건)                      │
│                                         │
│ 📋 내 판매 기록 (최근 6개월)             │
│    ├─ 2026-06-15 | ₩100,000 (완료)     │
│    └─ 2026-06-10 | ₩100,000 (완료)     │
│                                         │
│ ⚠️ 다른 사람 매출은 보이지 않음         │
│    (팀원 매출, 다른 지점 매출 안 보임)  │
│                                         │
└─────────────────────────────────────────┘
```

#### 🔒 마스킹 규칙
- **자기 판매건**: 이름, 전화 완전 보임 (참고용)
- **다른 사람**: 볼 수 없음 (애초에 조회 차단)

#### 🚫 할 수 있는 것
- ✅ 자기 판매건 **조회만** 가능
- ❌ 수정 불가능
- ❌ 삭제 불가능
- ❌ 다른 사람 판매건 볼 수 없음

#### 📹 기록되는 것 (감시)
```
❌ 기록 없음
"판매원은 자기 것만 보는 거니까 특별한 감시 불필요"
```

## 🔐 쿼리 필터 규칙 (개발자용)

### 1. 관리자 (GLOBAL_ADMIN) — 전체 조회
```sql
-- 필터 없음: 모든 결제 조회
SELECT * FROM CrmPayAppPayment
WHERE createdAt >= 6개월전
ORDER BY createdAt DESC;
```

### 2. 대리점장 (AGENT) — 팀 필터링
```sql
-- 자신이 속한 조직의 결제만 조회
SELECT * FROM CrmPayAppPayment pp
INNER JOIN CrmAffiliateSale af ON af.orderId = pp.orderId
WHERE af.organizationId = 자신의조직ID
  AND createdAt >= 6개월전
ORDER BY createdAt DESC;
```

### 3. 판매원 (FREE_SALES) — 403 차단
```sql
-- 접근 불가 (API 레벨에서 403 Forbidden 반환)
return NextResponse.json(
  { ok: false, message: '접근 권한이 없습니다.' },
  { status: 403 }
);
```

## 📊 마스킹 규칙 상세

### 이름 마스킹
```javascript
// 함수: maskCustomerName(name)

// GLOBAL_ADMIN: 마스킹 없음
"김철수" → "김철수"

// AGENT/FREE_SALES: 첫 글자만
"김철수" → "김*"
"이영희" → "이*"
"박만수" → "박*"
"나" → "나" (1글자는 그대로)
"-" (없음) → "-"
```

### 전화번호 마스킹
```javascript
// 함수: maskPhone(phone)

// GLOBAL_ADMIN: 마스킹 없음
"010-1234-5678" → "010-1234-5678"

// AGENT/FREE_SALES: 뒤 4자리 + 앞 3자리 보임
"010-1234-5678" → "010-****-5678"  ← 이렇게 표시됨

// 빈 전화번호
"" → "" (마스킹 안 함)
null → "" (마스킹 안 함)
```

## 🎓 용어 설명 (초등학생 버전)

| 용어 | 쉬운 설명 |
|------|---------|
| **RBAC** | 역할별 권한 (Role-Based Access Control) = 사람 역할에 따라 볼 수 있는 게 다름 |
| **마스킹** | 개인정보 가리기 = 이름 "김철수"를 "김*"으로, 전화 "010-1234-5678"을 "010-****-5678"로 표시 |
| **Audit Log** | 누가 뭘 봤는지 기록하는 것 = 관리자 행동만 기록 |
| **organizationId** | 조직 ID = "서울지점", "부산지점" 같은 팀 ID |
| **403 Forbidden** | "들어갈 수 없습니다" 신호 = 판매원이 판매관리 페이지에 접근하면 나오는 화면 |
| **API** | 앱과 데이터베이스 사이의 통로 = "이 데이터 줄래?" 하면 필터링해서 줌 |
| **Null** | 없음 = 데이터가 없으면 null (빈 칸) |
