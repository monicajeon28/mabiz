# 성능 최적화 검토 - 요약 보고서 (2026-06-15)

## 🎯 현황

**Jeff Bezos의 성능 평가:** 32/100점 (확장성 기준)

마비즈 CRM은 현재 100k 고객까지만 안정적이며, 1M 고객 규모 운영은 불가능한 상태입니다.

---

## 🔴 3가지 치명적 문제 (P0)

### 1️⃣ N+1 쿼리 문제 (응답시간: 5초+ → 200ms)
- **원인:** Contact 상세 조회 시 callLogs, memos, vipSequences 자동 로드
- **영향:** 100k Contact 동시 조회 시 Vercel 타임아웃
- **해결책:** Lazy Loading (탭 클릭 시만 로드)

**예상 개선:** 25배 성능 향상

---

### 2️⃣ 인덱스 누락 (쿼리 시간: 30초 → 50ms)
- **원인:** 활성 고객(deletedAt IS NULL), 공유 고객(visibility), 작성자별 조회 시 Full Table Scan
- **영향:** 대규모 Contact 조회 시 심각한 성능 저하
- **해결책:** 3가지 복합 인덱스 추가

**예상 개선:** 600배 성능 향상

---

### 3️⃣ 메모리 누수 (메모리: 1GB → 100MB)
- **원인:** lensMetadata JSON (평균 5KB × 100k = 500MB)
- **영향:** 100k Contact 동시 조회 시 메모리 폭발 (OOM)
- **해결책:** ContactLensMetadata 별도 테이블로 분리

**예상 개선:** 10배 메모리 절감

---

## 🟡 2가지 중요 개선사항 (P1)

### 1️⃣ Race Condition (ContactGroupMember)
- **문제:** 10명이 동시에 그룹 입장 → memberCount 불일치
- **해결책:** SELECT...FOR UPDATE 트랜잭션 처리

---

### 2️⃣ SMS 대량 발송 병목 (1시간+ → 10분)
- **문제:** 1M Contact × Day 0-3 = 4M Job → DB 부하
- **해결책:** Async Job Queue (Bull 또는 Inngest)

---
