## 📊 각 탭 상세 가이드

### 탭 1: 감시 로그 (Audit Logs)

**목적**: 누가 언제 무엇을 했는지 기록 확인

#### 기본 사용

```
1. 필터 설정
   └─ 액션: "DELETE" (삭제만 보고 싶을 때)
   └─ 날짜: "7일" (최근 일주일)
   └─ 사용자: "user@example.com" (특정 사용자만)

2. 테이블 스크롤
   └─ 모든 액션 시간순 표시
   └─ 상태: ✅ SUCCESS / ❌ FAILED / ⚠️ DENIED

3. 상세 보기
   └─ 행 클릭 → 수정 전/후 상태 확인
   └─ PII 접근 기록 확인
```

#### 실제 사용 예시

**Q1: "어제 contact-abc123을 누가 수정했나?"**

```
Step 1: 액션 = "WRITE"
Step 2: 날짜 = "어제"
Step 3: 리소스 = "Contact", ID = "abc123"
Step 4: 검색 → 결과 표시
  userId: user-john
  시간: 2026-05-26 14:30:00
  상태: SUCCESS
```

**Q2: "지난주에 전화번호를 누가 가장 많이 봤나?"**

```
Step 1: 액션 = "READ"
Step 2: 날짜 = "7일"
Step 3: 필터링 결과를 보며 PII 필드 확인
  → "piiFieldsAccessed: ['phone']" 찾기
Step 4: 이상 패턴 발견 시 바로 관리자에게 알리기
```

#### ⚠️ 주의

- 감시 로그는 **절대 삭제 불가** (GDPR 준수)
- 최대 1,000개까지 한 번에 표시
- 더 오래된 데이터는 페이지 버튼으로 이동

### 탭 2: PII 접근 제어 (PII Access)

**목적**: 개인정보 접근 통계 및 이상 탐지

#### 3개 카드 읽기

```
┌─────────────────────┐
│ 어제 PII 접근: 142  │ ← 일반적인 수준
│ 전일 대비: +8%      │ ← 정상 범위
└─────────────────────┘

┌─────────────────────┐
│ 이상 접근: 2        │ ← ⚠️ 주의!
│ 검토 필요           │ ← 클릭해서 확인
└─────────────────────┘

┌─────────────────────┐
│ 마스킹된 필드: 8/10 │ ← 80% 준수
│ 전체 필드 기준      │ ← 100% 목표
└─────────────────────┘
```

#### 이상 패턴 확인

**파란 경고 박스가 표시되면**:

```
🔵 user@example.com이 지난 1시간 동안
   124개 연락처의 전화번호에 접근했습니다.
```

**조치**:

```
1. user@example.com에게 전화
   "지금 많은 전화번호에 접근하고 있네요. 무엇을 하고 계신가요?"

2. 정상 업무라면 OK
   → 감시 로그에 이유 기록

3. 비정상이라면
   → 권한 일시 중단
   → 관리자 회의

4. 모든 내용을 감시 로그에 기록
```

#### 마스킹 필드 저조

**"마스킹된 필드: 5/10"이면**:

```
1. 어떤 필드가 마스킹되지 않았는가?
   → 개발팀에 알림

2. 예시
   ❌ Contact.phone은 원본 노출
   ❌ Contact.email은 원본 노출
   ✅ Contact.name은 마스킹됨

3. 개발팀이 수정할 때까지 사용자에게
   "현재 PII 마스킹 적용 중입니다"
```

### 탭 3: 규정 준수 (Compliance Status)

**목적**: GDPR/CCPA/한국 규정 준수 현황 모니터

#### 종합 점수

```
┌─────────────────────────┐
│ 규정 준수 점수: 87%     │
│                         │
│ 상태: AT_RISK ⚠️        │ ← 60-85%: 주의 필요
│      (목표: 85% 이상)   │ ← 85% 이상: 안전 ✅
│                         │ ← 60% 미만: 위험 🚨
└─────────────────────────┘
```

**해석**:
- ✅ **COMPLIANT (85%+)**: 조치 필요 없음
- ⚠️ **AT_RISK (60-85%)**: 월간 계획 필요
- 🚨 **NON_COMPLIANT (60% 미만)**: 즉시 조치 필수

#### 규정별 상세

**GDPR (🇪🇺)**:

```
점수: 80%  (4/5 항목 준수)

준수 항목:
✅ Consent Documentation (동의 기록)
✅ Deletion Request Processing (30일 이내 처리)
✅ Audit Log Retention (7년 보관)
✅ Data Processing Agreement (DPA 체결)

미준수 항목:
❌ Encrypted Transmission (HTTPS)
   → Action: HTTPS 설정 확인 (개발팀)
```

**권장사항 (Recommendations)**:

```
1. HTTPS 설정 검증 (3일 내)
   담당자: 개발팀 리더

2. 모든 Contact 동의 확보 (30일 내)
   담당자: 영업팀
   대상: consentGivenAt 없는 항목

3. DPA 검토 및 갱신 (7일 내)
   담당자: 법무팀
   대상: Aligo, SendGrid, Neon
```

#### 문제 해결

**"GDPR: 1,234개 Contact에 동의 기록 부재"**

```
Step 1: 확인
  SELECT COUNT(*) FROM Contact
  WHERE consentGivenAt IS NULL;
  → 1,234개

Step 2: 조치 선택
  A) 기존 Contact은 동의로 간주
     → Contact 팀이 consentGivenAt 일괄 설정
  
  B) 신규 Contact만 대상
     → 신규 가입 시 동의 필수화

Step 3: 마감일 설정
  → 30일 내 해결

Step 4: 완료 후 확인
  → 점수가 올라가는지 다음날 확인
```

### 탭 4: 데이터 요청 (Data Requests)

**목적**: 사용자의 삭제/접근 요청 관리

#### 테이블 읽기

```
Contact ID | 요청 사유 | 상태 | 요청일 | 삭제 예정일 | 작업
───────────┼─────────┼────┼────────┼──────────┼─────
cont-abc   | GDPR    | 유예중 | 05-01 | 06-01 | [상세보기]
cont-def   | 재입사  | 복구됨 | 05-10 | — | [상세보기]
```

**상태 설명**:

| 상태 | 의미 | 조치 |
|------|------|------|
| 🟡 유예중 (PENDING_DELETION) | 30일 대기 중 | 모니터링만 |
| 🟠 삭제예정 (SCHEDULED_FOR_DELETE) | 오늘 삭제될 예정 | 확인 후 실행 |
| ⚫ 삭제됨 (HARD_DELETED) | 이미 삭제됨 | 기록만 유지 |
| 🔵 복구됨 (RESTORED) | 삭제 요청 취소됨 | 재활성화됨 |

#### 일일 확인

**"삭제 예정일이 오늘인 항목이 있나?"**

```
Step 1: "삭제 예정일" 열 확인
Step 2: 오늘(2026-05-27)인 항목이 있는가?
Step 3: 있다면
  → [상세보기] 클릭
  → 정말 맞는지 재확인
  → "삭제" 버튼 클릭
  → 감시 로그에 이유 기록

Step 4: 없다면
  → 다음날 다시 확인
```

#### 유예기간 경과 주의

```
❌ "유예중" 상태인데 "삭제 예정일"이 지난 항목?

Step 1: 즉시 조사
Step 2: "삭제 예정일"이 과거인가?
Step 3: 그렇다면 지연 삭제 실행
  → Cron job이 실패했을 가능성
  → 개발팀에 알림
  → 수동으로 삭제 버튼 클릭
```

## 🔧 자주 묻는 질문 (FAQ)

### Q1: "규정 준수 점수는 왜 매일 바뀌나?"

A: 매일 자동 점검(Cron job)이 실행되고, 새로운 항목이 추가되거나 기존 항목이 변할 수 있습니다.
- Contact 동의 추가 → 점수 올라감 ⬆️
- DPA 만료 → 점수 내려감 ⬇️
- 미처리 삭제 요청 → 점수 내려감 ⬇️

### Q2: "PII 접근 이상은 자동으로 경고하나?"

A: 네! 시스템이 자동 감지합니다:
- 1시간에 PII 접근 100회 이상 → 경고 🚨
- 5회 이상 실패한 로그인 시도 → 경고 🚨
- 밤 12~5시 야간 접근 → 기록만 (경고 아님)

### Q3: "감시 로그를 Excel에 내보낼 수 있나?"

A: 아직 미지원입니다. 
향후 업데이트 예정입니다. 
당분간은 CSV 수동 생성 요청:
- privacy@mabiz.com으로 연락

### Q4: "Contact를 실수로 삭제 요청했어요. 취소할 수 있나?"

A: 네! 유예기간(30일) 내라면 취소 가능합니다:
1. 데이터 요청 탭 4 → 해당 항목 찾기
2. [상세보기] 클릭
3. [취소] 버튼 클릭
4. 사유 입력 후 확인

30일 경과 후에는 불가능합니다.

### Q5: "점수를 빠르게 올리려면?"

A: 우선순위:

```
1. Contact 동의 확보 (1주)
   → 동의 없는 모든 Contact에 consentGivenAt 추가

2. DPA 검토 (3일)
   → Aligo, SendGrid 등과 DPA 유효 확인

3. HTTPS 설정 (1일)
   → 개발팀과 함께 확인

4. 기타 (30일 내)
   → 분기별 보안 점검 일정 수립 등
```
