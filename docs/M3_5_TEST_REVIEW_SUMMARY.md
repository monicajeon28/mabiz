## ✅ 핵심 테스트 시나리오

### 성공 케이스 (Green Path)

```
1️⃣ 정상 목록 조회
   GET /api/backup/passport/123
   → 200 + files[] + pagination

2️⃣ 정상 파일 다운로드
   GET /api/backup/passport/123/download/file-1
   → 200 + Binary (JSON/WebP)

3️⃣ 정상 게스트 복구
   POST /api/backup/passport/123/restore/file-1
   → 200 + restoredCount=100

예상: 모두 성공 ✅
```

### 권한 실패 케이스 (Red Path)

```
1️⃣ 다른 조직 사용자
   조직 A의 사용자 → 조직 B의 Trip 접근
   → 403 Forbidden ✅

2️⃣ 낮은 권한 사용자
   VIEWER 역할 → restore 시도
   → 403 Permission Denied ✅

3️⃣ 토큰 없음
   Authorization 헤더 없음
   → 401 Unauthorized ✅
```

### 복원력 케이스 (Resilience)

```
1️⃣ 토큰 만료
   expiresAt < now + 5min
   → 자동 갱신 + 재시도 ✅

2️⃣ 네트워크 오류
   타임아웃 또는 500
   → 3회 재시도 (지수 백오프) ✅

3️⃣ 부분 실패
   50/100 게스트 유효
   → restoredCount=50, errorCount=50 ✅
```

## 🔐 보안 검증 체크리스트

- [ ] **인증**: 토큰 없음 → 401, 유효하지 않은 토큰 → 401
- [ ] **권한**: 역할별 RBAC (OWNER/ADMIN ✅, VIEWER ❌)
- [ ] **조직 격리**: organizationId 다르면 → 403
- [ ] **감사 로깅**: 모든 액션 → PassportBackupRestoreLog 생성
- [ ] **토큰 보안**: DB에 암호화되어 저장 (AES-256)
- [ ] **IP 로깅**: 요청 출처 → ipAddress 저장
- [ ] **SQL Injection**: Prisma 파라미터 바인딩
- [ ] **XSS 방지**: 파일명 HTML escape

## 🎯 주요 검증 항목

### 1. 기능성 (Functional)

| API | 성공 케이스 | 실패 케이스 |
|-----|-----------|-----------|
| **LIST** | 파일 목록 반환 | Trip 없음 (404), 권한 없음 (403) |
| **DOWNLOAD** | 파일 바이너리 반환 | 파일 없음 (404), 권한 없음 (403) |
| **RESTORE** | 게스트 복구됨 | 파일 없음 (404), 권한 없음 (403) |

| 위협 | 검증 | 결과 |

| **조직 격리** | 다른 organizationId | 403 Forbidden ✅ |
| **토큰 위조** | 수정된 JWT | 401 Unauthorized ✅ |
