📋 6가지 핵심 SOP 요약
================================================================================

1️⃣ SOP #1: 경로 오류 수정 (Route Correction)
   기간: 1시간 | 우선순위: P0 | 담당: 개발자
   ├─ 검토: 백슬래시 패턴 확인 (15분)
   ├─ 수정: /api\landing-pages\images → /api/landing-pages/images (30분)
   ├─ 테스트: 이미지 업로드 3회 성공 (20분)
   └─ 배포: Vercel 자동 배포 (10분)

   ✅ 완료 기준: 백슬래시 패턴 0개, 업로드 실패율 0%

2️⃣ SOP #2: 이미지 압축 (Image Compression)
   기간: 2-3일 | 우선순위: P0 | 담당: 백엔드
   ├─ JPEG: 3-5MB → 300-500KB (quality: 80, 85-92% 압축)
   ├─ PNG: 2-4MB → 400-700KB (quality: 75, 80-85% 압축)
   ├─ GIF: 1-8MB → 원본 유지 (재압축 금지)
   ├─ Sharp 라이브러리로 서버 자동 압축
   └─ 클라이언트 선택적 압축 (browser-image-compression)

   ✅ 완료 기준: 평균 파일크기 2.8MB → 0.6MB (-78%)

3️⃣ SOP #3: Drive 자동 저장 (Auto-Save to Google Drive)
   기간: 3-4일 | 우선순위: P0 | 담당: 전체 팀
   ├─ 아키텍처: 하이브리드 (중요 문서만 자동화)
   ├─ Google Drive Service Account 설정
   ├─ Prisma: DriveFile 모델 추가
   ├─ API: 업로드/동기/Cron 3개 엔드포인트
   ├─ Cron Job: 1분 주기 자동 동기화
   └─ UI: 진행상황 표시 + 수동 "Drive 동기" 버튼

   ✅ 완료 기준: 자동화율 95%+, 모든 이미지 Drive에 저장

4️⃣ SOP #4: 라이브러리 복수선택 (Multi-Select Image Library)
   기간: 1-2일 | 우선순위: P0 | 담당: 프론트엔드
   ├─ 체크박스 UI + 선택 상태 관리 (React Hook)
   ├─ 시각화: 파란색 테두리 + ✓ 체크마크
   ├─ 전체선택/해제 토글 버튼
   ├─ 일괄 삽입 로직 (커서 위치 유지)
   └─ 성공 토스트 + 선택 상태 자동 초기화

   ✅ 완료 기준: 3개 이미지 선택 → 삽입 → 본문 확인 ✅

5️⃣ SOP #5: 에러 처리 강화 (Error Handling Enhancement)
   기간: 3-4일 | 우선순위: P0 | 담당: 전체 팀
   ├─ 에러 코드 정의: 50+ 코드 (400/413/500 구분)
   ├─ 재시도 엔진: 지수 백오프 (500ms → 1s → 2s, 최대 3회)
   ├─ UI: 3가지 변형 (인라인/토스트/배너)
   ├─ API 수정: 기존 5-10개 API에 에러 처리 추가
   ├─ Webhook 재시도: DB 큐 기반 (5분 → 10분 → 20분)
   └─ 모니터링: 에러율 대시보드 + 자동 경고

   ✅ 완료 기준: 재시도 성공율 85%, 지원팀 요청 70% 감소

6️⃣ SOP #6: 사용자 메시지 및 진행률 (User Messaging & Progress)
   기간: 1-2일 | 우선순위: P1 | 담당: 프론트엔드
   ├─ 메시지 설계: 20+ 메시지 (따뜻한 톤, 행동 제안 포함)
   ├─ 진행률 컴포넌트: ProgressMessage (3가지 상태)
   ├─ 상태별 메시지 예시:
   │  ├─ 🔄 이미지 압축 중... (1.2MB → 450KB)
   │  ├─ 🔄 Google Drive에 저장 중... (1/1)
   │  ├─ ✅ 완료! 링크: [공유]
   │  ├─ 🔄 다시 시도 중... (2/3)
   │  └─ ❌ 오류 발생 (413: 파일 크기 초과). 파일을 분할하세요.
   ├─ 진행률 바 애니메이션
   └─ 다국어 지원 (한국어 + 영어)

   ✅ 완료 기준: 사용자 만족도 65% → 90%
