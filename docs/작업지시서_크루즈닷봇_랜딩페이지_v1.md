## ⚠️ 가장 먼저 읽을 것 — 거장단 만장일치 게이트

이 문서의 핵심 결론은 단 하나다. **"채팅하는 봇"이 아니라 "파는 봇"을 만들되, 그 전에 새는 양동이(클론·귀속 누수)를 먼저 막고, 봇은 미확인 사실을 단언하지 못하게 가둔다.**

거장 5명이 코드로 검증한 P0 합의 사항:
1. **클론은 1클릭이 아니다(거짓)** — `clone-shared`가 `groupId:null` + 13개 필드만 복사 → 결제·L6·퍼널·SMS가 끊긴 깨진 사본 생성. (코드 확인: `clone-shared/route.ts:74-103`, `groupId: null`은 84행)
2. **추적링크 targetUrl 경로 불일치** — `clone-shared`가 `targetUrl=/landing/{id}`로 생성하나 실제 공개 렌더러는 `/p/{slug|shortlink}`. 귀속(매출정산 SSoT)이 깨질 수 있음. (코드 확인: `clone-shared/route.ts:120` vs `p/[slug]/page.tsx:11-19`)
3. **공개 렌더러는 `htmlContent`만 렌더** — `blocksConfig`·인터랙티브 봇 UI는 `sanitizeHtml`에서 통째로 삭제됨(`p/[slug]/page.tsx:68`). 봇 UI를 htmlContent에 직렬화하면 50대 화면에 빈 페이지가 뜬다.
4. **귀속 위변조** — 익명 봇은 클라이언트가 보낸 `shortLinkCode`를 신뢰하면 셀프/탈취 어트리뷰션 + first-write-wins 선점 가능.
5. **프롬프트 인젝션** — 기존 sanitizer(`generate-copy/route.ts:15`, `\r\n` strip 한 줄)는 자유발화 봇에 무력. 시스템프롬프트·RAG·타고객 PII 유출, 환각 약속 생성 위험.
6. **환각/과장 = 광고법 위반** — 봇이 미확인 가격/할인/환불보장/의료를 단언하면 즉시 분쟁·과징금. 검수자AI 루브릭·Grounding 강제 없이는 실고객 연결 불가.

→ **이 P0들을 잠그기 전에는 봇을 실고객 트래픽에 연결하지 않는다.** Phase 분해에서 Phase 0(누수 선결)·Phase 5(검수 폐루프)·보안 게이트로 강제한다.

## 1. 한 줄 정의 + 비전

### 한 줄 정의
> **크루즈닷봇 랜딩페이지** = 관리자가 1회 제작·전체공유한 "AI 상담봇이 탑재된 랜딩페이지"를, 대리점장·판매원이 **딸깍 한 번**으로 본인 어필리에이트 링크가 박힌 사본으로 복제하면, 그 봇이 50대 고객을 스스로 상담·자격검증·1차 이의대응하고 **뜨거운 리드를 골든타임에 판매원에게 핸드오프**하여 크루즈닷몰 구매를 본인 실적으로 귀속시키는 시스템.

### 비전 흐름 (한 문장으로)
```
[관리자 제작]  봇 랜딩 1개 제작 → isGlobal 전체공유
   ↓ 딸깍
[대리점장/판매원]  "복사해서 쓰기" 1클릭 → 본인 소유·본인 조직·본인 추적링크·본인 그룹/퍼널 자동연결된 봇 랜딩 완성
   ↓ 본인 추적링크 공유 (카톡/문자)
[최종고객 50대]  링크 클릭 → 봇이 먼저 인사 + 선택칩 3개 → 자연스러운 상담 → 자격검증·이의 1차대응
   ↓ 핫리드 감지(예약금·날짜확정·특실)
[판매원]  즉시 푸시/SMS "지금 OOO 고객이 예약 직전, 전화하세요" → 사람이 클로징
   ↓ 구매
[크루즈닷몰]  어필리에이트 URL(+ref) 구매 → 웹훅 → BotConversation.attributedAgentId 경유 → 판매원 수당 귀속
```

### 비전이 성립하기 위한 전제 (거장단 합의)
- 봇의 역할은 **"클로징 단독 수행"이 아니라 "예열·자격검증·1차 이의대응 + 핫리드 핸드오프"** (고가 크루즈는 사람이 닫는 하이브리드가 정석 — 세일즈코치 P0)
- 봇은 **확정 상품 DB 값만 인용**, 가격·환불·일정·의료는 단정 금지하고 담당자 라우팅 (AI엔지니어·보안리드 P0)
- "딸깍"이 진짜 딸깍이려면 클론이 **원자적으로 발행 가능한 완성본**을 만들어야 함 (UX P0)

## 2. 사용자 시나리오 3종 (화면 단위 "딸깍" 흐름)

### 시나리오 A — 관리자(GLOBAL_ADMIN, 모니카/저스틴): 봇 베이스 제작·전체공유
| 단계 | 화면 | 동작 | 50대 직관 포인트 |
|---|---|---|---|
| A1 | `/landing-pages` 목록 → "새 봇 랜딩 만들기"(48px) | 봇 베이스 생성 시작 | 버튼 1개, 한글 라벨 |
| A2 | 봇 설정 화면(`pageType='bot'`) | 봇 대화 시나리오 선택(기본=크루즈콜모음 페르소나), 금칙 가드레일은 read-only 표시, 상품 카탈로그 연결 | "이대로 쓰기" 디폴트 선택 |
| A3 | 미리보기(48px) | 우측에 폰 모양 봇 챗 즉시 렌더(0.2초) | 폰 모양으로 실제처럼 |
| A4 | "전체 대리점에게 공유"(48px) → 확인 모달 | `isGlobal=true` 공유(`sharedToOrgId='__ALL__'`) | "모든 대리점장이 복사할 수 있어요" 1줄 안내 |

### 시나리오 B — 대리점장(OWNER): 딸깍 클론 → 본인 링크 발행
| 단계 | 화면 | 동작 | 50대 직관 포인트 |
|---|---|---|---|
| B1 | `/landing-pages` → "받은 페이지" 섹션 | 관리자가 공유한 봇 베이스 카드 표시(제목 16px, 출처 16px) | 큰 카드, 누가 공유했는지 한글 |
| B2 | "복사해서 쓰기"(min-h-48px, 16px) 1클릭 | `clone-shared` 호출 → **원자적으로**: 누락필드 전체복사 + 본인 기본그룹/퍼널 자동연결 + 고유 추적링크 발급 + 봇설정 read-only 상속 | **이것이 진짜 "딸깍"** |
| B3 | **결과 카드**(편집기 아님) | "✅ 내 봇 페이지가 준비됐어요" + [내 링크 복사하기(48px)] + [미리보기(48px)] 2버튼만 | 한 화면 2버튼, 설정 절벽 0 |
| B4 | 내 링크 복사 → 카톡/문자로 고객에게 공유 | `/p/{내shortlink}` 또는 `/l/{code}` | "이 링크로만 공유하세요" 명시 |

### 시나리오 C — 판매원(AGENT/FREE_SALES): 핫리드 받고 클로징
| 단계 | 화면 | 동작 | 50대 직관 포인트 |
|---|---|---|---|
| C1 | (B와 동일하게 클론) 또는 대리점장이 발급한 본인 링크 공유 | 본인 추적링크로 고객 유입 | — |
| C2 | `/partner/bot-dashboard`(신규) | 내 봇이 잡은 리드 목록(이름/마지막발화/구매의사점수/클로징 시도수/귀속근거) | 큰 글씨 목록, 점수 색상 구분 |
| C3 | 핫리드 푸시/SMS 수신 | "지금 OOO 고객이 OO 크루즈 예약 직전입니다. 전화하세요" | 즉각 알림 |
| C4 | "한 번 더 클로징 SMS" 버튼(48px) | 기존 funnel-sms 트리거로 후속 발송 | 버튼 1개 |

### 시나리오 D — 최종고객(50대, 비로그인): 봇과 대화 → 구매
| 단계 | 화면 | 동작 | 50대 직관 포인트 |
|---|---|---|---|
| D1 | `/p/{shortlink}` 진입 | 봇이 **먼저 인사** + 선택칩 3개("가격이 궁금해요"/"어디로 가나요"/"상담받고 싶어요", 각 48px) | 빈 입력창 금지, 손가락 한 번 |
| D2 | 칩 클릭 또는 입력(입력창 48px, 16px) | 봇 스트리밍 응답 + 타이핑 인디케이터(0.2초) | "멈췄나?" 불안 차단 |
| D3 | 봇이 SPIN 질문 → 가치제시 → trial close | 봇이 클로징을 향해 목적지향 대화 | 천천히·존댓말·공감 먼저 |
| D4 | 핫리드 신호 시 | "담당 전문가가 1분 내 연락드려요" + (영업시간 외 콜백예약 캡처) | 신뢰 핸드오프 |
| D5 | 구매 CTA | 단일 초록 버튼(48px) "예약 확정하기" → "크루즈닷 공식 안전결제로 이동합니다" 안심문구 + 사업자정보(13px+) → 어필리에이트 URL | 사기 의심 차단 |

## 6. 봇 런타임 설계

### 6-1. 모델 선택 (claude tier)
| 용도 | 모델(env) | 근거 |
|---|---|---|
| 기본 대화(인사·FAQ·라포·SPIN 질문) | `BOT_CHAT_MODEL` = `claude-haiku-4-5` | 응답속도=전환율(50대 0.2초), 대량 트래픽 비용 최적 |
| 클로징·이의대응·심리렌즈 멘트 | `BOT_CLOSING_MODEL` = `claude-sonnet-4-6` | 설득 품질=매출 직결 |
| 야간 배치 분석(대화품질·전환분석) | 기존 `analyze-calls` 패턴 haiku | 실시간 봇엔 opus 금지(레이턴시·비용) |

- **에스컬레이션 트리거(haiku→sonnet)** 명문화: ①이의유형 감지(가격/의료/가족반대) OR ②N턴 이상 미전환 OR ③사용자가 '구매/계약/예약' 의사 표현 시에만 sonnet **1회** 호출. 기본은 haiku 유지.
- `modelUsed`를 `BotMessage`에 로깅 → 비용/효과 사후검증.
- 🔴 **모델지정 SSoT 수렴**(AI엔지니어 P1): 현재 기존 5개 라우트가 env(1)·하드코딩최신(3)·날짜스냅샷(1) 혼재(`ai-draft`=`claude-sonnet-4-6` 하드코딩, `analyze-calls`=`claude-haiku-4-5-20251001` 날짜고정). 봇 도입과 함께 `src/lib/bot-model.ts`의 `getBotModel(role)` 단일 헬퍼로 통일하고, **날짜스냅샷 suffix를 답습하지 말 것**(별칭/env).

### 6-2. 스트리밍 방식
- **직접 `@anthropic-ai/sdk` 스트리밍** — `client.messages.stream({...})` + `stream.toReadableStream()` 또는 `new ReadableStream`으로 SSE 반환(`Content-Type: text/event-stream`). AI SDK·AI Gateway 도입 없이 의존성 0(기존 5라우트 일관).
- 🔴 **`export const runtime = 'nodejs'` 고정** — Prisma `@prisma/adapter-pg`(pg 드라이버)는 Node 전용. Edge 사용 시 DB 접근 전면 실패.
- `export const maxDuration = 90` (실측 p95 기반 60~120초, 무한 상향 금지). Vercel Fluid Compute로 장시간 스트리밍 지원.
- **스트리밍 견고성**(AI엔지니어 P1):
  - `req.signal`을 stream에 연결, 연결 끊김 시 `stream.abort()` + 누적 텍스트를 `BotMessage.status='partial'`로 저장(과금 토큰 추적).
  - `finally` 블록에서 **항상 영속화**(complete/partial/failed 명시), 다음 턴 히스토리 조립 시 partial/failed 제외.
  - maxDuration 임박 시 graceful 마감 + "잠시 후 다시" 메시지.
  - 최소 heartbeat 처리(긴 무응답 구간 연결 유지).

### 6-3. RAG 색인전략 (2단계)
**1단계(즉시 출시) — 사실/설득 소스 분리 + Grounding 강제**
- 🔴 **사실원천 = 상품 DB**(가격/일정/객실/환불정책): 키워드가 아니라 **구조화 조회**(`$queryRaw` 정확매칭/프리픽스)로 결정적 인출. hit 없으면 봇은 사실 단언 금지 → "담당자 확인/등록폼"으로 전환.
- **설득원천 = ScriptPattern**(톤·이의대응 멘트): `organizationId + personaType + objectionType` WHERE 조회 → 상위 N개 `patternText` 시스템프롬프트 주입(키워드/유사도).
- 모든 봇 응답에 `ragSourceIds` 기록(사후 환각 감사).
- (선택) QA 420개·상품설명은 기존 `@google/generative-ai`(^0.24.1) text-embedding으로 사전 임베딩 → 소량 코퍼스 인메모리/DB 코사인 근사검색(pgvector 없이도 가능).

**2단계(트래픽·콘텐츠 증가 후)**
- Neon `CREATE EXTENSION vector`(콘솔/Management API) → `BotKnowledgeChunk.embedding vector` 추가 → `$queryRaw` 코사인유사도 검색. 임베딩은 google/OpenAI.

### 6-5. 구매유도 클로징 로직 (대화 FSM)

## 7. 어필리에이트 개인링크 자동치환 메커니즘

### 7-1. 클론 시 판매원 코드 주입 (Phase 0)
`clone-shared/route.ts` 트랜잭션에서 이미 `ShortLink(code=고유, createdBy=ctx.userId)` 생성 → **이것이 판매원 개인 추적코드**. 추가 구축 불필요하나 아래 2가지 수정 필수:
- 🔴 **`targetUrl` 경로 수정**: `clone-shared/route.ts:120` `${appUrl}/landing/${newPage.id}` → `${appUrl}/p/${shortlink}` (실제 공개 렌더러 경로와 일치). `createShortLinkForPage`(`landing-page-utils.ts:59`)도 동일 수정.
- 🔴 **end-to-end 통합테스트**: `/l/{code}` 클릭추적 → `/p/{shortlink}` 리다이렉트 → 봇 진입 → 귀속 스냅샷이 클론 페이지에서 동작하는지 검증.

### 7-2. 봇 진입 → 귀속 스냅샷 (`src/lib/bot-attribution.ts`)
```
봇 진입(ref=shortLinkCode 쿼리 또는 /p/{shortlink})
 → 🔴 클라이언트 입력 신뢰 금지: 서버가 ShortLink 조회
   → isActive·organizationId·createdBy 재확인
   → createdBy가 실제 활성 판매원(role∈{AGENT,FREE_SALES,OWNER}, 동일 org)인지 검증
 → HttpOnly·HMAC 서명 쿠키로 attribution 토큰 1회 발급
   → 이후 메시지는 쿠키 기준으로만 귀속(body의 code는 무시)
 → BotConversation.attributedAgentId 스냅샷(고정, 이후 변동 금지)
 → attributionSource 기록(shortlink|last_touch|round_robin|none)
```
- **ref 누락 폴백 체인**(명문화): `본인 숏링크 > 최근 SMS/통화 접촉 판매원(쿠키 last-touch) > 조직 라운드로빈 > 무귀속 풀(pool)`. null은 조용히 삼키지 말고 **"무귀속 전환" 리포트로 가시화**.
- **셀프 어트리뷰션 차단**: visitor IP/디바이스가 판매원 본인 세션과 동일하면 귀속 제외 + 이상치 로그.

### 7-3. 구매 CTA URL → 웹훅 귀속
```
봇 구매 CTA = https://cruisedot.co.kr/...?ref={shortLinkCode}&inquiry={id}
 → 고객 구매 → 어필리에이트 구매 웹훅(payapp/cruisedot, HMAC 검증)
 → CruiseProductInquiry.agentId = BotConversation.attributedAgentId 매핑
 → 🔴 신뢰등급: 검증 웹훅(고신뢰)이 봇 추정 귀속(저신뢰)을 덮어쓸 수 있음
   (first-write-wins에 출처 신뢰등급 적용 — 봇=추정, 웹훅=확정)
 → settlement-saga / affiliate 수당 파이프라인(기존)
```

- **(선결) Phase 0** 클론·귀속 누수 수정 — 봇과 무관해도 최우선.
- **(권장) 1주 컨시어지 검증**: 기존 `/p/[slug]` 정적 랜딩 + "실시간 상담" 버튼 뒤에 사람(또는 사람이 Claude 콘솔로 답변)을 붙여 "챗 상담→문의 전환율" 먼저 측정. 대리점장 3~5명에게 봇 목업 인터뷰("클론해 쓰겠는가").
- **최소 봇**(가설 확인 후): 비스트리밍 `messages.create`(기존 5라우트와 동일) + 단일 haiku + **큐레이션 시스템프롬프트(RAG는 상품DB 결정조회만, ScriptPattern 주입은 v0.1)** + BotConversation/BotMessage 영속화 + ShortLink 귀속 스냅샷.
- **봇 역할 = "문의수집 어시스턴트 + 사람 핸드오프"**(클로징/구매유도 단정 아님) + 가격·환불·의료·일정 단정 금지 하드가드 + 면책고지.
- **per-user 격리 + PII 마스킹**(타협 불가).
- **북극성 지표 3개 대시보드**: ①대리점장 클론·활성화율 ②봇 세션→문의 전환율 vs 폼 단독 ③귀속 구매확정률 + 무귀속 비율.

### 🔵 MVP 이후로 연기
- 스트리밍 SSE(첫 출시는 응답완료 후 표시로 충분) → 가설 확인 + 실트래픽 비용 측정 후.
- 모델 tier 분리(haiku/sonnet) → 트래픽·비용 데이터 후.
- 프롬프트 캐싱·pgvector·익명 서명토큰·온라인 검수 → 후속.
- 봇 단독 클로징(trial close N회) → 사람 핸드오프 하이브리드 검증 후.

## 13. 미해결 결정사항 (사용자에게 물어야 할 것)

1. **봇 역할 범위**: 거장단(베조스·보안·세일즈) 모두 "클로징 단독 ❌ → 문의수집+사람 핸드오프 ✅"를 권고. 동의하는가? 아니면 봇이 직접 결제까지 끌고 가길 원하는가? (법적/브랜드 리스크 차이 큼)
2. **컨시어지 선검증 수용 여부**: 코드 착수 전 1주 사람-백업 챗으로 "봇 상담이 전환을 올리는가"를 먼저 측정할 것인가, 아니면 바로 최소 봇 구축으로 갈 것인가?
3. **MVP 스트리밍 여부**: 첫 출시를 비스트리밍(응답완료 후 표시)으로 단순화해도 되는가? (50대 체감은 타이핑 인디케이터로 보완 가능하나 스트리밍이 더 자연스러움)
4. **귀속 폴백 정책**: ref 누락 직접진입 시 ①조직 라운드로빈 ②최근접촉 판매원 ③무귀속 풀(선착 클레임) 중 어느 정책인가? (판매원 간 분쟁 핵심)
5. **상품 사실 데이터 원천**: 봇이 인용할 "확정 가격/일정/객실/환불정책"의 SSoT는 어느 테이블/연동인가? (CruiseProductInquiry는 문의용 — 실제 상품 카탈로그 위치 확인 필요. 크루즈닷 API 연동 여부)
6. **검수 게이트 임계값**: Grounding≥90 / Overclaim=0 / Compliance=0이 적정한가? 더 엄격/완화?
7. **핫리드 알림 채널**: 판매원 핸드오프 알림을 SMS(알리고)·카카오·앱푸시 중 무엇으로? 영업시간 외 콜백예약 정책은?
8. **봇 운영 비용 상한**: 조직별 일일 토큰/금액 하드캡 기준값은? 초과 시 자동 비활성 vs 보수모드(haiku 강등)?
9. **새 도메인 코드 `Agent-BOT` 신설 여부**: CLAUDE.md 도메인 테이블에 봇 전용 도메인(`api/bot/`, `p/[slug]/BotLandingClient`)을 추가할 것인가, 아니면 Agent-MKT 산하로 둘 것인가?
