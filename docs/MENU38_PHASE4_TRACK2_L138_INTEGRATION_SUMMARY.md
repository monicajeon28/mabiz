## 📋 의사결정 사항

### Q1: 3개 렌즈를 순차 vs 병렬로 개발할까?

**옵션 A: 순차 (L1 → L3 → L8)**
- 기간: 3-4주
- 리소스: 1-2명
- 안정성: 높음
- 버그 위험: 낮음

**옵션 B: 병렬 (L1, L3, L8 동시)**
- 기간: 1주일
- 리소스: 3명
- 안정성: 중간
- 버그 위험: 중간

**→ 추천**: **B (병렬)** — "속도·성능·효율·자동화 원칙"

---

### Q2: L1 배포 후 즉시 모니터링, 아니면 L3/L8 같이?

**옵션 A: L1만 먼저 (1주일 모니터링)**
- 위험도: 낮음
- 버그 발견: 용이
- 다음 렌즈: 2주 늦음

**옵션 B: L1 배포 + L3/L8 동시 개발**
- 위험도: 중간
- 버그 발견: 복잡
- 다음 렌즈: 정시 완료

**→ 추천**: **B (동시 진행)** — 병렬 에이전트 3명 필요

---

### Q3: L1 전환율이 목표(42%)에 못 미치면?

**시나리오: 실제 전환율 35% 나옴 (목표 42% vs 실제 35%)**

**원인 분석**:
1. Phase 0 오프닝이 약함 (→ A/B 테스트)
2. SMS Day 1 클릭율 낮음 (→ 메시지 수정)
3. 신규 선박 우선권이 충분하지 않음 (→ 추가 긴급성 강조)

**대응**:
- Week 4에 5명 고객 대상 심화 콜 인터뷰
- Phase 0 오프닝 3가지 변형 A/B 테스트
- SMS 메시지 톤/내용 변경 후 재발송
