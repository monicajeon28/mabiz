| 버튼 | 관리자 | 대리점장 | 판매원 | 일반사용자 |
|------|--------|----------|--------|----------|
| 💰 월말정산 | ✅ 사용 가능 | 🔒 사용 불가 | ❌ 안 보임 | ❌ 안 보임 |

| ✅ 확인 | ✅ 사용 가능 | ✅ 사용 가능 | ✅ 사용 가능 | ❌ 안 보임 |
| 📥 엑셀다운 | ✅ 전체 다운 | ✅ 자기팀만 | ✅ 본인만 | ❌ 안 보임 |
| 🔄 재계산 | ✅ 사용 가능 | 🔒 사용 불가 | ❌ 안 보임 | ❌ 안 보임 |

## 🎯 5개 버튼 상세 설명

### 1️⃣ 💰 **월말정산** 버튼

**"모든 판매원의 수당을 계산해서 돈을 주는 거예요"**

#### 누가 사용할 수 있나?
- ✅ **관리자만** 사용 가능
- 🔒 **대리점장**: 사용 불가 (회색으로 보임)
- ❌ **판매원**: 버튼이 안 보임
- ❌ **일반사용자**: 버튼이 안 보임

#### 왜 이렇게 설정했나?
- 월말정산은 **모든 팀의 돈을 처리하는 일**이에요
- 이건 본사 관리자만 책임질 수 있어요
- 대리점장이 임의로 자기 팀 정산을 하면 회계 문제가 생길 수 있으니까요

#### 대리점장이 사용하고 싶다면?
- 회색 버튼에 **마우스를 올리면** 설명이 나타나요:
  ```
  "정산은 본사 관리자만 처리할 수 있어요.
   본사에 정산 요청을 해주세요."
  ```

---

### 2️⃣ 🚨 **이의제기** 버튼

**"이 수당이 잘못된 것 같아요' 라고 말하는 거예요"**

#### 누가 사용할 수 있나?
- ✅ **관리자** 사용 가능 (모든 팀의 수당에 이의 가능)
- ✅ **대리점장** 사용 가능 (자기 팀 수당에만 이의 가능)
- ❌ **판매원**: 버튼이 안 보임
- ❌ **일반사용자**: 버튼이 안 보임

#### 왜 이렇게 설정했나?
- 수당 계산 오류가 있으면 **누구든 지적할 수 있어야 해요**
- 관리자는 모든 팀을 책임지므로 전체 이의 가능
- 대리점장은 자기 팀만 책임지므로 자기 팀만 이의 가능
- 판매원은 이의제기보다는 **"확인" 버튼으로 물어보는 방식**이 더 적합해요

#### 예시
```
"어, 나 이번 달 수당이 작은 것 같은데?"
→ 이의제기 버튼 클릭
→ "이번 달 수당을 다시 확인해달라"는 요청 제출
→ 관리자가 확인 후 수정
```

---

### 3️⃣ ✅ **확인** 버튼

**"이 수당이 뭐에서 나온 거야?' 라고 자세히 보는 거예요"**

#### 누가 사용할 수 있나?
- ✅ **관리자** 사용 가능
- ✅ **대리점장** 사용 가능
- ✅ **판매원** 사용 가능
- ❌ **일반사용자**: 버튼이 안 보임

#### 왜 이렇게 설정했나?
- 누구든 자신의 수당이 **어디에서 나왔는지 알아야 해요**
- 이건 **중요한 교육용 기능**이에요
- 투명성을 위해 누구든 확인할 수 있게 했어요

#### 예시
```
관리자: "A팀의 7월 수당 내역을 확인하고 싶어"
        → 확인 버튼 클릭 → 판매기록/여행출발/환불 등 상세 보기

판매원: "내 7월 수당이 얼마인지 알고 싶어"
        → 확인 버튼 클릭 → "음.. 이 판매건이 있네?" 확인
```

---

### 4️⃣ 📥 **엑셀다운** 버튼

**"수당 기록을 컴퓨터에 저장하는 거예요"**

#### 누가 사용할 수 있나?
- ✅ **관리자**: 모든 팀의 모든 수당을 다운로드
- ✅ **대리점장**: 자기 팀 판매원의 수당만 다운로드
- ✅ **판매원**: 자신의 수당만 다운로드
- ❌ **일반사용자**: 버튼이 안 보임

#### 왜 이렇게 설정했나?
- 누구든 **자신의 기록은 가져야 해요**
- 하지만 **다른 팀의 기밀 정보는 볼 수 없어야 해요**
- 회계 감사 때 필요하니까요

#### 예시

**관리자가 엑셀 다운로드**
```
버튼 위에 마우스를 올리면:
"전체 팀 원의 수당 기록을 다운로드합니다"

다운로드되는 파일:
[마비즈] 전체 수당 기록.xlsx

포함 내용: 
- A팀 김철수: 1,000,000원
- A팀 이영희: 800,000원
- B팀 박민준: 1,200,000원
... (모든 팀, 모든 사람)
```

**대리점장(A팀)이 엑셀 다운로드**
```
버튼 위에 마우스를 올리면:
"당신 팀 판매원의 수당만 다운로드합니다"

다운로드되는 파일:
[마비즈] 우리 팀 수당 기록.xlsx

포함 내용:
- 김철수: 1,000,000원
- 이영희: 800,000원
... (A팀만)
```

**판매원(김철수)이 엑셀 다운로드**
```
버튼 위에 마우스를 올리면:
"당신의 수당 기록을 다운로드합니다"

다운로드되는 파일:
[마비즈] 내 수당 기록.xlsx

포함 내용:
- 6월: 1,000,000원
- 7월: 950,000원
- 8월: 1,100,000원
... (김철수만)
```

### 5️⃣ 🔄 **재계산** 버튼

**"수당을 처음부터 다시 계산하는 거예요"**

#### 누가 사용할 수 있나?
- ✅ **관리자만** 사용 가능
- 🔒 **대리점장**: 사용 불가 (회색으로 보임)
- ❌ **판매원**: 버튼이 안 보임
- ❌ **일반사용자**: 버튼이 안 보임

#### 왜 이렇게 설정했나?
- 재계산은 **시스템의 데이터를 변경하는 일**이에요
- 이걸 잘못하면 **모든 팀의 수당이 엉망**이 될 수 있어요
- 그래서 관리자만 이 권한을 가져요

#### 대리점장이 사용하고 싶다면?
- 회색 버튼에 **마우스를 올리면** 설명이 나타나요:
  ```
  "재계산은 본사 관리자만 처리할 수 있어요.
   본사에 재계산을 요청해주세요."
  ```

## 👥 역할별 권한 정리

### 1️⃣ 관리자 (GLOBAL_ADMIN)
**"마비즈 전체를 관리하는 사람"**

- 모든 버튼을 사용할 수 있어요
- 모든 팀의 데이터를 볼 수 있어요
- 엑셀 다운로드: 전체 팀 데이터

**권한**:
```
💰 월말정산  → ✅ 활성
🚨 이의제기  → ✅ 활성
✅ 확인      → ✅ 활성
📥 엑셀다운  → ✅ 활성 (전체 다운)
🔄 재계산    → ✅ 활성
```

---

### 2️⃣ 대리점장 (OWNER)
**"자기 팀 판매원들을 관리하는 사람"**

- 자기 팀의 데이터만 볼 수 있어요
- 월말정산과 재계산은 관리자가 해야 해요
- 엑셀 다운로드: 자기 팀 데이터만

**권한**:
```
💰 월말정산  → 🔒 비활성 (관리자만)
🚨 이의제기  → ✅ 활성 (자기 팀만)
✅ 확인      → ✅ 활성 (자기 팀만)
📥 엑셀다운  → ✅ 활성 (자기 팀만 다운)
🔄 재계산    → 🔒 비활성 (관리자만)
```

**호버했을 때 메시지**:
```
💰 월말정산 호버:
"정산은 본사 관리자만 처리할 수 있어요.
 본사에 정산 요청을 해주세요."

🔄 재계산 호버:
"재계산은 본사 관리자만 처리할 수 있어요.
 본사에 재계산을 요청해주세요."
```

---

### 3️⃣ 판매원 (AGENT)
**"여행을 파는 판매원"**

- 자신의 데이터만 볼 수 있어요
- 다른 판매원의 데이터는 볼 수 없어요
- 엑셀 다운로드: 본인 데이터만

**권한**:
```
💰 월말정산  → ❌ 버튼이 안 보임
🚨 이의제기  → ❌ 버튼이 안 보임 (확인으로 물어보세요)
✅ 확인      → ✅ 활성 (내 수당만)
📥 엑셀다운  → ✅ 활성 (나만 다운)
🔄 재계산    → ❌ 버튼이 안 보임
```

---

**"수당 시스템을 사용하지 않는 사용자"**

- 수당 메뉴를 볼 수 없어요
- 모든 버튼이 안 보여요

**권한**:

## 💡 자주 묻는 질문

### Q1: 대리점장이 "월말정산" 버튼을 정말 사용할 수 없나요?
**A**: 네, 사용할 수 없어요.
- 왜? 월말정산은 **모든 팀의 돈을 처리하는 일**이기 때문이에요
- 대리점장은 자기 팀만 책임지니까, 본사가 정산을 하도록 설계했어요
- 대리점장이 정산을 하고 싶으면 → 본사 관리자에게 요청하세요

### Q2: 판매원은 왜 "이의제기" 버튼을 사용할 수 없나요?
**A**: 판매원은 **"확인" 버튼으로 물어보는 방식**을 사용해요.
- 판매원: "이게 뭐에서 나온 거야?" → 확인 버튼 클릭
- 대리점장/관리자가 보고 설명해줘요
- 만약 계산 오류가 있다면, 관리자/대리점장이 "이의제기" 버튼을 사용해요

### Q3: 엑셀 다운로드할 때 데이터가 다르게 나와요?
**A**: 네, 역할에 따라 다르게 나와요. 맞아요!
- 관리자: 모든 팀 데이터 포함
- 대리점장: 자기 팀만 포함
- 판매원: 본인 데이터만 포함

이건 **개인정보 보호와 보안**을 위해서예요.

### Q4: 호버(마우스 올리기) 메시지가 안 보여요.
**A**: 모바일에서는 호버가 없어요.
- PC/태블릿: 마우스를 버튼 위에 올리면 메시지 나타남
- 모바일: 버튼을 누르면 메시지 나타남 (또는 비활성 버튼은 안 보임)

## 🔐 보안 설계

**UI 권한** + **API 권한** = 완벽한 보안

### 1️⃣ UI 권한 (Phase 3) ← 지금 여기
- 버튼을 보거나 안 보이게 함
- 클릭 가능하게 하거나 못하게 함
- 사용자의 실수를 방지해요

### 2️⃣ API 권한 (Phase 2)
- 서버에서 **역할을 다시 확인**해요
- "이 사람이 정말 관리자가 맞나?"
- "이 사람이 정말 자기 팀 데이터만 다운로드하는 건가?"
- 다운로드 데이터를 자동으로 필터링해요

**예시**:
```
대리점장이 엑셀 다운로드 버튼을 누르면:
1️⃣ UI: "당신 팀 판매원의 수당만 다운로드합니다" 메시지 표시 (안내)
2️⃣ API: 서버가 역할 확인 → 자동으로 자기 팀만 필터링 (보안)
3️⃣ 파일: [마비즈] 우리 팀 수당 기록.xlsx (자기 팀만)
```
