기술 요약
================================================================================

API: GET /api/contacts/:id/360
응답: Contact360Response (1 JSON, ~50KB)

구조:
  Contact (중앙)
    ├─ GoldMember (1:1)
    ├─ Partner (N:1)
    ├─ Groups (N:N)
    ├─ Orders (N:N, cruisedot)
    └─ Communications (SMS/Email/Call)
    └─ Psychology + Risk + Affiliate

성능:
  응답시간: 3-5s (현재) → 1s (캐시) / 2s (신규) = 50% ↓
  N+1 제거: 5+ 쿼리 → 1 쿼리
  DB 부하: 100% → 30%

기술 스택:
  • Prisma DataLoader (배치 쿼리)
  • Redis (30분 캐시, 80% 히트율 목표)
  • PostgreSQL Materialized View
  • PII 마스킹 (권한 기반, GDPR/PIPA)

================================================================================
주요 기능 3가지
================================================================================

1. 통합 360도 뷰 (/api/contacts/:id/360)
   - Contact + GoldMember + Partner + Groups + Orders
   - Psychology Lens + Risk Flags + Affiliate Tracking
   - 모든 커뮤니케이션 로그 (SMS, Email, Call)
   응답 예시: 18개 필드 그룹, 총 ~50KB JSON

2. PII 마스킹 (권한 기반)
   Admin: 마스킹 안함 (원본)
   Manager: 부분 마스킹 (전화번호 뒤 4자리)
   Agent: 전체 마스킹 (010XXXX5678)
   준수: GDPR, CCPA, 한국 PIPA

3. 위험도 + 권장액션
   Risk Score: 0-100 (낮을수록 좋음)
   Flags: 7가지 (준비불안, 경쟁사언급, 배우자동의 등)
   Recommended Actions: 자동생성 (3-5개)

================================================================================
성능 최적화 4단계
================================================================================

단계 1: DataLoader (N+1 제거)
  기술: Batch Query (Contact, GoldMember, Partner 등)
  효과: DB 쿼리 80% ↓

단계 2: Redis 캐싱 (30분 TTL)
  기술: Contact360Response 전체 캐시
  효과: 응답시간 50% ↓, 캐시 히트율 80% 목표

단계 3: Materialized View
  기술: 읽기 최적화 View, 매시간 갱신
  효과: 조회 성능 30% ↑

단계 4: 추가 인덱스
  기술: (orgId, userId, status) 등
  효과: 조회 속도 2배 ↑

최종: 응답시간 3-5s → 1s (캐시) / 2s (신규)
