# Tools 페이지 UX 개선 로드맵 (2026-06-02)

## 현황 분석 (As-Is)

### 1. Tools 페이지 구조
- **메인 탭**: sms | playbook | call-feedback | call-playbook | qa-library
- **SMS 탭**: CARE_VIP, SEQUENCE, LIVE_BROADCAST (3개)
- **Playbook 탭**: REJECTION, RECONTACT, CLOSING, PERSONA, SUCCESS_CASE, FORBIDDEN, OPENING, NEEDS (8개)
- **구성**: 템플릿 라이브러리 + 콜 피드백 분석 + 플레이북 뷰어 + Q&A 라이브러리

### 2. PlaybookViewer 기능 (현재)
- 세그먼트 필터 (A-E, ALL)
- 제품코드 필터 (GOLD, GENERAL)
- 심리학 렌즈 필터 (L6=손실회피, L10=즉시구매)
- PASONA 스테이지 배지
- 신민형 5단계 배지
- 모니카 욕망증폭 레벨
- Day 0-3 스케줄 미리보기

### 3. Call Feedback 분석
- 텍스트/파일 입력 가능
- 제품 유형 선택 (GOLD/GENERAL)
- 성약 여부 선택 (True/False)
- 분석 결과: 점수(100점)/등급/요약/강점/개선점
- 페르소나 자동 감지 (5가지)
- 추천 후속 문자 생성

### 4. 현재 UX 문제점

❌ **플레이북 데이터 없을 때**
- 로딩 중: Spinner만 표시 (메시지 없음)
- 데이터 없음: 빈 목록 (가이드 없음)
- 필터 혼란: 필터는 많은데 결과 0건

❌ **콜 피드백 에러 처리**
- 분석 중 오류 시: 에러 텍스트만 표시
- 사용자 원인 불명확 (입력 오류? API 오류? 토큰 초과?)
- 재시도 버튼 없음

❌ **PlaybookViewer 필터 불편**
- 필터 10개인데 UI에 산재
- 한 번에 볼 수 있는 필터 3-4개만 가능
- 선택 조건을 한눈에 못 봄
- 빠른 리셋 기능 없음

❌ **SMS 자동화 빌더 부재**
- Day 0-3 타임라인 시각화 없음
- 각 메시지 미리보기 구조화 안 됨
- 세그먼트별 변형 비교 불가능
- 발송 전 최종 확인 단계 없음

❌ **대시보드 시각화 부족**
- 자동화 워크플로우 노드 다이어그램 없음
- 실시간 발송 현황 미제공
- 성과 메트릭 추이 추적 없음

## To-Be: UX 개선 로드맵 (3개월)

### Phase 1: Tools 페이지 개선 (Week 1-2)

#### 1.1 용어 명확화 + 가이드
```
현재: "VIP 케어" → 개선: "우수고객 케어 (특별 관리 + 긴박감 연속발송)"
```
- **타이틀 명확화**: 각 탭 옆에 아이콘 + 설명 추가
  - SMS 탭: 🎯 "예약 확정용", 📅 "체계적 시퀀스", 📹 "라이브 중 실시간"
  - Playbook 탭: 🚫 "거절한 고객", 📞 "재접촉 스크립트" 등

#### 1.2 플레이북 데이터 부재 시 UX
```
상태 1: 로딩 중 → 스켈레톤 3개 + "데이터 로드 중... (2-3초)"
상태 2: 데이터 없음 → 빈 상태 아이콘 + 가이드
  "이 필터 조건으로는 플레이북이 없습니다."
  "→ 다른 세그먼트 선택" / "→ 필터 초기화"
상태 3: 에러 → Red Alert + 재시도 버튼 + 관리자 연락 안내
```

#### 1.3 콜 피드백 에러 메시지 개선
```
현재: "분석 실패" (모호)
개선:
- 입력 오류: "텍스트가 너무 짧습니다. 최소 100자 이상 입력하세요."
- API 오류: "일시적 오류가 발생했습니다. 30초 뒤 재시도하거나 관리자에 연락하세요."
- 토큰 초과: "분석 한도를 초과했습니다. 내일 다시 시도하세요. (기본 10건/일)"
- 타임아웃: "분석에 1분 이상 걸리고 있습니다. 다시 시도하세요."
```

#### 1.4 Q&A 라이브러리 → 고객 질문 섹션 분리
```
구조:
├─ 콜 스크립트 (현재 Playbook)
├─ 콜 피드백 분석 (현재)
├─ 콜 플레이북 (현재)
└─ 📋 자주 묻는 질문 (NEW)
   ├─ 가격 관련 (10개)
   ├─ 일정 관련 (8개)
   ├─ 객실 관련 (7개)
   └─ 기타 (5개)
```

### Phase 2: PlaybookViewer 필터 UI 개선 (Week 3-4)

#### 2.1 필터 패널 리디자인 (Fixed Sidebar)

**레이아웃**:
```
┌─────────────────────────────────────┐
│ 콜 플레이북 라이브러리                │
├──────────────┬──────────────────────┤
│  필터         │  결과 (12개)          │
│ (LEFT 250px) │  [플레이북 카드들]    │
│              │                      │
│ [초기화]      │  선택된 필터:        │
│ [저장]        │  Segment: A ✓        │
│ [공유]        │  Phase: 클로징 ✓     │
│              │  Psychology: L6 ✓   │
└──────────────┴──────────────────────┘
```

#### 2.2 필터 항목 10개

1. **고객 세그먼트** (라디오)
   - ○ 모든 고객
   - ○ A: 30대 커플
   - ○ B: 40대 가족
   - ○ C: 중년 부부
   - ○ D: 50-60대
   - ○ E: 60대+

2. **제품 유형** (라디오)
   - ○ 모든 상품
   - ○ GOLD (프리미엄)
   - ○ GENERAL (일반)

3. **심리학 렌즈** (다중선택)
   - ☐ L6: 손실회피
   - ☐ L10: 즉시구매

4. **PASONA 스테이지** (다중선택)
   - ☐ Problem (문제 인식)
   - ☐ Affinity (공감)
   - ☐ Solution (해결책)
   - ☐ Offer (제안)

5. **신민형 5단계** (다중선택)
   - ☐ Step 1: 라포 형성
   - ☐ Step 2: 니즈 SPIN
   - ☐ Step 3: 욕망 증폭
   - ☐ Step 4: 감정 피크
   - ☐ Step 5: 클로징

6. **ROI 기대 효과** (슬라이더)
   - [50%] ―― [100%] ―― [150%+]

7. **사례 유형** (다중선택)
   - ☑ ✅ 성공사례 (진짜 클로징)
   - ☐ 📊 개선사례 (교정)
   - ☐ 📚 교육사례 (기초)

8. **음성 톤** (라디오)
   - ○ 조용한 설득
   - ○ 활발한 유도
   - ○ 긴급한 클로징

9. **난이도** (라디오)
   - ○ 입문: 기초
   - ○ 중급: 이의대응
   - ○ 고급: 복합

10. **즐겨찾기** (체크)
    - ☑ 저장된 플레이북만

#### 2.3 필터 상호작용
- **"초기화"**: 모든 필터 → 기본값
- **"저장"**: 현재 필터 조합 → 즐겨찾기 (최대 5개)
- **"공유"**: 필터 URL 복사 (팀원과 공유)
- **선택된 필터**: 칩 형태로 결과 위에 표시
  - `[Segment: A ✕] [Phase: 클로징 ✕] [Psychology: L6 ✕]`
