## 🎯 핵심 원칙 (30초 요약)

판매관리 페이지는 "누가 뭘 봤는지"만 다릅니다.
- **관리자**: 전체 + 개인정보 완전 노출 → 감시 기록됨
- **대리점장**: 팀 + 개인정보 가림 → 팀 기록됨  
- **판매원**: 차단 → 접근 불가

## 🎨 마스킹 규칙 (시각화)

### 이름 마스킹
```
관리자가 봄:
"김철수"    → "김철수" (완전)

대리점장이 봄:
"김철수"    → "김*"    (마스킹)

예외:
"나"        → "나"     (1글자는 그대로)
""  또는 null → "-"    (없으면 대시)
```

### 전화번호 마스킹
```
관리자가 봄:
"010-1234-5678"  → "010-1234-5678"  (완전)

대리점장이 봄:
"010-1234-5678"  → "010-****-5678"  (중간 4자리만 가림)

예외:
""  또는 null    → ""                (없으면 빈칸)
```

## 🔐 쿼리 필터 규칙 (개발자)

### 관리자 (GLOBAL_ADMIN)
```sql
-- 모든 결제 조회 (필터 없음)
SELECT * FROM CrmPayAppPayment
WHERE createdAt >= 6개월전
ORDER BY createdAt DESC;
```

### 대리점장 (AGENT) — 현재 미활성화 ⚠️
```sql
-- 자신의 조직만 조회
SELECT * FROM CrmPayAppPayment pp
INNER JOIN CrmAffiliateSale af ON af.orderId = pp.orderId
WHERE af.organizationId = 자신의조직ID
ORDER BY createdAt DESC;
```

### 판매원 (FREE_SALES)
```
HTTP 403 Forbidden
(접근 차단)
```

## 💡 FAQ

### Q1: 왜 대리점장은 개인정보를 가려요?
A: 팀원 프라이버시 보호. 팀장이 다른 팀원의 고객 정보까지 완전히 알 필요는 없습니다.

### Q2: 판매원이 자기 판매건을 보고 싶으면?
A: 별도의 "나의 판매건" 페이지가 필요합니다. (현재 미구현)

### Q3: 환불 금액이 잘못되면?
A: CRM에서 수당 계산이 잘못된 것. 판매관리는 조회만 하므로 CRM을 확인하세요.

### Q4: 감시 로그는 어디서 보나?
A: 데이터베이스의 `AuditLog` 테이블. (사용자 UI에서는 미제공)

### Q5: AGENT가 현재 왜 차단되어 있나?
A: Phase 1 구현에서 관리자 기능만 먼저 완료했습니다. 향후 AGENT 기능을 활성화할 예정입니다.

## ✍️ 용어 정리 (초등학생 버전)

| 용어 | 의미 |
|------|------|
| **RBAC** | Role-Based Access Control = 역할에 따라 권한이 다름 |
| **마스킹** | 개인정보 가리기 = "김철수" → "김*" |
| **403** | "들어갈 수 없습니다" = 문은 있는데 열쇠가 없음 |
| **감사 로그** | "누가 뭘 봤는지" 기록 = 관리자가 어제 뭘 봤는지 나중에 확인 가능 |
| **organizationId** | 팀 ID = "서울지점", "부산지점" 같은 팀 번호 |
| **GLOBAL_ADMIN** | 전체 관리자 = 회사 전체를 보는 사람 |
| **AGENT** | 대리점장 = 자신의 팀만 보는 사람 |
| **FREE_SALES** | 판매원 = 차단된 사람 |
