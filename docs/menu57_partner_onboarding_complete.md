| 렌즈 | 개념 | 신입 파트너 적용 | 기대 효과 |
|------|-----|-----------------|----------|

| **L1** | 가격 이의 대응 | "가치 재정의" 5단계 콜 | 전환율 42-48% |

### PASONA 6단계 콜 구조

```
신입 파트너 일일 콜 흐름:

P (Problem): "크루즈 여행 계획 중이신가요?" → 고객 상황 파악
A (Agitate): "일반 관광과 다른 매력이 있습니다" → 동기 자극
S (Solution): "저희 프리미엄 크루즈가..." → 솔루션 제시
O (Offer): "이번 달 특가 20% 할인" → 구체적 제안
N (Narrow): "자리가 5개만 남았어요" → 희소성 강조
A (Action): "지금 예약하시면..." → 즉시 행동 촉구
```

### SPIN 질문 기법 (이의 대응용)

```
S (Situation): "현재 여행 일정은 언제쯤 생각하세요?"
P (Problem): "혹시 가족과 함께하시는 거라 일정이 복잡한가요?"
I (Implication): "그럼 가족이 모두 참여하는 특별한 경험이 되겠네요?"
N/P (Need/Payoff): "프리미엄 서비스로 그런 경험을 확실히 할 수 있습니다"
```

### 신규 모델: PartnerPerformance

```prisma
model PartnerPerformance {
  id                String    @id @default(cuid())
  partnerId         String
  year              Int
  month             Int
  week              Int?
  
  // 기본 KPI
  totalCalls        Int       @default(0)
  appointmentsMade  Int       @default(0)
  salesClosed       Int       @default(0)
  revenue           BigInt    @default(0)
  
  // 전환율 지표
  callToAppointmentRate Float? // (appointmentsMade / totalCalls)
  appointmentToSaleRate Float? // (salesClosed / appointmentsMade)
  overallConversionRate Float?
  
  // CPA 및 LTV
  costPerAcquisition BigInt?
  lifetimeValue     BigInt?
  
  // 자동화 상태
  automationScore   Int       @default(0) (0-100)
  
  // 위험 신호
  riskScore         Int       @default(0) (0-100)
  riskFlags         String[]  @default([])
  
  partner           Partner   @relation(fields: [partnerId], references: [id])
  
  @@unique([partnerId, year, month, week])
  @@index([partnerId, year, month])
  @@index([riskScore])
}
```

## 🤖 자동화 시스템

### 주간 성과 리포팅 (자동)

```
매주 금요일 18:00 이메일 + SMS 발송:

「[파트너명] 주간 KPI 리포트 📊」

총 콜: 75회 / 목표 75회 ✅
약속: 11명 / 목표 10명 ✅
성약: 2명 / 목표 2명 ✅
매출: 1,200만원 / 목표 1,000만원 ✅

이번주 성과: 우수! 계속 화이팅! 🎉
다음주 목표: 약속 12명, 성약 2명

담당자 연락: 010-XXXX-XXXX
```

### 위험 신호 자동 감지

```
시스템 규칙:

Rule 1: 3주 연속 저성과 (목표의 50% 미만)
  → SMS 자동 발송 + 담당자 알림

Rule 2: 활동 급감 (전주 대비 50% 이상 감소)
  → 이탈 징후 감지 + 개입 예약

Rule 3: 불정직 신호 (콜 수 vs 성약율 불일치)
  → 콜 녹음 검토 요청

Rule 4: 기술 부족 (특정 이의 처리율 20% 이하)
  → 자동 재교육 자료 발송
```

### 인센티브 자동 계산

```
매월 1일 자동 실행:

1. 지난달 KPI 수집
2. 성약수 × 단가 = 기본 매출
3. 보너스 조건 체크:
   - 목표 달성: +10%
   - 전월 대비 증가: +5%
   - 전환율 상위 10%: +5%
4. 총 인센티브 자동 계산
5. 회계팀 보고서 생성
```
