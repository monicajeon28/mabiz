## 📋 개요

Phase 2에서 API와 데이터 필터링을 완료했고, Phase 3에서는 **5개 버튼에 역할별 권한을 적용**합니다.

초등학생도 이해할 수 있는 수준으로 설계했습니다:
- **활성** = 클릭 가능 (파란색)
- **비활성** = 보이지만 클릭 불가 (회색) + 비활성 이유 표시
- **숨김** = 아예 안 보임

## 🔧 구현 방식

### Phase 2 → Phase 3 연결

**Phase 2** (API & 데이터 필터링):
```typescript
// GET /api/commission-ledger
// 역할별로 자동으로 데이터 필터링됨
GLOBAL_ADMIN → 전체 데이터
OWNER → 자신의 팀 데이터만
AGENT → 본인 데이터만
```

**Phase 3** (UI 버튼 권한):
```typescript
// 같은 역할 정보를 사용하되, UI 버튼의 활성/비활성을 결정
getAllButtonPermissions(userRole)
 → { settle, dispute, verify, excel, recalculate }
```

**핵심**: Phase 2 API는 데이터를 제한하고, Phase 3은 UI에서 버튼을 보여줄지 말지 결정합니다.
- **안전성**: 데이터는 API에서 먼저 필터링됨
- **UX**: 버튼은 UI에서 미리 숨겨서 혼동 방지

## 💡 주요 기능

### 1. 버튼 상태 3가지

```typescript
type ButtonStatus = 'enabled' | 'disabled' | 'hidden';

// enabled  = 파란색, 클릭 가능
// disabled = 회색, 보이지만 클릭 불가 (이유 표시)
// hidden   = 아예 안 보임
```

### 2. 비활성 이유 표시 (호버 시)

```
비활성 이유 (초등학생 이해 수준):
"정산은 본사 관리자만 처리할 수 있어요.
 본사에 정산 요청을 해주세요."
```

### 3. 엑셀다운 데이터 범위 표시 (호버 시)

```
"전체 팀원의 수당 기록을 다운로드합니다"
"당신 팀 판매원의 수당만 다운로드합니다"
"당신의 수당 기록을 다운로드합니다"
```

### 4. 역할 설명 배너

```
👤 당신은: 본사 관리자
마비즈 전체를 관리하는 사람
모든 팀의 수당을 보고 정산할 수 있어요
```
