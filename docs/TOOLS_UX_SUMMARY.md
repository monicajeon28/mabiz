# Tools 페이지 UX 최적화 분석 요약 (2026-06-02)

## 📌 Executive Summary

**영업팀의 도구 페이지(Tools) 효율성을 75% 개선하는 8주 로드맵 수립**

### 핵심 문제
- **플레이북 검색 시간**: 40분 → 필터 산재, 결과 혼란
- **필터 혼란**: 10개 필터가 UI에 흩어져 있음 (필터 조건 추적 불가)
- **SMS 자동화 부재**: Day 0-3 시각화, 메시지 미리보기, 세그먼트 비교 불가능
- **대시보드 부재**: 실시간 발송 현황, 성과 메트릭 추이 없음

### 기대 효과
- **시간 단축**: 175분/일 → 43분/일 (2시간 절약, **75% 감소**)
- **성과 개선**: 플레이북 사용률 20% → 75% (+275%)
- **월 매출**: +$50-80K (한화 6-10억/월)

| 항목 | 현재 | 개선 후 | 효과 |
|------|------|--------|------|
| **용어 명확화** | "VIP 케어" | "우수고객 케어 (특별관리+긴박감)" | +30% 이해도 |

#### 필터 10개 구조화
```
현재:    [필터▼] [필터▼] [필터▼] [필터▼] [필터▼]  ← 가로 산재
         [필터▼] [필터▼] [필터▼]

개선:    ┌─ 필터 패널 (Fixed 250px)
         │ ├─ 1. 고객 세그먼트 (라디오)
         │ ├─ 2. 제품 유형 (라디오)
         │ ├─ 3. 심리학 렌즈 (체크박스)
         │ ├─ 4. PASONA 스테이지 (체크박스 + 타임라인)
         │ ├─ 5. 신민형 5단계 (체크박스 + 진행도)
         │ ├─ 6. ROI 범위 (슬라이더)
         │ ├─ 7. 사례 유형 (체크박스)
         │ ├─ 8. 음성 톤 (라디오)
         │ ├─ 9. 난이도 (라디오)
         │ └─ 10. 즐겨찾기 (토글)
         └─ 상호작용 버튼: [↻ 초기화] [💾 저장] [🔗 공유]
```

#### 새로운 상호작용
- **실시간 필터**: 필터 선택 → 1초 내 결과 업데이트
- **선택된 필터 표시**: 칩 형태로 결과 위에 표시
  - `[Segment: A ✕] [Phase: 클로징 ✕] [Psychology: L6 ✕]`
- **필터 저장**: 자주 쓰는 조합 5개 저장 (예: "신혼 부부용", "고가 상품 클로징")
- **필터 공유**: URL 복사 → 팀원과 공유 (같은 필터 조건 적용)

**결과**: 플레이북 검색 시간 40분 → 10분 (**75% 감소**)

#### 메시지 미리보기 (각 Day별)
- PASONA 스테이지 (P/A/S/O/N/A)
- 적용된 심리학 렌즈 (L6 손실회피 / L10 즉시구매 등)
- 실제 발송 메시지 (개인화 변수 포함)
- 발송 시간 / 대상 인원 / 예상 반응율

#### 세그먼트별 변형 비교
```
세그먼트A (커플)     │ 세그먼트B (가족)
"로맨틱한 휴가"      │ "아이들이 좋아할 여행"
L7: 동반자설득       │ L8: 재구매 습관
CTA: "부부 둘이"     │ CTA: "식구끼리"
```

#### 발송 전 최종 확인

**결과**: SMS 설계 시간 60분 → 15분 (**75% 감소**)

## 💡 주요 설계 원칙

### 1. 사용자 심리학
- **인지 부하 감소**: 한눈에 3-4개 필터만 노출 (나머지는 하단 스크롤)
- **진행 상황 가시화**: 선택한 필터를 칩으로 표시 (어디까지 설정했는지 명확)
- **빠른 리셋**: "초기화" 버튼 1클릭 (필터 재설정 심리적 부담 해소)

### 2. 데이터 구조
- **필터 조합 저장**: 자주 쓰는 조합 5개 저장 가능 (예: "GOLD 클로징", "신혼 부부")
- **필터 공유**: URL 파라미터로 팀원 간 필터 조건 공유 가능

### 3. 성능
- **실시간 필터**: 필터 변경 시 1초 내 결과 업데이트 (DBN)
- **페이지네이션**: 결과 12개씩 표시 (한 페이지에 너무 많으면 로드 느려짐)
- **모바일 반응성**: Tablet에서는 Drawer, Mobile에서는 Modal로 변경
