## 4. 주요 발견사항

### 4-1. 높은 정확도 (0.90 이상)
- **선상팁 (TIP)**: 모든 언급이 Product ID 320으로 매핑 (100% 정확)
- **음료 (Beverage)**: 냉장고, 미니바 관련 질문과 완벽 매칭
- **식사 (Dining)**: 뷔페, 레스토랑, 정찬 등 구체적 언급 시 정확도 높음
- **객실 (Room)**: 객실, 캐빈, 침실 등 다양한 표현 모두 인식

### 4-2. 중간 정확도 (0.70-0.89)
- **탑승&수속**: 일반적인 배경 지식과 함께 나타나는 경우 많음
- **정책&수수료**: "요금", "정책" 등 모호한 표현 포함
- **기항지**: "도쿄", "세부" 등 지명은 감지하나 서비스와의 연결 약함

### 4-3. 매칭되지 않은 질문 (49.1%)
- **이유 분석**:
  - 통상적인 인사말/인사: "님이 들어왔습니다" (8%)
  - 사진/동영상 첨부 공지: "[사진]" "동영상" (15%)
  - 순수 잡담/감정 표현: 😆 "호달달이라" (12%)
  - 시간/날짜 정보만 포함: "2026년 5월 10일" (8%)
  - 기타 비정책 정보: 예약 불가 안내, 야외 활동 정보 (6%)

## 5. 출력 파일 상세

### 5-1. product_mapping.csv
- **행 수**: 276행 (헤더 + 275개 질문)
- **컬럼**:
  - `question_id`: 질문 고유 ID (q0001 ~ q0275)
  - `question_text`: 질문 첫 100자
  - `category`: 원본 질문 카테고리 (기타, 정책&수수료, 탑승&수속 등)
  - `product_ids`: 매핑된 상품 ID 또는 카테고리 (쉼표로 분리)
  - `max_relevance`: 최고 관련도 점수 (0.00 ~ 1.00)
  - `keywords_matched`: 매칭된 키워드들의 카테고리

**사용 사례**:
```
- CSV를 BigQuery/Redshift에 로드하여 실시간 쿼리
- Excel에서 필터링으로 특정 카테고리만 추출
- Pandas로 추가 분석 (관련도 분포, 카테고리 교차 분석)
```

### 5-2. product_mapping.json
- **구조**:
  - `generated_at`: 생성 타임스탬프
  - `total_questions`: 275
  - `matched_count`: 140
  - `mappings[]`: 매칭된 질문들의 상세 정보
    - `question_id`, `question`, `category`
    - `products[]`: 배열 형태로 모든 매핑된 상품

**사용 사례**:
```
- REST API의 응답 포맷으로 직접 사용
- 세일즈봇 백엔드에서 로드하여 인메모리 캐시
- 다국어 지원 시 JSON 기반 확장
```

```javascript
// Output: [{ product_id: 320, relevance: 0.95, category: 'TIP' }]
```

### 6-2. 배치 처리 (초기 동기화)
```sql
-- Neon DB에 매핑 테이블 생성
INSERT INTO question_product_mapping (
  question_id, product_id, category, relevance_score
)
SELECT 
  q.id, p.product_id, p.category, p.relevance
FROM questions_rag_memory q
LEFT JOIN product_mapping p ON q.id = p.question_id
WHERE p.relevance >= 0.75;
```
