## 🎓 배운 점 & 개선사항

### What Went Well ✅

1. **검색 알고리즘이 견고함**
   - 키워드 기반 방식이 크루즈 도메인에 적합
   - 한글/영문 혼합 처리 우수
   - 이모지, 특수문자 정규화 효과적

2. **데이터 품질이 높음**
   - 275개 Q&A가 깔끔하게 정리됨
   - 8개 카테고리가 균형있게 분포
   - 상품 매핑이 체계적

3. **성능이 탁월함**
   - JSON 기반 처리 (DB 없이도 가능)
   - 응답 시간 <1ms (프론트엔드 캐싱 불필요)
   - 스케일링에 최적화됨

### What Could Be Better 🔧

1. **판매톤 신뢰도 부족**
   - 키워드 기반 방식의 한계
   - 문맥 이해 필요 (NLP 도입)
   - 더 많은 학습 데이터 필요

2. **테스트 설계 개선**
   - 통합 시나리오에서 카테고리명 오류
   - 향후에는 사전에 데이터 구조 확인 필수

3. **모니터링 부족**
   - 실제 배포 후 사용자 피드백 수집 메커니즘 필요
