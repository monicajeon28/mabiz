## 2. 최적화 시나리오 3가지

### 시나리오 1️⃣ : 기본 최적화 (2주, 권장)

**목표:** 저장소 35% 감소, 성능 20% 향상

#### 변경 사항
- ✅ 키워드: 10개 → 5개 제한
- ✅ 불필요 필드 제거: hash, length, source
- ✅ Type 코드화: 4자 → 1자 (P/T/Q/S)
- ✅ Category 코드화: 8가지 → 1-8 숫자
- ✅ 메타데이터 최소화

#### 결과
```
현재:         2.8 MB → 기본 최적화: 1.8 MB
저장소 감소:  35% ↓
쿼리 성능:    150ms → 98ms (35% 향상)
로드 시간:    1.54배 빨라짐
```

#### 비용 & 일정
- **구현 기간:** 2주
- **팀 규모:** 2명
- **예상 비용:** $2,000
- **위험도:** 🟢 낮음
- **복잡도:** 🟢 낮음


## 3. 권장사항 (Best Practices)

### 배치 업로드 최적화
```
배치 크기: 50개 (권장)
네트워크 시간: ~50ms
재시도 전략: Exponential Backoff (100ms → 10s)
병렬 업로드: 4개 스트림
성공률 목표: 99.9%
```

### 캐싱 전략
```
Cache Backend: Redis 7.0+
TTL: 300초 (5분)
Key Pattern: qa:category:{id}:page:{n}
캐시 워밍: 시작 시 상위 20% 데이터
무효화: 수정 시 즉시 + 6시간 주기
히트율 목표: 80% 이상
```

### 페이지네이션
```
기본값: limit=20
최대값: limit=100
최소값: limit=5
권장방식: 커서 기반 (ID 기반)
```

### 인덱싱 전략
```
1️⃣ 기본 인덱스 (B-Tree)
   - category + travelPhase
   - 커버리지: 65%
   - 성능: 5배 빨라짐

2️⃣ 키워드 인덱스 (GIN)
   - 역색인 (keyword → id 매핑)
   - 커버리지: 30%
   - 성능: 100배 빨라짐

3️⃣ 검색 인덱스 (FTS)
   - question + answer 전문검색
   - 커버리지: 80%
   - 성능: 50배 빨라짐
```
