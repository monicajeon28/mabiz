## 📋 역할별 권한 정의 (초등학생 수준)

### 👔 **관리자** (최고 권한자)
- 모든 메뉴 볼 수 있음 ✅
- 모든 고객 관리 가능 ✅
- 모든 문자/이메일 발송 가능 ✅ (조직 대표 발송)
- 다른 사람 수당/매출 볼 수 있음 ✅
- 설정 변경 가능 ✅

### 🎯 **대리점장** (관리자 못할 때 대신 하는 사람)
- 자기 조직의 고객만 관리 ✅
- 자기 조직 고객한테 문자/이메일 발송 ✅ (대리점 대표)
- 자기 조직의 수당/매출 볼 수 있음 ✅
- 자기 조직 설정 변경 가능 ✅
- **못하는 것**: 다른 대리점 데이터 볼 수 없음 ❌

### 👤 **판매원** (고객 관리하는 사람)
- 자기 담당 고객 정보 볼 수 있음 ✅
- 자기 고객한테만 개인적으로 문자/이메일 발송 ✅ (자신의 알리고/이메일 설정)
- 자신의 고객 목록만 봄 ✅
- **못하는 것**: 다른 판매원 고객 못 봄, 조직 전체 발송 못 함 ❌

## ✅ P0 (지금 당장 - 이번주) - 4개 Task

### **P0-1: 수익계산기 → 관리자만 보기** (1시간)

**문제**: 지금은 대리점장도 볼 수 있음 (근데 보안상 관리자만 봐야 함)

**파일**:
```
src/lib/route-rules.ts → /tools/profit-calculator 경로 수정
```

**수정 방법** (초등학생 수준):
```
현재: "대리점장도 들어올 수 있어" → if (role === 'AGENT')
바뀜: "관리자만 들어올 수 있어" → if (role === 'GLOBAL_ADMIN')
```

**테스트**:
- 관리자: 수익계산기 클릭 → 보임 ✅
- 대리점장: 수익계산기 클릭 → "접근 불가" ✅
- 판매원: 수익계산기 클릭 → "접근 불가" ✅

---

### **P0-2: 문자 발송 권한 추가 (조직 전체)** (2시간)

**문제**: 누구나 조직 전체 고객한테 문자 보낼 수 있음 (위험! 판매원이 실수로 보낼 수 있음)

**원하는**: 관리자/대리점만 조직 전체 발송 가능, 판매원은 자신의 고객한테만

**파일**:
```
src/app/(dashboard)/messages/page.tsx → 발송 버튼 권한 체크
src/app/api/groups/[id]/blast/route.ts → API 권한 검증
```

**구체적인 수정**:

```javascript
// "조직 전체 발송" 버튼 (현재: 항상 보임)
// 바뀜: 관리자/대리점만 보임

if (role === 'GLOBAL_ADMIN' || role === 'OWNER') {
  // "전체 발송" 버튼 표시 ✅
} else {
  // "내 고객만 발송" 버튼 표시 (판매원/개인용)
}
```

**테스트**:
- 관리자: "전체 발송" + "내 것만 발송" 버튼 둘 다 보임 ✅
- 대리점장: "전체 발송" + "내 것만 발송" 버튼 둘 다 보임 ✅
- 판매원: "내 것만 발송" 버튼만 보임, "전체" 버튼은 회색 처리 ✅
- 판매원이 "전체" 클릭하면: "권한이 없습니다" 알림 ✅

---

### **P0-3: 이메일 발송 권한 추가** (1.5시간)

**문제**: 이메일도 누구나 보낼 수 있음

**원하는**: 문자와 동일하게 (관리자/대리점은 조직 전체, 판매원은 자신의 고객만)

**파일**:
```
src/app/(dashboard)/messages/page.tsx (이메일 탭)
src/app/api/messages/send-email/route.ts → API 권한 검증
```

**수정 방법**:
```
문자와 동일한 로직 복사-붙여넣기
(이메일도 같은 권한 체계 사용)
```

**테스트**: 문자와 동일 ✅

---

### **P0-4: 문자 기록 조회 권한** (2시간)

**문제**: 누구나 모든 발송 기록을 봄 (개인정보 노출 위험)

**원하는**: 
- 관리자: 모든 문자 기록 봄 ✅
- 대리점장: 자기 조직 문자만 봄 ✅
- 판매원: 자신이 보낸 문자만 봄 ✅

**파일**:
```
src/app/api/sms-logs/route.ts → 조회 권한 필터 추가
```

**구체적인 수정**:

```javascript
// 현재: organizationId 필터만 있음
// 바뀜: organizationId + 역할별 필터 추가

if (role === 'GLOBAL_ADMIN') {
  // 모든 문자 기록 조회
  query = { organizationId }
} else if (role === 'OWNER') {
  // 자기 조직 것만
  query = { organizationId }
} else if (role === 'AGENT' || role === 'SALES') {
  // 자신이 보낸 것만 (createdBy = userId)
  query = { organizationId, createdBy: userId }
}
```

**테스트**:
- 관리자: 전체 발송 기록 봄 ✅
- 대리점장: 자신의 조직 문자만 봄 ✅
- 판매원 A: 자신이 보낸 문자만 봄, 판매원 B 것은 못 봄 ✅

## 🔄 P1 (곧하기 - 다음주) - 3개 Task

### **P1-1: 그룹 관리 클라이언트 권한 체크** (1.5시간)

**문제**: 그룹을 누구나 만들고 삭제할 수 있음 (데이터 손상 위험)

**원하는**: 소유자만 자신의 그룹을 지울 수 있음

**파일**:
```
src/app/(dashboard)/groups/page.tsx → "삭제" 버튼 권한 체크
```

**수정**:
```javascript
// "삭제" 버튼을 조건부로 표시
if (group.createdBy === userId || role === 'OWNER' || role === 'GLOBAL_ADMIN') {
  // "삭제" 버튼 표시 ✅
} else {
  // "삭제" 버튼 회색 처리 (클릭 불가)
}
```

**테스트**:
- 소유자: "삭제" 버튼 보임 ✅
- 다른 사람: "삭제" 버튼 회색 (클릭 불가) ✅

---

### **P1-2: 랜딩페이지 조직 전체 보기 권한** (2시간)

**상황**: 랜딩페이지는 **마케팅 도구** (CRM이 아님)
- 개인이 마케팅용으로 만든 랜딩페이지 → 자신만 봄
- 조직 전체에 공개된 랜딩페이지 → 대리점장/관리자가 공개 범위 관리

**원하는**:
- 관리자: 모든 랜딩페이지 봄 ✅
- 대리점장: 자신의 조직 페이지만 봄 ✅
- 판매원/개인: 자신이 만든 것만 봄 ✅

**파일**:
```
src/app/(dashboard)/landing-pages/page.tsx → 조회 필터
src/app/api/landing-pages/route.ts → 권한 검증
```

**테스트**: "조직 전체" 탭이 관리자/대리점장만 보임 ✅

---

### **P1-3: 파트너 데이터 격리** (1시간)

**문제**: 모든 조직의 파트너 데이터가 보임 (멀티테넌트 위반)

**원하는**: 자신의 조직 파트너만 봄

**파일**:
```
src/app/api/partner/list → organizationId 필터 추가
```

**테스트**:
- 대리점1: 자신의 파트너만 봄 ✅
- 대리점2: 대리점1 파트너 못 봄 ✅
