## 📋 측정 지표

### Primary Metrics (핵심 지표)

#### 1. 전환율 (Conversion Rate)
```
정의: (예약 완료 고객 수 / 총 콜 수) × 100%
목표: Group B > Group A + 7%

계산:
- Group A: 예약 7건 / 30건 = 23% (baseline)
- Group B: 예약 10건 / 30건 = 33% (목표)
```

**측정 방법**:
- CRM에서 "Status = Booked" 필터링
- 콜 완료일로부터 7일 이내 예약 확정 기준

#### 2. 통화 길이 (Call Duration)
```
정의: 평균 콜 시간 (분)
목표: Group A 18분 → Group B 15분 (3분 단축, 효율 +17%)

가설:
- 세일즈봇 QnA 참고로 불필요한 설명 반복 제거
- 담당자가 더 정확한 정보 제공
- 고객 의사결정 가속화
```

**측정 방법**:
- 콜 로그 시스템에서 자동 기록
- 요약 통계 (평균, 중앙값, 사분위수)

#### 3. 고객 만족도 (Customer Satisfaction)
```
정의: 콜 후 즉시 설문 (5점 만점)
목표: Group A 3.8점 → Group B 4.3점 (+0.5점)

설문 항목:
- Q1: 담당자의 설명이 명확했나요? (1-5)
- Q2: 예약을 권하고 싶나요? (1-5)
- Q3: 이 경험을 다시 원하나요? (1-5)
```

**측정 방법**:
- 콜 후 SMS/WhatsApp으로 설문 링크 자동 발송
- 3분 내 응답 권장 (최고 응답율 시간대)

---

### Secondary Metrics (보조 지표)

#### 4. 재접촉 가능성 (Callback Rate)
```
정의: "생각해봐요" → 재접촉 요청한 고객 비율
목표: Group B가 Group A보다 15% 이상 높음

가설:
- Group B는 자동 이메일로 "욕구 발굴" QnA 발송
- 감정적 재접촉으로 재예약 가능성 증가
```

**측정 방법**:
- CRM Status = "thinking" → "scheduled_callback"으로 변경된 고객 추적
- 1주일 추적 기간

#### 5. 세일즈봇 QnA 활용율 (Adoption Rate)
```
정의: (세일즈봇 앱에서 QnA를 보기만 한 횟수) / (총 콜 횟수)
목표: 신민형 담당자가 70% 이상의 콜에서 세일즈봇 사용

가설:
- 사용율이 높을수록 전환율도 높음
- 담당자의 숙련도 향상 추적 가능
```

**측정 방법**:
- API 로그 분석 (SalesbotQnALog.userAction = 'clicked')
- 담당자별 일일 리포트

#### 6. 가장 효과적인 QnA (Top Performers)
```
정의: 가장 높은 전환율을 기록한 Q&A
목표: Step별 Top 3 QnA 식별

예상 결과:
- Step 1: Q16 (배타성) 가장 효과적
- Step 2: Q3 (감정적 이미지) 또는 Q5 (시간 손실회피)
- Step 3: Q8 (가격) 또는 Q6 (일정 설명)
- Step 4: Q8 + Q18 조합
- Step 5: Q22 (의미) 또는 Q37 (기회 상실)
```

**측정 방법**:
- SalesbotQnALog에서 conversionResult = 'yes'인 것만 추출
- 카테고리/Step별 전환율 계산

## 🚫 Risk & Mitigation

### Risk 1: 신민형 담당자의 적응 곡선
```
문제: 세일즈봇 앱을 사용하며 대화 자연성 손상 가능
완화:
- 충분한 사전 교육 (Week 1 전체)
- 실제 콜 5건 연습
- Supervisor의 실시간 모니터링 (첫 5콜)
- "자연스럽게 녹여내기" 강조

통과 기준:
- Week 1: 신민형의 고객 만족도 > 3.8점
- 만약 < 3.5점이면 Week 2 추가 교육
```

### Risk 2: 고객 표본 편향
```
문제: 모니카 vs 신민형이 받는 고객 특성 다를 수 있음
완화:
- CRM에서 무작위로 콜 대상 선정
- 고객 세그먼트 균형 유지
- 콜 시간대 동일화

검증:
- 고객의 연령, 여행 경험, 가격 민감도 분포 비교
- 만약 분포가 너무 다르면 재샘플링
```

### Risk 3: 학습 효과 (Learning Effect)
```
문제: 신민형이 2주간 콜하며 자연스럽게 실력 향상
개선 vs 세일즈봇 효과 분리 어려움
완화:
- 콜을 시간 순서대로 분석 (Week 1 vs Week 2)
- 신민형의 "초기 5콜" vs "후기 5콜" 비교
- 모니카도 같은 기간 콜하며 자연 학습 포착

통과 기준:
- 만약 신민형의 증가율이 모니카와 같으면 세일즈봇 효과 없음
- 신민형 > 모니카 + 3%이면 유의미한 효과
```

### Risk 4: 기술 장애
```
문제: 세일즈봇 앱 다운, API 지연 등
완화:
- App 안정성 사전 테스트
- Fallback: 만약 앱 안 되면 종이 노트로 QnA 준비
- 신민형의 노트북/태블릿 백업 준비

영향:
- 만약 24시간 이상 다운되면 그 날은 제외
```
