# 04. 체크리스트 (CRM 구축 → 몰 이관 → 검증)

## STEP 1. CRM 구축 (CRM팀)
- [ ] 발급 폼: 역할(BRANCH_MANAGER/SALES_AGENT/PRE_SALES/HQ) 선택
- [ ] 발급 시 `User` 생성: `phone=mallUserId=prefix+번호`, `password=bcrypt(초기값)`, `role='community'`
- [ ] mallUserId 채번: prefix(boss/user/free) + 충돌검사(`User.phone startsWith` → 빈 번호) — `01_데이터모델` §2 규칙 동일 구현
- [ ] `AffiliateProfile` 생성: type/status='ACTIVE'/contractStatus='SIGNED'/affiliateCode(unique)/withholdingRate/계좌/계약증적/metadata{source:'CRM'}
- [ ] 판매원·프리세일즈: `AffiliateRelation(managerId, agentId, ACTIVE)` 생성
- [ ] 비번 bcrypt + `PasswordEvent` 기록(평문 금지)
- [ ] 계약서 싸인 플로우 + 증적 저장(contractSignedAt/Signature/IP/Version)
- [ ] (KYC를 CRM이 하면) 신분증·통장 수집/승인
- [ ] 발급 직후 몰 `POST /api/internal/affiliate/provision` 호출(기본 링크·고객그룹)

- [ ] 🔴 로그인 자동생성 차단 (`app/api/auth/login/route.ts`) — 최우선

- [ ] 계약 승인(approve) 발급 제거/비활성 (계약=CRM 확정 시)
- [ ] (선택) `PRE_SALES` 타입 분기 보강(requirePartnerContext/대시보드)

## STEP 3. 검증
