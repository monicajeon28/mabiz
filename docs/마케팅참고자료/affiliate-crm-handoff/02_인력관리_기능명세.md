# 02. 인력관리 기능 명세 (CRM이 재현해야 할 목록)

> 현재 몰 `/admin/affiliate/profiles` 화면이 제공하는 기능 전체.
> CRM이 "발급"을 맡으려면 아래 중 **[발급] 표시 기능**을 동등하게 갖춰야 한다.
> **[운영] 표시 기능**은 정책상 몰에 남길 수 있음(옵션 D).

## 역할(type) 체계
- `BRANCH_MANAGER` 대리점장 / `SALES_AGENT` 판매원 / `PRE_SALES`(=PRESALES) 프리세일즈 / `HQ` 본사
- 프리세일즈 = "판매원인데 소속 대리점장이 없는" 형태 (프론트 별도 옵션 없이 type/소속으로 구분)
- ⚠️ **CRM 구축 시 `PRE_SALES`까지 정식 지원** 필요 (현재 몰 일부 로직이 BRANCH_MANAGER/SALES_AGENT만 분기 → 이관 시 몰도 PRE_SALES 분기 보강 예정)

## 발급 시 입력받는 항목 (CRM 발급 폼이 가져야 할 필드)
- 역할(type), 이름/표시명/닉네임, 연락처(전화/이메일)
- 소속 대리점장(판매원/프리세일즈일 때) → AffiliateRelation
- 정산 계좌(은행/계좌번호/예금주), 원천징수율(기본 3.3), 수당율
- 보증인(이름/ID)
- 계약 싸인 증적(서명/IP/버전/시각) → 계약을 CRM에서 받으므로 자연 확보
- (선택) 랜딩 슬러그, 프로필 이미지/소개

## 현재 몰 코드의 알려진 갭(참고 — 이관과 별개)
- 편집 PUT의 `password`가 서버에서 미반영(비번 변경 안 됨) → CRM이 발급/초기화를 맡으면 해소
- 프로필 생성 POST는 평문 `1101` 저장(레거시) → CRM은 bcrypt
- 목록 GET의 카운트(팀원/링크/리드 수)는 0으로 하드코딩(실값 아님)
- 일부 API(fix-relations/documents)는 구식 권한검사 + fix-relations POST에 `req` 참조 버그
