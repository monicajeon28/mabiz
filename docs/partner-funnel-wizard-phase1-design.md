## 🎯 Phase 1 목표

### 1단계: 렌즈별 템플릿 설계
- L0-L10 각 렌즈별 Day 0-3 메시지 템플릿 확정 (PASONA + Grant Cardone 기법)
- 3가지 전략 (추천/인기/최신) 매핑

- 미리보기 레이아웃 설계 (폰 모양)

### 2단계: DB 스키마 확장
- FunnelSms / FunnelEmail 테이블에 10개 신규 필드 추가
- 인덱스 3개 설계 (조직별 + 생성자별 + 렌즈별)
- Prisma 마이그레이션 스크립트

### 3단계: 시스템 템플릿 라이브러리

## 🗂️ DB 스키마 확장 (Phase 1)

### FunnelSms / FunnelEmail 신규 필드 (10개)

```prisma
model FunnelSms {
  // 기존 필드
  id              String
  organizationId  String
  title           String
  senderPhone     String?
  category        String?
  description     String?
  sendHour        Int @default(10)
  sendMinute      Int @default(0)
  arsNum          String?
  isActive        Boolean @default(true)
  createdByUserId String?
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
  
  // Phase 1 신규 필드 (10개)
  // ===================================
  
  // 1. 렌즈 타입 (L0-L10)
  lensType        String? @db.VarChar(3)  // 예: "L0", "L1", "L6", "L10"
  
  // 2. 가시성 (개인/팀/조직 공개)
  visibility      String @default("PERSONAL") @db.VarChar(10)  // PERSONAL | TEAM | PUBLIC
  
  // 3. 생성자 권한 (누가 수정 가능한가?)
  createdByRole   String? @db.VarChar(20)  // PARTNER | MANAGER | ADMIN (권한 검증용)
  
  // 4. 공유 대상 (JSON 배열: [userId1, userId2, ...])
  sharedWith      String[] @default([])  // 배열로 저장 (PostgreSQL native)
  
  // 5. 템플릿 여부
  isTemplate      Boolean @default(false)  // true면 조직 공용 템플릿
  
  // 6. 위험도 점수 (0-100, 자동계산)
  riskScore       Int @default(0)  // 욕설/스팸 감지 점수
  
  // 7. 위험 신호 (JSON 배열: ["욕설", "외부링크", "과다마케팅", ...])
  riskFlags       String[] @default([])  // 감지된 위험신호들
  
  // 8. 버전 관리
  versionNumber   Int @default(1)  // 수정할때마다 +1
  
  // 9. 부모 퍼널 ID (템플릿에서 복제한 경우)
  parentFunnelId  String?  // 원본 템플릿 ID
  
  // 10. 메타데이터 (JSON 자유형)
  metadata        Json? // { strategy: "추천", duration: "3일", tone: "따뜻함", ... }
}
```

## 📋 3가지 전략 매핑

### Strategy 1: "추천" (시스템 기본)
- 사용: 가장 많이 선택된 렌즈별 템플릿
- 메시지: 검증된 PASONA 구조
- 데이터 출처: `LensTemplate.isSystemTemplate = true`
- 변수: 기본값 (고객명, 상품명만)

### Strategy 2: "인기" (조직 Top 3)
- 사용: 조직 내에서 가장 높은 응답율 퍼널
- 메시지: 실제 판매원들이 쓰는 메시지
- 데이터 출처: `FunnelSms.visibility = "PUBLIC"` + 응답율 순정렬
- 변수: 동적 변수 최대 5가지

### Strategy 3: "최신" (조직 최근 생성)
- 사용: 조직에서 최근 1주일 내 생성된 퍼널
- 메시지: 최신 마케팅 트렌드 반영
- 데이터 출처: `FunnelSms.createdAt >= (now - 7days)` 정렬
- 변수: 최신 동적 변수

## 🔄 동적 변수 5가지

### 1. {고객명} - Contact.name
```
예시: "김철수님, 안녕하세요?"
폴백: 이름 없으면 -> "회원님" 또는 Contact.phone 앞4자리
```

### 2. {상품명} - Contact.inquiryProductCode
```
예시: "MSC 매그니피카는"
폴백: 상품코드 없으면 -> "최신 상품" 또는 "선택하신 상품"
```

### 3. {가격} - Contact.inquiryPrice / Product.standardPrice
```
예시: "월 5만원부터"
폴백: 가격 없으면 -> "특별가격" 또는 "[가격 보기]"
포맷: 숫자 -> 천단위 쉼표 (500000 -> "50만원")
```

### 4. {톤} - 자동선택 (렌즈별 결정)
```
L0: "따뜻함"
L1: "논리적"
L6: "긴박함"
L10: "축하함"
```

### 5. {시간} - 발송예정시간
```
예시: "내일 10시"
계산: (now + daysAfter) + sendHour:sendMinute
포맷: "Day {0-3} ({요일})"
```

## 🔐 보안 & 검증 (Phase 1 설계)

### 1. 조직별 격리
```
GET /api/funnels/templates?organizationId=org123&lensType=L0
→ 다른 조직 템플릿 절대 보여주지 않음
→ 데이터베이스 쿼리: WHERE organizationId = ctx.organizationId
```

### 2. 권한 검증
```
FunnelSms.visibility = "PERSONAL"
  → createdByUserId만 수정 가능
FunnelSms.visibility = "TEAM"
  → createdByRole이 MANAGER 이상만 수정 가능
FunnelSms.visibility = "PUBLIC"
  → 조직 ADMIN만 수정 가능 (판매원은 읽기만)
```

### 3. Rate Limit
```
판매원당 하루 최대 10개 퍼널 생성
→ Redis key: "funnel_create_{userId}_{date}" : count
→ TTL: 24시간
```

### 4. 욕설/스팸 감지 (Phase 2에서)
```
저장 전 검증:
  1. 욕설 필터 (25%): TextRazor 또는 규칙기반
  2. 거짓 검증 (20%): Product 재고 확인
  3. 외부링크 (15%): URL 검출 후 차단
  4. 개인정보 (15%): 주민등록번호/계좌 패턴 차단
  5. Grant Cardone 기법 (30%): 심리학 신호 검증

riskScore >= 65 → 자동 거부 (개선안 제시)
55 <= riskScore < 65 → 수동 검토 필요 (관리자 승인)
riskScore < 55 → 자동 승인
```
