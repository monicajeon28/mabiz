# Aligo 공용 계정 IP 등록 프로세스 가이드

**문서 작성일**: 2026-06-08  
**버전**: 1.0 (공용 계정 IP 등록 표준화)  
**대상**: 마비즈 CRM 관리자

---

## 📋 개요

마비즈 CRM의 모든 판매원(판매원/대리점장/파트너)이 **단일 공용 알리고 계정**으로 SMS를 발송하도록 통합했습니다. 이 문서는 공용 계정의 IP 등록 및 운영 절차를 단계별로 설명합니다.

### 주요 변경사항
- **이전**: 역할별(판매원/대리점장) 개인 알리고 계정 → 각자 발신번호 등록
- **현재**: 공용 알리고 계정 1개 → 모든 역할이 공동 사용 → 발신번호 1개 (회사 대표번호)

---

## ⚡ 빠른 시작 (5분)

### 1단계: 공용 계정 준비 확인
```
공용 계정 생성 여부: ________
계정 Email: admin@mabizcruisedot.com
User ID: ________
API Key: ________
```

### 2단계: Aligo 대시보드 IP 등록
1. https://aligo.in 접속 → **로그인** (공용 계정)
2. **[설정]** → **[API 설정]** → **[IP 화이트리스트]** 이동
3. **[+ 추가]** 클릭 → 아래 CIDR 범위 입력:
   ```
   76.32.0.0/11
   ```
   (Vercel의 아웃바운드 IP 범위 - 동적 변경 대비)
4. **[저장]** 클릭 → 5-10분 대기

### 3단계: 발신번호 등록
1. **[발신번호 관리]** 이동
2. **[+ 새 발신번호]** → 회사 대표번호 입력
   ```
   예: 02-1511-4560 (고정전화) 또는 010-XXXX-XXXX (휴대)
   ```
3. ARS 인증 완료
4. 검증 상태 확인 (1-2시간 소요 가능)

### 4단계: CRM 설정 확인
1. mabizcruisedot.com → **[Settings]** → **[SMS Configuration]**
2. 공용 계정 정보 입력:
   - **API Key**: (위의 API Key)
   - **User ID**: (위의 User ID)
   - **Sender Phone**: (ARS 인증 완료한 발신번호)
3. **[Test Send]** 클릭 → 테스트 문자 수신 확인

## 🔧 상세 설정 프로세스

### 공용 Aligo 계정 생성 (사전 작업)

#### 1.1 계정 생성
1. [Aligo 공식 사이트](https://aligo.in) 접속
2. **[회원가입]** → **[사업자 회원]** 선택
3. 기본 정보 입력:
   - **회사명**: 마비즈(주)
   - **사업자번호**: (등록번호)
   - **담당자 Email**: admin@mabizcruisedot.com
   - **담당자 전화**: 02-1511-4560

#### 1.2 API 인증 정보 확인
계정 생성 후 Aligo 대시보드에서:

1. **[설정]** → **[API]** 메뉴 접속
2. 다음 정보 확인 및 기록:
   ```
   User ID (API Key): ___________________
   Secret Key: __________________________ (외부 노출 금지)
   ```

**주의**: Secret Key는 `.env.local` 또는 DB의 `OrgSmsConfig` 테이블에만 저장 (Git 저장소 금지)

#### 1.3 충전금 확인
- 최소 충전금: **50,000원 이상** 권장
- Aligo → **[충전]** 메뉴에서 충전
- SMS: 약 30원/건, LMS(장문): 약 110원/건

### 단계 1: Aligo 대시보드에서 IP 화이트리스트 등록

#### 1.1 IP 화이트리스트 페이지 접속

**URL**: https://aligo.in/mypage/api_management/

**로그인 계정**: admin@mabizcruisedot.com (공용 계정)

#### 1.2 CIDR 범위 등록 (권장 - 동적 IP 대비)

Vercel의 아웃바운드 IP는 다양한 범위를 사용합니다. 다음 3가지 방법 중 선택:

**방법 A: Vercel 전체 IP 범위 (권장)**
```
76.32.0.0/11
```
- 장점: 한 번 등록으로 모든 Vercel IP 대응
- 단점: 너무 광범위 (보안상 아래 방법 권장)

**방법 B: 최소 필요 범위 (중간 권장)**
```
76.76.0.0/16
```
- Vercel의 주요 출발지 범위

**방법 C: 정확한 IP 기반 등록 (보안 최고)**
1. CRM 실행 후 SMS 발송 시도
2. 오류 로그에서 정확한 IP 확인
3. 그 IP만 등록
4. 동적 변경 시마다 재등록 필요

#### 1.3 등록 절차

1. Aligo 대시보드 → **[설정]** 클릭
   
   ![image](image-aligo-dashboard.png)  
   *Aligo 메인 대시보드*

2. **[API 설정]** 클릭

   ![image](image-api-setting.png)  
   *API 설정 메뉴*

3. **[IP 화이트리스트]** 탭 클릭

   ![image](image-ip-whitelist-tab.png)  
   *IP 화이트리스트 탭*

4. **[+ 추가]** 버튼 클릭

   ![image](image-add-ip-button.png)  
   *IP 추가 버튼*

5. CIDR 범위 입력 (예: `76.32.0.0/11`)

   ![image](image-ip-input.png)  
   *IP 입력 필드*

   | 필드 | 값 | 설명 |
   |------|-----|------|
   | IP/CIDR | 76.32.0.0/11 | Vercel 아웃바운드 범위 |

   ![image](image-save-button.png)

## 🐛 문제 해결

### 오류: "-107" (IP 미등록)

**증상**:
```
Aligo API Error -107: IP 미등록. 
Aligo 대시보드에서 IP를 화이트리스트에 추가하세요.
```

**원인**: 현재 서버의 IP가 Aligo 화이트리스트에 없음

**해결 방법**:

1. 현재 아웃바운드 IP 확인:
   ```
   Settings → [Check My IP] (있으면)
   또는 CRM 로그에서 발신 IP 확인
   ```

2. Aligo 대시보드 → **[IP 화이트리스트]** 재확인:
   - 등록된 IP 목록 확인
   - CIDR 형식 정확성 재확인 (예: `76.32.0.0/11` ← 마지막 `/11` 필수)

3. 등록 직후 5-10분 대기했는지 확인

4. 재설정 방법:
   ```
   a) 기존 IP 항목 삭제
   b) 새로운 IP 또는 CIDR 범위 추가
   c) 5-10분 대기
   d) 테스트 재발송
   ```

---

### 오류: "-99" (인증 실패)

**증상**:
```
Aligo API Error -99: 인증 실패. 
API Key 또는 User ID를 확인하세요.
```

**원인**: 입력한 API Key 또는 User ID가 잘못됨

**해결 방법**:

1. Aligo 대시보드에서 정확한 값 재확인:
   - **[설정]** → **[API]** → User ID와 API Key 복사
   
2. CRM Settings에 정확히 입력:
   - 앞뒤 공백 없음
   - 대소문자 구분
   - 특수문자 확인

3. 재테스트:
   ```
   Settings → [Save Changes] → [Test Send]
   ```

### 오류: "-97" (발신번호 미등록)

**증상**:
```
Aligo API Error -97: 발신번호 미등록. 
Aligo 대시보드에서 발신번호를 등록하세요.
```

**원인**: 입력한 발신번호가 Aligo에 등록되지 않음 또는 ARS 인증 미완료

**해결 방법**:

1. Aligo 대시보드 → **[발신번호 관리]** 접속

2. 발신번호 목록 확인:
   - "✓ 인증완료" 상태인 번호 있는지 확인
   - 없으면 "✗ 인증대기" → ARS 다시 수행

3. CRM Settings에 정확한 번호 입력:
   - 예: `02-1511-4560` (하이픈 포함)
   - 또는 `0215114560` (하이픈 제외, 형식 통일)
   - Aligo와 동일한 형식 사용

4. 저장 후 재테스트

---

### 오류: "-96" (충전금 부족)

**증상**:
```
Aligo API Error -96: 충전금 부족. 
계정에 최소 50,000원 이상 충전하세요.
```

**원인**: Aligo 계정의 잔액 부족

**해결 방법**:

1. Aligo 대시보드 → **[충전]** 메뉴

2. 충전 금액 선택:
   - 초기: 100,000원 이상 권장
   - SMS 단가: ~30원/건

3. 결제 완료 후 5분 내 사용 가능

---

### 오류: "-98" (야간 발송 차단)

**증상**:
```
Aligo API Error -98: 야간 차단. 
21:00 ~ 08:00 시간대에는 발송이 차단됩니다.
```

**원인**: 한국 통신위원회 규정 (야간 스팸 차단)

**해결 방법** (자동 처리 - 조치 불필요):
1. SMS는 자동으로 다음날 08:00 이후에 발송됨
2. 수동 발송 필요 시 시간대 조정:
   ```
   08:00 ~ 21:00 (KST) 사이에 발송하도록 예약
   ```

## 🎓 FAQ

### Q1: 공용 계정으로 여러 명이 동시에 SMS를 보낼 수 있나요?
**A**: 예. API Key는 공용이므로 Aligo는 발신자를 구분하지 않습니다. CRM 내부에서는 `createdBy` 필드로 발신자 추적 가능.

### Q2: 발신번호를 여러 개 등록할 수 있나요?
**A**: Aligo는 여러 발신번호 등록 가능하지만, CRM의 `OrgSmsConfig.senderPhone`는 1개만 지정. 필요 시 개발팀에 문의.

### Q3: IP가 자주 바뀌면 매번 재등록해야 하나요?
**A**: 아니요. Vercel은 일정 범위의 IP를 사용하므로 CIDR 범위(예: `76.32.0.0/11`)로 등록하면 자동 대응됨.

### Q4: 공용 계정에서 개별 판매원으로 돌아갈 수 있나요?
**A**: 가능하지만, CRM 코드의 `resolveUserSmsConfig()` 함수 로직 변경 필요. 개발팀 문의.

### Q5: SMS 요금은 누가 지불하나요?
**A**: Aligo 공용 계정의 충전금에서 자동 차감. 월 예산은 마비즈가 관리.

### Q6: 발송 기록은 어디서 볼 수 있나요?
**A**: 
- **CRM 내부**: Settings → SMS Logs 또는 Messages 탭
- **Aligo 대시보드**: https://aligo.in → [통계] → [발송현황]
