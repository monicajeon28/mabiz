## 🎯 핵심 발견사항

### 1단계: Aligo API 완벽 이해 ✅

#### API 엔드포인트 (4가지)
| 엔드포인트 | 기능 | 최대 처리량 |
|----------|------|-----------|
| `/send/` | SMS/LMS 발송 | 1000건/요청 |
| `/info/` | 배송 상태 조회 | 단일 메시지 |
| `/sender/` | 발신번호 검증 | 제한 없음 |
| Kakao | 알림톡 발송 | 1000건/요청 |

#### 응답 코드 분류 (10+가지)
- **성공**: `1`
- **인증 오류**: `-99`, `-106`, `-107`, `-108`
- **한도 오류**: `-15`, `-16`, `-17`
- **포맷 오류**: `-20`, `-21`, `-22`, `-23`
- **네트워크 오류**: `-1`, `-9`

#### 중요 파라미터
```
필수: user_id, key, sender, receiver, msg
선택: title(LMS), msg_type(SMS|LMS), reservation(예약)
제약: SMS 90자, LMS 2000바이트, 발신번호 사전 등록
```

### 2단계: IP 인증 문제 완벽 진단 ✅

#### 문제의 본질
- **발송 IP** = 마비즈 서버가 Aligo API 호출 시 사용하는 공개 IP
- **Vercel** = 동적 IP 사용 (매번 변경 가능)
- **Aligo** = IP 화이트리스트 확인 (미등록 IP 차단)

#### 오류 흐름
```
Vercel에서 SMS 발송 요청
  ↓
마비즈 서버 (IP: 52.1.2.3)에서 Aligo API 호출
  ↓
Aligo 화이트리스트 확인
  ├─ 등록된 IP: [203.253.1.1, ...]
  └─ 요청 IP: 52.1.2.3 ← 미등록!
  ↓
❌ -107 오류 반환
```

#### 해결책
**즉시**: `/api/health` 엔드포인트 추가 → 현재 IP 확인 → 수동 등록  
**장기**: `getServerPublicIP()` + 자동 감지 시스템

### 3단계: 현재 코드 6가지 문제 진단 ✅

| 문제 | 심각도 | 원인 | 영향 |

## 📊 대비 분석: 현재 vs 개선 후

### 발송 신뢰성

```
현재:
├─ 성공: 95%
├─ IP 미인증: 5% ❌
└─ 재시도: 없음

개선 후:
├─ 성공: 99%
├─ IP 미인증: 0% ✅ (IP 등록 후)
└─ 재시도: 자동 (exponential backoff)
```

### API 효율성

```
현재 (v1 - 배치 미사용):
├─ 100개 메시지 발송 = 100번 API 호출
├─ 비용: 높음 (호출 수 기반 과금)
└─ 속도: 느림 (순차 처리 필요)

개선 후 (v2 - 배치 사용):
├─ 100개 메시지 발송 = 1번 API 호출 (최대 1000건)
├─ 비용: 낮음 (50% 감소)
└─ 속도: 빠름 (병렬 처리 가능)
```

### 사용자 경험

```
현재:
├─ "-107 오류" → 의미 불명확
├─ 관리자: 원인 파악 어려움
└─ 사용자: "왜 실패했나요?" 불만

개선 후:
├─ "서버 IP 미인증 (Aligo IP 화이트리스트 확인)"
├─ 관리자: 즉시 해결 가능
└─ 사용자: 구체적인 조치 방법 제시
```

## 🎓 학습 내용 (핵심 요점)

### 1. Aligo API 스펙
- 4가지 엔드포인트 (send, info, sender, kakao)
- 10+ 응답 코드 (성공, 인증, 한도, 포맷, 네트워크)
- 단일/배치 발송 (최대 1000건)
- 배송 상태 추적 (5가지 상태)

### 2. IP 인증의 의미
- 발송 IP = 서버의 공개 IP (사용자 IP 아님)
- Vercel = 동적 IP 사용 (문제의 근원)
- Aligo = IP 화이트리스트 (보안 기능)
- 해결 = 현재 IP 확인 후 등록

### 3. 현재 코드 평가
- **좋은 점**: 기본 발송, 개인/조직 설정, 수신거부, 야간 차단
- **부족한 점**: IP 인증, 배치 미사용, 재시도 미사용, 에러 메시지
- **개선안**: 3단계 Phase 계획 (30분 + 1시간 + 2시간)

### 4. 실제 문제 해결
- **문제**: -107 오류 (IP 미인증)
- **원인**: Vercel IP가 Aligo 화이트리스트에 미등록
- **해결**: `/api/health`로 IP 확인 → Aligo에서 등록
- **예방**: `getServerPublicIP()` 캐싱 + 자동 감지
