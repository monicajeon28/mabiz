#### P0-4: 테스트 — 초심자 검증 절차 부재
**문제:** 스크립트가 이론적이고 실전 검증 없음
```

- 침묵 3초/2초가 실제로 적절한지 미검증
- 초심자가 실제로 이해하는지 체크 없음
- 각 PHASE별 성공률 기록 없음

위험:
- 실제로는 18-25분 걸릴 수 있음
- 초심자가 복잡해하면 콜 중단 가능성
- 침묵 타이밍 부적절하면 어색함
```

**해결책:**
```
1단계: 롤플레이 시뮬레이션 (5명 초심자)
  - 선택: 렌탈 경험 없는 일반인 5명
  - 진행: 수정 없이 그대로 스크립트 실행
  - 측정:
    * 소요 시간: 15분 ±2분 확인
    * 침묵 타이밍: 고객 반응 실제 기록
    * 이의처리: 예상과 다른 우려사항
    * 중단 지점: 고객이 콜을 중단하려는 순간

2단계: A/B 테스트 (실제 고객 10명)
  - 그룹 A (5명): 현재 스크립트 (침묵 3초 고정)
  - 그룹 B (5명): 개선 스크립트 (침묵 5초 탄력적)
  - 측정:
    * 콜 완료율
    * 신청율 (클로징 성공)
    * 고객 만족도
    * 진행 시간

3단계: 이해도 체크리스트
  Q1: "렌탈이랑 구매의 차이 이해되셨나요?" → Yes/No
  Q2: "월 정액이 어떻게 나가는지 명확한가요?" → Yes/No
  Q3: "위약금이 없다는 거 확실한가요?" → Yes/No
  Q4: "고장 시 처리 과정이 이해되셨나요?" → Yes/No
  
  기대: 모두 Yes → 85%+ 달성

V2 적용 상태: ✅ 타이밍 시트 추가 (실전 테스트 대기)

#### P2-1: 에러처리 — 콜 중단 시나리오
```
V1 문제:
- 모든 시나리오가 긍정 경로만 가정
- "복잡해요", "정확히 얼마인지 불명확" 대응 없음
- 고객이 갑자기 거절해도 대응 부족

V2 해결 (4가지 콜 중단 시나리오):

상황 1: 설명 후 고객이 "복잡해요"
응답: "미안해요. 더 간단하게 설명할게요.
렌탈과 구매, 딱 한 가지만 다릅니다:
- 구매: 처음에 많은 돈 / 고장 시 비용
- 렌탈: 매달 조금씩 / 고장 시 무료"

상황 2: 계산 후 고객이 "정확히 얼마인지 불명확"
응답: "명확하게 해드릴게요. 예를 들어 TV:
- 구매: 150만 원 초기 + 수리비 30-50만 = 약 200만
