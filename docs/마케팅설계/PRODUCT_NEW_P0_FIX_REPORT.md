```json
{
  "product": {
    "name": "[신제품명 - 실제 출시 시 변경]",
    "description": "[제품 설명 한 문장]",
    "launchDate": "2026-05-14"
  }
}
```

### 체크리스트 강화
```
출시 전 필수 확인:
- [ ] **신제품명** 최종 확정 ([신제품명] → 실제명 변경)
- [ ] **가격** 크루즈닷 내부 승인 (30,000원)
- [ ] **할인율** 최종 확정 (30%)
- [ ] **할인 기한** 최종 확정 (6개월)
```

**사용 방법**:
```
Step 1: 설정값 JSON에서 product.name 변경
{
  "name": "크루즈AI(실제 제품명)" ← 이 부분 변경
}

Step 2: 스크립트는 그대로 [신제품명] 사용
"[신제품명] 들어보셨어요?" ← 자동으로 실제명으로 처리됨

Step 3: 상담사 대면 훈련에서 실제명으로 대사 연습
```

## 👥 P0-4: 기존 고객 공평성 확보

**해결 방법**:

### 언어 재포지셔닝
```
변경 전: "첫 출시 가격", "정가의 30% 할인"
변경 후: "선발 고객 혜택", "6개월 한정 특가"

의미:
- 차별이 아닌 "감사"로 재해석
- 신제품이라 초반에 구매하는 분들을 배려
```

### Phase 2 [2-3] 강화
```
상담사: "이 할인가는 **6개월 동안만** 유효해요.

왜냐하면,
시간이 지나면서 더 많은 고객이 알게 되고
가격이 정상가로 올라가기 때문이에요.

쉽게 말해서,
지금 3만 원이면,
**6개월 후에는 4만 원대**가 돼요.

**지금이 최고의 기회**라는 뜻이에요."
```

**심리학 원리**: 
- 차별→감사 (감정 전환)
- FOMO (지금만의 기회)
- 공평성 강화 (모든 신제품에 동일 적용)

## 🎯 P0-7: 클로징율 현실화

**문제**: 25% → 45% 목표 (비현실적)
- 신제품은 신뢰도 낮음
- 달성 불가 시 팀 사기 저하
- 지속 가능한 목표 필요

**해결 방법**:

### 목표 수정
```
변경 전: "25% → 45% (+80% 개선)"
변경 후: "25% → 35% (현실적 설정)"

근거:
- 신제품 = 신뢰도 낮음
- 초기 출시 기준: 20~30% 현실적
- 4~6개월 운영 후 40% 목표 설정 추천
```

### 스크립트 헤더 수정
```
변경 전: **클로징율 목표**: 25% → 45%
변경 후: **클로징율 목표**: 25% → 35% (현실적 설정)
```

### 체크리스트 수정
```
- [ ] **추천 클로징율** 35% 설정 (기존 45% → 현실적 목표)
```

### 주의사항 P0-7 추가
```
P0-7: 비현실적 클로징율 수정 (목표)
  - ❌ "25% → 45% 목표" (신제품이라 불가능)
  - ✅ "25% → 35% 현실적 목표" (초기 신제품 기준)
```

**근거**:
- 신제품의 초기 신뢰도: 낮음
- 기존 제품 개선 vs 신규 제품: 클로징율 차이 10~15%
- SMS + 환불 보장으로 예상 35~40% 달성 가능
