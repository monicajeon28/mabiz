## 🔧 각 P0별 주요 변경 사항

### P0-1: 기밀정보 제거 ⛔
```
금지:
  ❌ "LG와 협력"
  ❌ "100,000명 사용"
  ❌ "4.8/5 평점"

필수 표현:
  ✅ "크루즈닷 신제품팀"
  ✅ "사전 테스트 고객 만족도 92%"
  ✅ "출시 후 30일 데이터"
```

### P0-2: 시간 단축 🏃
```
변경 전:
  Phase 4: "화면 보면서 따라가세요" (5분+)
  합계: 15분 ❌

변경 후:
  Phase 4: "SMS 링크를 누르세요" (10초)
        "3단계만 거치면 됩니다" (3분 동안 진행)
  합계: 12분 ✅

효과: 20% 시간 단축
```

### P0-3: 신제품명 명시 📋
```
설정값 JSON:
{
  "product": {
    "name": "[신제품명 - 실제 출시 시 변경]"
  }
}

스크립트: "[신제품명]" 그대로 사용
          ↓
CRM: 자동으로 실제명으로 치환 (구현 필요)

체크리스트: ☐ 신제품명 최종 확정 (필수 확인)
```

### P0-4: 공평성 확보 💬
```
변경 전: "첫 출시 가격 30%", "나중에 4만 원"
변경 후: "선발 고객 혜택 6개월", "초반 구매 배려"

의미: 차별 아닌 "감사" 로 재해석

Phase 2에서 명시:
"저희가 심사숙고해서 출시한 신제품이라서,
초반에 구매하는 분들을 배려하는 거죠"
```

### P0-5: 정보 정확성 ✔️
```
제거: ❌
  - 100,000명 이상
  - 4.8/5 평점
  - 85% 재구매율
  - 시간 절약 2시간
  - 비용 절약 2만 원

추가: ✅
  - 사전 테스트 고객 만족도 92% (검증)
  - "많은 고객이 좋다고 말씀해주세요" (추정)
  - "한 달 후 다들 좋다고 했어요" (사례)
```

### P0-6: 신뢰 강화 🔒
```
기밀 처리 (P0-1과 연계):
  "기밀이라 자세히 말씀드릴 수 없지만"

신뢰 신호:
  ✅ "환불 100% 보장"
  ✅ "30일 안에 환불 가능"
  ✅ "아무 문제 없어요"

효과: 기밀→환불보장로 신뢰 전환
```

### P0-7: 목표 현실화 📊
```
변경 전: 25% → 45% (비현실적, 신제품)
변경 후: 25% → 35% (현실적, 초기 신제품)

근거:
  - 신제품 신뢰도: 낮음
  - 초기 기준 클로징율: 20~30%
  - SMS + 환불보장: +5~10%
  - 합계: 35% 현실적

4~6개월 후: 40% 목표 설정 추천
```

## 📞 긴급 Q&A

**Q: 신제품명이 아직 정해지지 않으면?**
A: "[신제품명]" 그대로 스크립트 진행 + JSON 설정값에 실제명 입력 후 자동 치환

**Q: SMS 링크 기술이 아직 구현 안 되면?**
A: 현재는 불가능. 기술팀 즉시 개발 필요 (P0-2 사항)

**Q: 콜이 15분 넘으면?**
A: Phase 2-3 시간 단축 필요 + SMS 링크 발송 확인 (중복 불가)

**Q: 고객이 "LG 협력" 물으면?**
A: "기밀이라 자세히 말씀드릴 수 없지만, 저희 환불 100% 보장이 그 신뢰의 증거예요"

**Q: 92% 만족도를 증명해달라 하면?**
A: "사전 테스트 고객 기준 데이터이고, 출시 후 30일 데이터입니다. 구체적 내용은 기밀입니다"

**Q: 클로징율이 35% 미만이면?**
A: 스크립트 재교육 + SMS 링크 전환율 분석 + 환불율 확인
