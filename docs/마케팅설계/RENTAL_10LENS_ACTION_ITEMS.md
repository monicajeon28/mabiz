#### P2-1: 콜 중단/고객 혼란 시 대응 시나리오 부족 [에러처리]
**상태:** 🟡 MEDIUM  
**영향:** 중단 고객 손실율 증가  
**담당자:** [마케팅팀]  
**일정:** 1일

**현황:**
- 모든 시나리오가 긍정적 진행만 가정
- "복잡해요", "정확히 얼마인지 불명확" 시나리오 없음
- "근데 저는 필요 없어요" 즉시 거절 시나리오 부족
- 콜 중단 후 재시도 절차 없음

**액션:**
```
4가지 콜 중단 시나리오 추가:

상황 1: 설명 후 고객이 "복잡해요"
응답: "미안해요. 더 간단하게 설명할게요.
렌탈과 구매, 딱 한 가지만 다릅니다:
- 구매: 처음에 많은 돈 / 고장 시 비용
- 렌탈: 매달 조금씩 / 고장 시 무료"

상황 2: 계산 후 고객이 "정확히 얼마인지 불명확"
응답: "명확하게 해드릴게요. 예를 들어 TV:
- 구매: 150만 원 초기 + 수리비 30-50만 = 약 200만
- 렌탈: 월 4.5만 × 60 = 270만
근데 렌탈은 2년 후 더 좋은 TV로 바꿀 수 있어요"

상황 3: 갑자기 "저는 안 필요할 것 같아요"
응답: "혹시 가장 큰 불안은 뭔가요?
- 가격 부담?
- 필요 없는 상황?
- 우리 신뢰?
어느 부분이 가장 중요한지 말씀해주면 해결해드릴 수 있어요"

상황 4: "다시 전화 줄게요" (콜 중단)
응답: "언제쯤 생각 정리되실 것 같으세요?
내일? 3-4일? 일주일?
저는 [날짜 오후 O시]에 다시 전화 드릴게요"
→ CRM 기록: follow_up_date 설정
```

**완료 기준:**
- ✅ 4가지 콜 중단 시나리오 추가 (V2에 포함)
- ✅ 각 상황별 재시도 절차 문서화
- ✅ CRM 자동 재콜 일정 설정 방법
- ✅ 상담사 훈련 자료 포함

#### P2-2: SMS Day 0-2 시퀀스 연동 절차 불명확 [호환성]
**상태:** 🟡 MEDIUM  
**영향:** 후속 자동화 불가 + 수동 작업 증가  
**담당자:** [개발팀]  
**일정:** 1일

**현황:**
- "카톡으로 정보 드리겠습니다" (언제? 어떤 내용?)
- 배송 후 후속 SMS 절차 없음
- 고객 피드백 수집 절차 미명시

**액션:**
```
Day 0-3 SMS 시퀀스 상세화:

Day 0 신청 완료 (즉시)
SMS: "[크루즈닷] 렌탈 신청이 완료되었습니다! 
내일 배송됩니다. 확인 링크: [링크]"
CRM: rental_status = 'pending' → 'pending_delivery'

Day 1 배송 전 (오전 8시)
SMS: "[크루즈닷] 오늘 오전 10시-12시 배송 예정입니다. 
배송 조회: [링크]"
CRM: rental_status = 'pending_delivery' → 'in_delivery'

Day 2 배송 후 (오후 6시)
SMS: "[크루즈닷] 제품 잘 받으셨나요? 
불편한 점은? [1클릭 피드백]"
CRM: rental_status = 'in_delivery' → 'active'
      rental_satisfaction 입력 (1-5)

Day 3 Follow-up (오전 10시)
SMS: "[크루즈닷] 사용 중 궁금한 점 있으세요? 
고객센터: 1234-5678"
CRM: call_follow_up = true

Day 7 만족도 (오후 3시)
SMS: "[크루즈닷] 렌탈 서비스 만족도를 평가해주세요. 
[별점 평가 링크] 참여 시 추첨!"
CRM: rental_satisfaction 최종 기록

Day 30 업그레이드 (오전 9시)
SMS: "[크루즈닷] 더 좋은 제품이 나왔어요! 
업그레이드는? [조회 링크]"
CRM: upgrade_offer_sent = true
```

**완료 기준:**
- ✅ SMS 6가지 템플릿 작성
- ✅ 자동화 절차 개발팀에 전달
- ✅ CRM 상태 전이 다이어그램
- ✅ V2 부록 3에 포함
