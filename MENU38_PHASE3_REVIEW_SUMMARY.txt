P0 이슈 (배포 차단) - 7개

P0-1: Cron 엔드포인트 파일 미발견
  - 현상: 크론잡 함수는 구현되어 있지만, API 엔드포인트 없음
  - 영향: 자동 검증 실행 불가 (배포해도 작동 안 함)
  - 해결: /api/cron/verify-execution-log/route.ts 파일 생성 (30분)

P0-2: 행 수 일관성 검증 - 단방향만 확인
  - 현상: consistency = executionLog / sendingHistory (한쪽만)
  - 문제: SendingHistory가 많으면 오류 감지 불가
  - 해결: 양방향 최소값 사용 (10분)

P0-3: 타임스탐프 P99 계산 - 샘플 크기 부족
  - 현상: 샘플 < 100개일 때 P99 계산 (통계적 무의미)
  - 해결: 최소 샘플 크기 100개 검증 (15분)

P0-4: 롤백 후 무한 루프 위험
  - 현상: 롤백 후 재검증 시 무한 롤백 가능
  - 해결: 롤백 상태 감지 후 검증 로직 조정 (30분)

P0-5: Redis 오류 시 기본값 = true (위험!)
  - 현상: Redis 다운 → 기본값 true → ExecutionLog 계속 사용 (❌)
  - 해결: 기본값을 false로 변경 (안전 모드) (5분)

P0-6: N+1 쿼리 (타임스탐프 검증)
  - 현상: 1000번 루프에서 각각 DB 쿼리 (1001개 쿼리!)
  - 영향: 검증 시간 5-10초 (목표 < 10초)
  - 해결: 조인 쿼리로 개선 (1시간)

P0-7: 토큰 검증 미실행
  - 현상: "Bearer" 형식만 확인, 실제 검증 X
  - 위험: "Bearer anything"이면 통과 가능 (보안 취약)
  - 해결: verifyAdminToken() 호출 추가 (10분)
