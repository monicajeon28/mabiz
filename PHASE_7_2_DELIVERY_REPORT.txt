2. ThompsonSamplingBandit (300 lines)
   File: src/lib/ai/multi-armed-bandit.ts

   ✅ selectArm() - Selects optimal channel using Thompson Sampling
   ✅ updateReward(channel, success) - Updates Beta distribution with feedback
   ✅ getConfidence() - Returns confidence score (0-100%)
   ✅ getBestArm() - Returns highest performing channel

   Algorithm: Bayesian Beta Distribution
   - Each channel modeled as Beta(successes+α, failures+β)
   - 80% exploitation (use best), 20% exploration (try new)
   - Automatic convergence in 1-2 weeks
   - Per-contact learning capability

KEY FEATURES & ALGORITHMS
================================================================================

FEATURE 1: Real-Time ROI Optimization (Every 30 minutes)
────────────────────────────────────────────────────────
Transforms static allocation into dynamic, data-driven decisions:

Before:  SMS 40% / Kakao 35% / Email 25% (fixed)
After:   SMS 45% / Kakao 38% / Email 17% (dynamic, every 30 min)

Basis: Last 30 minutes of actual performance data
Output: Confidence-weighted allocation changes
Constraints: 10% min, 60% max per channel

FEATURE 2: Thompson Sampling (Bayesian Multi-Armed Bandit)
──────────────────────────────────────────────────────────
Automated A/B testing with Bayesian convergence:

Traditional A/B: Test A vs B at 50/50 → Wastes 50% budget on loser
Thompson:        Data-driven allocation → Minimizes waste, mathematically optimal

Process:
1. Model each channel as Beta(successes, failures)
2. Each decision: Sample from all Beta distributions
3. Select arm with highest sample (exploitation)
4. 20% random selection for exploration (avoid local optimum)
5. Feedback updates Beta distribution
6. Automatic convergence to optimal in 1-2 weeks

FEATURE 3: Auto Budget Allocation (Weekly)
───────────────────────────────────────────
Month $10,000 budget intelligently split:

Calculation:
1. Calculate ROI score for each channel (last 3 months)
2. Weight allocation by ROI
3. Reserve 5% for A/B testing
4. Distribute remaining 95%
5. Apply constraints (min 10%, max 60%)

Result: SMS $3,500 / Kakao $3,500 / Email $2,500 / A/B $500

FEATURE 4: Optimal Send Time Learning
──────────────────────────────────────
Learns when each contact opens each channel:

Data Source: 6 months of historical sending + opening data
Dimensions:
  - Per-contact patterns ("John opens SMS at 9 AM")
  - Per-segment aggregates ("20-30 year olds peak at noon")
  - Channel-specific ("Email opens at 8 AM")
  - Day-of-week ("Tuesday opens are 10% higher")

Output: "Send to John on SMS at 9:00 AM (32% open rate, 85% confidence)"

FEATURE 5: Predictive Offer Optimization
────────────────────────────────────────
Recommends best offer per contact:

Factors:
- L1 Lens: Price sensitivity (high → big discount)
- LTV: Customer lifetime value (VIP → special offers)
- Purchase frequency (frequent → loyalty rewards)
- Message type (promotional allows bigger discounts)

Output: "Recommend 15% discount to Jane (82% acceptance probability)"

With A/B testing: "Test 3 offers this week, expand winner next week"
