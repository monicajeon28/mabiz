### 3. API Endpoints (586 lines) ✅

#### a) POST /api/contacts/detect-lens (123 lines)
**Purpose**: Execute lens detection for a contact

**Request**:
```json
{
  "contactId": "clxxxxx",
  "organizationId": "org_xxxxx",
  "force": false
}
```

**Response** (200 OK):
```json
{
  "success": true,
  "data": {
    "lens": {
      "primaryLens": "L0",
      "confidenceScore": 62,
      "allScores": {...},
      "detectedSignals": {...}
    },
    "classification": {
      "lensType": "L0",
      "lensLabel": "부재중 재활성화",
      "confidenceScore": 62
    }
  }
}
```

**Features**:
- OAuth authentication
- Organization validation
- Redis cache integration
- Automatic DB persistence

---

#### b) GET /api/lens/templates (279 lines)
**Purpose**: Retrieve message templates by lens & day

**Query Params**:
- organizationId (required)
- lensType (optional: L0-L10)
- templateType (optional: sms/email/call_script)
- day (optional: 0-3)

**System Templates Included**:
- L0 (Reactivation): Day 0-3 messaging
- L1 (Price): Value redefinition sequence
- L6 (Timing): Loss aversion + urgency
- L10 (Closing): Immediate purchase CTA

---
