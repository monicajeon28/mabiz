================================================================================
ENDPOINT SPECIFICATION
================================================================================

REQUEST:
  GET /api/landing/live-stats/:id[?include_trend=true]

  Path param:
    id (string, required): Landing page CUID (e.g., c1234567890123456789abc)

  Query param:
    include_trend (boolean, optional): Include 7-day trend data (default: false)

RESPONSE (200 OK):
  {
    "totalCustomers": 1250,
    "totalRevenue": 125000000,
    "activeBookings": 42,
    "lastUpdated": "2026-05-30T14:30:25.123Z",
    "trendInfo": {
      "revenueGrowth": 15.8,      // percentage
      "customerGrowth": 23,        // count
      "bookingTrend": "accelerating"  // 'accelerating' | 'stable' | 'declining'
    }
  }

RESPONSE HEADERS (200):
  Cache-Control: no-cache, no-store, must-revalidate, max-age=0
  Pragma: no-cache
  Expires: 0
  X-RateLimit-Limit: 60
  X-RateLimit-Remaining: 59
  X-RateLimit-Reset: 2026-05-30T15:30:25.123Z

ERROR RESPONSES:
  400 Bad Request: Invalid CUID format
  404 Not Found: Landing page not found
  429 Too Many Requests: Rate limit exceeded (with Retry-After header)
  500 Internal Server Error: Database error

================================================================================
DATABASE QUERIES
================================================================================

1. Landing page lookup (O(1) — indexed by CUID primary key)
   db.landingPage.findUnique({ where: { id } })

2. Contact stats aggregation (O(n) — needs index: contact(landingPageId, createdAt))
   db.contact.aggregate({ where: { landingPageId: id }, _sum: { totalPaid }, _count })

3. Active bookings count (O(n) — needs index: booking(landingPageId, status))
   db.booking.count({ where: { landingPageId, status: in ['confirmed', 'pending', 'in_progress'] } })

4. Optional trend data (7-day comparison, only if include_trend=true)
   db.contact.aggregate({ where: { landingPageId, createdAt: { gte: sevenDaysAgo } } })

Required Prisma relations:
  - LandingPage.contacts: Contact[] (one-to-many)
  - LandingPage.bookings: Booking[] (one-to-many)
  - Contact.landingPageId: String (foreign key)
  - Booking.landingPageId: String (foreign key)
