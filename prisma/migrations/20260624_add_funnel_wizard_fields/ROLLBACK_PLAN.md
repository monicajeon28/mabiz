## Troubleshooting

### If migration fails:

1. **Check PostgreSQL logs:**
   ```bash
   # Check for constraint violations
   psql -U $DATABASE_USER -d $DATABASE_NAME -c "SELECT * FROM pg_stat_user_tables WHERE relname IN ('Funnel', 'FunnelSmsMessage', 'FunnelEmailMessage');"
   ```

2. **Manually verify column existence:**
   ```bash
   psql -U $DATABASE_USER -d $DATABASE_NAME -c "\\d \"Funnel\""
   ```

3. **If stuck, force reset (⚠️ Production only with backup):**
   ```bash
   # Backup database first!
   pg_dump $DATABASE_NAME > backup_$(date +%s).sql
   
   # Then reset Prisma state
   npx prisma migrate reset --force
   ```
