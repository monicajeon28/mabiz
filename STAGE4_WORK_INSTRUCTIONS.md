## 🔧 기술 구현 필수 사항

### SMS Cron 구조
```typescript
// Day 0 (30분 후)
POST /api/cron/sms-day0-init
- Contact에서 lastCruiseEndDate ±24시간 조건
- PASONA P + A 단계 메시지 발송
- SMS 발송 로그 기록
- Day 1 자동 스케줄링

// Day 1-3
POST /api/cron/sms-day{1-3}-{stage}
- 응답율 추적 (SMS 클릭, 콜 수신 등)
- 이의 감지 시 자동 대응 메시지
- Risk Flag 실시간 업데이트

// Day 7/14/30/60/90
POST /api/cron/sms-followup
- Grant Cardone 7회 접촉 규칙
- 누적 응답율에 따라 강도 조절
- 최종 클로징 메시지
```

### KPI 실시간 추적 구조
```typescript
// 일일 KPI 계산
GET /api/analytics/realtime/kpi
- 전환율 (L0-L10 별, 세그먼트별)
- CPA/LTV 계산
- SMS 응답율
- 예상 월간 수익

// 세그먼트 분석
GET /api/analytics/realtime/segment
- 호텔 경험도별 성과
- 나이/성별별 성과
- 렌즈별 성과 분석
- A/B테스트 결과
```
