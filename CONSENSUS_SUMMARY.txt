🎯 핵심 변경사항
─────────────────────────────────────────────────────────────────────────────

1️⃣ Schema 변경
   ├─ CommissionLedger.organizationId: String (NOT NULL)
   ├─ FK: Organization(id) ON DELETE CASCADE
   ├─ @unique([saleId, organizationId]) → Race Condition 100% 차단
   └─ Index: [organizationId, isSettled, createdAt] → 정산 쿼리 97% 빠름

2️⃣ API 필터링 (모든 정산 API)
   ├─ GET organizationId = resolveOrgId(req)
   ├─ WHERE organizationId = orgId (필수)
   └─ 403 Forbidden for cross-org access

3️⃣ Webhook 멱등성
   ├─ eventId 기반 deduplication
   ├─ Transaction Isolation: SERIALIZABLE
   └─ Smart Retry + DLQ (비정상 처리 자동 격리)

4️⃣ 3계층 권한 검증
   ├─ JWT: 사용자 토큰에 organizationId 포함
   ├─ App: resolveOrgId() + enforceOrgIdMatch() 필수 호출
   └─ DB: PostgreSQL RLS 정책 (SELECT/INSERT 필터링)

5️⃣ 감시 & 감사
   ├─ Audit Log: 모든 접근 기록 (성공/실패/이유)
   ├─ Security Alert: 권한 위반 시 즉시 알림
   └─ Dashboard: 관리자가 audit log 조회 가능
