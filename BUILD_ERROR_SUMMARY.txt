================================================================================
🔴 P0: 빌드 차단 에러 (즉시 수정)
================================================================================

1️⃣ JSON Import 경로 오류
   위치: 3개 파일
     - src/lib/objections/validation.ts:1
     - src/app/(dashboard)/contacts/[id]/ContactCallTab.tsx
     - src/app/(dashboard)/contacts/[id]/page.tsx

   문제: import objectionsData from '@/../../TRACK_A_OBJECTIONS.json';
   원인: @/../../ 경로가 프로젝트 루트 밖으로 나감
   해결: @/../../ → @/../로 변경 (1자 수정!)

   영향: Contacts 페이지 전체 로드 불가
   빌드: 완전 실패

2️⃣ Prisma 클라이언트 생성 실패
   원인: npm install 미완료 → @prisma/client 없음
   해결: npm install && npx prisma generate

   영향: 데이터베이스 쿼리 불가
   빌드: 첫 단계에서 실패

================================================================================
🟠 P1: 배포 위험 (권장 수정)
================================================================================

1️⃣ 라우트 그룹 간 상대 Import
   위치: src/app/(dashboard)/b2b-editor/[id]/page.tsx
   문제: ../../landing-pages/[id]/components/ 상대 경로 import
   원인: 라우트 그룹 사이의 상대 경로는 환경에 따라 달라질 수 있음
   해결: @/app/(dashboard)/landing-pages/[id]/components/ 절대 경로 사용

   영향: Vercel 배포 환경에서 경로 해석 불일치 가능
   배포: 가능하지만 위험

================================================================================
🟡 P2: 타입 경고 (배포 후 수정 가능)
================================================================================

1️⃣ 느슨한 'any' 타입 사용
   위치: src/lib/objections/validation.ts:24, 32, 41
   문제: (o: any) 대신 구체적인 타입 사용 필요
   해결: ObjectionItem 인터페이스 정의 후 적용

   영향: 타입 안전성 하락, 런타임 에러 위험
   배포: 가능하지만 코드 품질 저하

================================================================================
⚡ 즉시 해결책 (5분)
================================================================================

Step 1: 3개 파일의 import 경로 변경
  sed -i "s/@\/..\/..\//@\/..\//" src/lib/objections/validation.ts
  sed -i "s/@\/..\/..\//@\/..\//" src/app/\(dashboard\)/contacts/\[id\]/ContactCallTab.tsx
  sed -i "s/@\/..\/..\//@\/..\//" src/app/\(dashboard\)/contacts/\[id\]/page.tsx

Step 2: 의존성 재설치
  rm -rf node_modules package-lock.json
  npm install

Step 3: Prisma 재생성
  npx prisma generate

Step 4: 빌드 테스트
  npm run build

Step 5: 커밋 및 배포
  git add -A
  git commit -m "fix: JSON import 경로 수정 (@/../../ → @/../)"
  git push origin main

📞 추가 질문
================================================================================

Q: 왜 로컬에서는 작동했나?
A: next dev는 모듈 로딩이 관대하지만, next build는 엄격합니다.

Q: @/../가 맞나요?
A: 맞습니다. @/ = src/, @/../ = src/../ = 프로젝트 루트

Q: 파일을 src/ 안으로 옮기면?
A: 더 나은 해결책입니다. src/data/TRACK_A_OBJECTIONS.json 권장

Q: 다른 에러도 있나?
A: P0 2개 고정 후 P1, P2 순으로 처리하면 됩니다.
