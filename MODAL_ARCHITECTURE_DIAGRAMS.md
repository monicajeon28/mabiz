### 개선 (상태 기반)

```
<Root>
  <NavBar />
  <ContactsContextProvider> {/* 중앙 상태 관리 */}
    <ContactsLayout>
      {/* Route 하나로 통일 */}
      <Route path="/contacts">
        <ContactsListPage>
          <ContactListTable>
            <ContactRow
              onClick={() => openModal(id)} {/* 상태 업데이트만 */}
            >
              {contactName}
            </ContactRow>
          </ContactListTable>
        </ContactsListPage>

        {/* 모달: 리스트 위에 오버레이 */}
        <ContactDetailModalWrapper
          contactId={selectedContactId} {/* Context에서 주입 */}
          isOpen={isModalOpen}
          onClose={closeModal}
        >
          <ContactDetailModal>
            <ContactInfoPanel />
            <Tabs>
              <Suspense fallback={<Loader />}>
                <Tab id="call">
                  <ContactCallTab /> {/* 필요할 때만 로드 */}
                </Tab>
                <Tab id="memo">
                  <ContactMemoTab />
                </Tab>
                ...
              </Suspense>
            </Tabs>
          </ContactDetailModal>
        </ContactDetailModalWrapper>
      </Route>
    </ContactsLayout>
  </ContactsContextProvider>
</Root>
```

**장점**: 
- 리스트 컴포넌트 유지 (상태 유지, 스크롤 위치)
- 모달만 오버레이 (레이아웃 변경 없음)
- Lazy + Suspense로 번들 최소화

## ⏱️ 시간 라인 (Timeline)

### 현재: 페이지 이동 (1.5-2.0초)

```
0ms
│
├─ 100ms   ┌─ 라우트 변경 (useRouter.push)
│          │
├─ 300ms   │ 페이지 언로드 + useEffect cleanup
│          │
├─ 700ms   │ 번들 파싱 + 평가
│          │
├─ 900ms   │ React 하이드레이션
│          │
├─ 1000ms  ├─ API 호출 시작
│          │ GET /api/contacts/[id]
│          │ ├─ Contact 조회
│          │ ├─ callLogs (30건)
│          │ ├─ memos (30건)
│          │ └─ Transfer 링크 조회
│          │
├─ 1400ms  │ API 응답 도착
│          │
├─ 1550ms  ├─ 컴포넌트 렌더 (6개 탭 마운트)
│          │
└─ 1850ms  └─ TTI (첫 상호작용 가능)
           ↑
        사용자가 느끼는 지점
```

---

### 개선: 모달 팝업 (0.3-0.6초)

```
0ms
│
├─ 5ms     ┌─ 상태 업데이트 (setSelectedContactId)
│          │
├─ 55ms    │ 모달 렌더 시작 + 애니메이션
│          │
├─ 100ms   │ 배경 오버레이 표시 (완료)
│          │
├─ 255ms   ├─ 애니메이션 완료 (모달 시각적 완성)
│          │
├─ 255ms   ├─ API 호출 시작 (동시 진행)
│          │ GET /api/contacts/[id]
│          │
├─ 555ms   │ API 응답 도착 + 콘텐츠 렌더
│          │
└─ 600ms   └─ TTI (모달 상호작용 가능)
           ↑
        사용자가 느끼는 지점
```

**개선**: 애니메이션으로 시각적 피드백 제공 → 기다리는 느낌 감소

## 🎨 상태 다이어그램 (State Machine)

### ContactsContext 상태 관리

```
┌─────────────────────────────────────────────────────────────────┐
│                     초기 상태                                    │
│   selectedContactId: null                                       │
│   isModalOpen: false                                            │
│   contactCache: {}                                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
                    고객 이름 클릭
                              ↓
        openModal(contactId) 호출
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                     모달 오픈 상태                               │
│   selectedContactId: "contact_affiliate_102"                   │
│   isModalOpen: true                                             │
│   contactCache: { ... }                                         │
│                                                                  │
│   [useEffect] API 호출 시작                                     │
│   GET /api/contacts/contact_affiliate_102                      │
└─────────────────────────────────────────────────────────────────┘
                    ↙              ↘
              캐시 히트          캐시 미스
                ↓                   ↓
        API 스킵            API 응답 대기
        즉시 렌더          200-500ms 대기
                ↘              ↙
                   모달 렌더링
                      ↓
┌─────────────────────────────────────────────────────────────────┐
│                 모달 콘텐츠 표시 상태                            │
│   contactData: { id, name, phone, ... }                         │
│   activeTab: "call"                                             │
│   contactCache[id] = contactData                                │
└─────────────────────────────────────────────────────────────────┘
                    ↙              ↘
              탭 변경            닫기 클릭
                ↓                   ↓
        Suspense로         closeModal()
        다음 탭 로드        호출
                ↘              ↙
                   모달 닫기
                      ↓
┌─────────────────────────────────────────────────────────────────┐
│                     모달 닫기 상태                               │
│   selectedContactId: null                                       │
│   isModalOpen: false                                            │
│   contactCache: { ... } (유지)                                  │
│   [Framer Motion] exit 애니메이션 실행                          │
└─────────────────────────────────────────────────────────────────┘
                              ↓
                    모달 언마운트 (완료)
                              ↓
                    고객 목록 표시 (스크롤 위치 유지)
```

## 🔌 API 호출 비교

### 현재 (페이지 이동) - 매 진입마다 호출

```
T0: 고객 이름 클릭
  ↓
T100: 라우트 변경 시작
  ↓
T900: useEffect 트리거
  ↓
T1000: GET /api/contacts/contact_affiliate_102
  ├─ SELECT Contact WHERE id = ...
  ├─ SELECT CallLog WHERE contactId = ... LIMIT 30
  ├─ SELECT Memo WHERE contactId = ... LIMIT 30
  ├─ SELECT ContactTransferLog WHERE ...
  ├─ SELECT Contact (linked IDs)
  └─ SELECT GlobalAdmin/OrganizationMember (batch)
  ↓
T1400: API 응답 도착
  ↓
T1550: 화면 렌더링
  ↓
T1850: TTI (상호작용 가능)

[SQL 쿼리 실행 횟수: 8회]
[API 응답 시간: 400ms]
```

### 개선 (모달) - 첫 진입은 동일, 재진입은 캐시

#### 첫 진입 (캐시 미스)
```
T0: 고객 이름 클릭
  ↓
T5: 상태 업데이트 + 모달 렌더
  ↓
T55: 모달 애니메이션 시작
  ↓
T100: GET /api/contacts/contact_affiliate_102 (동시 진행)
  [SQL 쿼리: 8회]
  ↓
T500: API 응답 도착
  ↓
T550: 모달 콘텐츠 렌더링 + 캐시 저장
  ↓
T600: TTI (상호작용 가능)

[SQL 쿼리 실행 횟수: 8회]
[API 응답 시간: 400ms]
하지만 애니메이션으로 기다리는 시간 감소
```

#### 재진입 (캐시 히트)
```
T0: 같은 고객 이름 클릭
  ↓
T5: 상태 업데이트 + 모달 렌더
  ↓
T50: 캐시 확인
  ↓
T50: 캐시 히트! 즉시 렌더링
  ↓
T100: TTI (상호작용 가능)

[SQL 쿼리 실행 횟수: 0회]
[API 응답 시간: 0ms]
[개선율: 550ms → 100ms (-82%)]
```
