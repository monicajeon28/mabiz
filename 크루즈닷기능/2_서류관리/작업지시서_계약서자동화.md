# 작업지시서: 계약서 자동화 시스템

**작성일**: 2026-05-13  
**우선순위**: P0  
**대상 계약서**: ① 크루즈닷 파트너스 ② 직속마케터/직속인솔스탭 ③ 대리점장

---

## 1. 목표

계약서 제출 완료 시 자동으로:
1. **계약서 보관** — DB 저장 + Google Drive 업로드 (HTML/webp)
2. **이메일 발송** — 계약자에게 계약서 사본 링크 전달
3. **링크 확인** — `/affiliate/contracts/[id]` 로 언제든 열람 가능

---

## 2. 아키텍처 결정

### 왜 Puppeteer/서버사이드 PDF가 아닌가?
- Vercel Serverless = 50MB 함수 크기 제한 → Puppeteer(>150MB) 불가
- `html2canvas` + `jsPDF` = **클라이언트 렌더링** → Vercel 부담 없음

### 선택: 클라이언트 → 서버 흐름
```
[브라우저] 계약서 HTML → html2canvas → webp blob
        → POST /api/affiliate/contracts/[id]/snapshot
        → Google Drive 업로드 → 공유 링크 생성
        → 이메일 발송 (계약자 + 관리자)
        → DB: snapshotUrl, emailSentAt 저장
```

### 열람 링크 방식
- `/affiliate/contracts/[id]` — Next.js App Router 서버 컴포넌트
- DB에서 계약 데이터 조회 → 서버에서 계약서 HTML 재생성 → 렌더링
- 서명 이미지, 도장 이미지 포함한 완전한 계약서 복원
- **별도 인증 불필요** (contractId = UUID, 추측 불가)

## 3. 필요한 신규 파일/수정 목록

### Phase 1 — 열람 링크 (P0, 가장 먼저)
```
src/app/affiliate/contracts/[id]/page.tsx       ← 계약서 열람 (서버 컴포넌트)
src/app/api/affiliate/contracts/[id]/route.ts   ← GET: 계약 상세 조회
```

### Phase 2 — 스냅샷 + Drive 업로드 (P0)
```
src/app/api/affiliate/contracts/[id]/snapshot/route.ts
  POST body: { imageDataUrl: string }  ← html2canvas webp base64
  1. base64 → Buffer → Drive 업로드
  2. Drive 파일 공개 링크 설정
  3. DB GmAffiliateContract.metadata.snapshotUrl = 링크
  4. DB GmAffiliateContract.metadata.snapshotUploadedAt = now
  반환: { ok: true, viewUrl: string }
```

### Phase 3 — 이메일 발송 (P0)
```
src/app/api/affiliate/contracts/[id]/send-email/route.ts
  POST: contractId, recipientEmail
  1. DB 계약 조회
  2. 계약서 열람 링크 생성: https://crm.mabiz.kr/affiliate/contracts/{id}
  3. 이메일 HTML 템플릿 렌더링
  4. 발송: 계약자 이메일 + 관리자 이메일 (고정값 환경변수)
  5. DB metadata.emailSentAt = now
  - SMTP: 기존 Nodemailer (sendEmail 함수) 활용
  - 관리자 수신 이메일: process.env.AFFILIATE_ADMIN_EMAIL
```

### Phase 4 — 프론트엔드 통합 (P1)
```
- apply/page.tsx: 제출 성공 후 → snapshot 호출 → send-email 호출
- pre-sales/page.tsx: 동일
- 성공 화면에 "계약서 이메일 발송됨" 표시
- 성공 화면에 "계약서 열람하기" 버튼 (링크)
```

### Phase 5 — 관리자 UI (P2)
```
- admin/organizations 페이지 계약 목록 → "계약서 보기" 버튼
- 스냅샷 없을 때 "스냅샷 재생성" 버튼
- 이메일 재발송 버튼
```

## 7. 환경 변수 (신규)

```env
AFFILIATE_ADMIN_EMAIL=admin@mabiz.kr   # 관리자 알림 수신 이메일
AFFILIATE_SMTP_HOST=smtp.gmail.com      # 계약서 발송 전용 SMTP
AFFILIATE_SMTP_PORT=587
AFFILIATE_SMTP_USER=noreply@mabiz.kr
AFFILIATE_SMTP_PASS=...                 # 암호화 저장
AFFILIATE_FROM_NAME=마비즈스쿨 원격평생교육원
```

> **주의**: 기존 OrgEmailConfig (조직별 SMTP)가 아닌 시스템 전역 SMTP 사용.  
> 이유: 계약서 발송은 특정 조직이 아닌 본사 발송.

## 8. html2canvas 캡처 방법

```typescript
// 계약서 DOM 캡처 후 서버 전송
import html2canvas from 'html2canvas';

async function captureAndUpload(contractId: number) {
  const el = document.getElementById('contract-preview');
  if (!el) return;
  
  const canvas = await html2canvas(el, {
    scale: 1.5,         // 고해상도
    useCORS: true,      // 외부 이미지(stamp) 허용
    logging: false,
  });
  
  const imageDataUrl = canvas.toDataURL('image/webp', 0.85);
  
  await fetch(`/api/affiliate/contracts/${contractId}/snapshot`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ imageDataUrl }),
  });
}
```

## 9. 구현 순서 (권장)

1. `GET /api/affiliate/contracts/[id]` API (인증 없음, contractId = 비공개 UUID 대용)
2. `/affiliate/contracts/[id]/page.tsx` 열람 페이지 (계약서 재생성 + 서명 이미지)
3. `POST /api/affiliate/contracts/[id]/snapshot` (Drive 업로드)
4. `POST /api/affiliate/contracts/[id]/send-email` (이메일 발송)
5. apply/page.tsx + pre-sales/page.tsx 제출 성공 후 자동 호출

## 11. 보안 고려사항

- contractId는 DB 자동증가 숫자 → **URL에 직접 노출 금지**
- 열람 링크에는 `accessToken` (랜덤 UUID) 별도 생성 → DB 저장
- `/affiliate/contracts/[id]?token=xxx` 방식으로 보호
- 또는: 짧은 기간(30일) 만료 토큰
