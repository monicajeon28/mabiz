### P0-1. SHA-256 → bcrypt 교체 (보안/안정성/비즈니스 복합)

**현황:** `createAffiliateAccountPair.ts`의 `hashPassword()`가 SHA-256 사용. TODO 주석만 달린 채 방치.
**위험:** DB 유출 시 모든 파트너 계정 비밀번호 즉시 역산 가능. 더 큰 문제: bcrypt 전제로 작성된 CRM 인증 로직이 로그인 자체를 차단함 (SHA-256 해시로 bcrypt 검증 통과 불가).

```typescript
// 삭제
function hashPassword(pw: string) {
  return crypto.createHash('sha256').update(pw).digest('hex');
}

// 교체 — src/lib/password.ts의 기존 함수 import
import { hashPassword } from '@/lib/password';
```

**트랜잭션 외부에서 병렬 처리:**
```typescript
// bcrypt는 CPU 집약적 → 트랜잭션 밖에서 미리 실행
const [managerHash, agentHash] = await Promise.all([
  hashPassword(managerPassword),
  hashPassword(agentPassword),
]);
// 해시된 값만 트랜잭션 내부에 전달
```

### P0-3. 두 트랜잭션 → 단일 트랜잭션 통합 (안정성)

**현황:** `createAffiliateAccountPair`(TX1) 성공 후 `generateAffiliateLinksPair`(TX2) 실패 시 계정은 생기고 링크는 없는 불완전 상태 영구 고착. TX1 내부에서 이미 `status: 'APPROVED'`로 업데이트하므로 재시도 차단됨.

**해결:** 단일 `prisma.$transaction` 블록으로 통합.
```typescript
// approve/route.ts PUT 핸들러
const result = await prisma.$transaction(async (tx) => {
  // 1. 비밀번호 해시 (트랜잭션 외부에서 미리 실행해야 함 — 위 P0-1 참조)
  // 2. Manager User + AffiliateProfile 생성
  // 3. Agent User + AffiliateProfile 생성
  // 4. AffiliateRelation 연결
  // 5. Manager + Agent AffiliateLink 생성
  // 6. AffiliateContract status: 'APPROVED' + metadata 업데이트 (단 한 번)
  return { manager, agent, links };
});
```
`generateAffiliateLinksPair` 내부의 `tx.affiliateContract.findUnique` 재조회 제거 → `existingMetadata` 파라미터로 전달.

### P0-6. DB 소속 확인 후 마이그레이션 진행 (DB - 공유DB 보호)

**현황:** `migration.sql`의 `ALTER TABLE "AffiliateContract"`가 어느 DB(Neon/Supabase) 대상인지 불명확. Neon은 크루즈몰 공유 DB — 무단 변경 절대 금지.

**확인 절차:**
```sql
-- Neon 콘솔에서 확인
SELECT table_name FROM information_schema.tables 
WHERE table_schema = 'public' AND table_name = 'AffiliateContract';
```

- Neon에 있으면 → 크루즈몰 팀 승인 후 진행
- Supabase(CRM)에 있으면 → Prisma schema.prisma에 모델 추가 후 migrate

**인덱스 순서 수정 (기존 인덱스 설계 오류):**
```sql
-- 기존 (잘못됨 - amount 선행 컬럼)
CREATE INDEX ON "AffiliateContract"("amount", "status");

-- 수정 (status 선행 - 대기 목록 조회 최적화)
CREATE INDEX "AffiliateContract_status_amount_idx" 
ON "AffiliateContract"("status", "amount");
```

async function generateUniqueAffiliateCode(prefix: string, tx: Prisma.TransactionClient): Promise<string> {
  for (let i = 0; i < 5; i++) {
    const code = `${prefix}-${randomBytes(4).toString('hex').toUpperCase()}`;

## 자동화 신규 구현 5선 (핵심 가치)

### AUTO-1. 계약 승인 → 온보딩 전체 자동화 (One-Click)

```
관리자 "승인" 클릭 1회
  ↓
[단일 TX] Manager User + Agent User 생성
  ↓
[단일 TX] AffiliateLink Manager + Agent 생성
  ↓
[단일 TX] AffiliateContract APPROVED + OrganizationMember 등록
  ↓
[비동기] SMS 발송: 임시비밀번호 + 로그인URL + 어필리에이트링크
  ↓
완료 — 사람 손 개입 0회
```

### AUTO-2. 클릭 추적 → 전환 귀속 자동화

```
파트너 링크 클릭
  ↓ GET /api/r/[code]
쿠키에 ref=aff_XXXX 저장 (30일)
  ↓ 결제 완료 시
AffiliateSale.affiliateCode 자동 기록
AffiliateSale.commissionAmount 자동 계산
AffiliateLink.conversionCount + totalRevenue 원자적 업데이트
```

### AUTO-3. 수수료 자동 정산

```
매월 1일 크론
  ↓
AffiliateSale(status=EARNED, travelCompletedAt 기준) 집계
  ↓
파트너별 정산액 계산
  ↓
정산 PDF 자동 생성 → 이메일 발송 (미래)
  ↓
status: EARNED → PAID 자동 업데이트
```

### AUTO-4. PartnerMetrics 자동 갱신

월별 자동 배치 (위 P2-4). 현재 완전 수동 → 크론으로 전환.

### AUTO-5. 비활성 파트너 자동 알림

30일 무활동 파트너 → 담당 Manager SMS 자동 발송 (위 P2-3).
