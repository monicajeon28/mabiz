# 🚢 크루즈닷 기능 라이브러리

cruiseai 프로젝트의 주요 6가지 관리자 패널 기능을 모두 모아놓은 폴더입니다.

---

## 📂 폴더 구조

### **1️⃣ 이미지 라이브러리** (14개 파일)
```
1_이미지라이브러리/
├── images/                    # 이미지 관리 페이지 UI
├── batch-1-images-sync/       # 배치 동기화 API 1
├── batch-2-images-sync/       # 배치 동기화 API 2
└── batch-sync-images/         # 배치 동기화 통합
```
**포함 내용**: 이미지 업로드, 검색, 필터, WebP 변환, 메타데이터 관리

---

### **2️⃣ 서류관리** (2개 파일)
```
2_서류관리/
└── documents/                 # 서류 관리 페이지 + API
```
**포함 내용**: 서류 CRUD, 조회, 삭제, 파일 관리

---

### **3️⃣ B2B 랜딩페이지** (20개 파일)
```
3_B2B랜딩페이지/
├── b2b-landing/              # B2B 랜딩페이지 관리
├── landing-pages/            # 일반 랜딩페이지 관리
├── b2b/                       # B2B 고객 공개 페이지
├── landing/                   # 공개 랜딩페이지
└── landing/ (components)      # 랜딩 UI 컴포넌트
```
**포함 내용**: 페이지 CRUD, 미리보기, SEO 설정, CTA 관리

---

### **4️⃣ 퍼널** (119개 파일)
```
4_퍼널/
├── funnel/                    # 퍼널 대시보드 + 관리
├── funnel-messages/           # 퍼널 메시지 API
├── funnel-stage-transition/   # 단계 전환 로직
├── marketing/                 # 마케팅 퍼널 설정
├── funnel-visualizer/         # 퍼널 시각화 컴포넌트
└── partner/                   # 파트너 퍼널 관리
```
**포함 내용**: 퍼널 설계, 고객 여정 추적, 자동화, 분석

---

### **5️⃣ 관리자 공용** (509개 파일)
```
5_관리자공용/
├── admin/                     # 관리자 API (핵심)
│   ├── settings/              # 설정 관리
│   ├── marketing/             # 마케팅 기능
│   ├── customer-groups/       # 고객 그룹
│   └── ... (기타 20개 API)
├── admin/ (lib)               # 관리자 유틸리티
└── admin/ (components)        # 관리자 UI 컴포넌트
```
**포함 내용**: 인증, 권한, 설정, 고객 관리, 분석, 리포팅

---

### **6️⃣ 문자 퍼널 (SMS)** (51개 파일)
```
6_문자퍼널/
├── sms/                       # SMS API 기본 기능
├── messages/                  # 메시지 발송 API
├── settings/                  # SMS 설정 (인증키 등)
├── customers/                 # 고객별 SMS 발송
└── message-center/            # 메시지 센터
```
**포함 내용**: 문자 발송, 템플릿, 스케줄, 통계, 고객 세분화

## 💡 사용 가이드

### 1. 코드 이해하기
```
각 폴더 내 route.ts, page.tsx를 먼저 확인하세요.
├── route.ts = API 엔드포인트
└── page.tsx = UI/관리자 페이지
```

### 2. API 엔드포인트 확인
```
/app/api/admin/...
/app/api/sms/...
/app/api/b2b/...
```

### 3. 스키마 & 검증
```
Zod 스키마는 각 API 폴더의 route.ts 상단에 정의됨
예: imageSchema, documentSchema, funnelSchema
```

## 📝 주요 기능 연결도

```
관리자 공용 (5️⃣)
  ├─ 이미지라이브러리 (1️⃣) ← 상품 이미지 관리
  ├─ 서류관리 (2️⃣) ← 법률 서류 저장
  ├─ B2B랜딩페이지 (3️⃣) ← 파트너 페이지
  ├─ 퍼널 (4️⃣) ← 마케팅 자동화
  └─ 문자퍼널 (6️⃣) ← SMS 자동 발송
```
