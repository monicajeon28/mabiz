## 📊 재구매 로직 상세

```
첫 구매:
  - tripCount = 0 → 1 ✓
  - RePurchaseTrigger 생성 안 함

두 번째 구매:
  - tripCount = 1 → 2 ✓
  - 이전 여행의 endDate 조회
  - RePurchaseTrigger 생성
  - converted: true 자동 설정 ◄─ 재구매 유저 마킹!

세 번째 이상:
  - tripCount = 2, 3, 4... ✓
  - 매번 새로운 RePurchaseTrigger 생성
  - 또는 기존 트리거 업데이트
```

## 🔒 상태 머신

### ProductInquiry
```
pending ──[어드민 /confirm]──> confirmed
```

### ChatBotSession
```
ONGOING ──[lastQuestion 도달]──> COMPLETED
ONGOING ──[중도 이탈]──> ABANDONED
ONGOING ──[구매로 전환]──> CONVERTED
```

## ⚠️ 에러 처리

```
구매 확정 시 발생 가능한 에러:

1. 401 Unauthorized
   └─ 어드민 세션 없음 또는 role !== "admin"

2. 400 Bad Request
   └─ startDate 필수 또는 형식 오류

3. 404 Not Found
   └─ inquiryId가 존재하지 않음

4. 400 Conflict
   └─ 이미 confirmed 상태인 문의

5. 500 Internal Server Error
   └─ APIS 생성 실패 (무시되고 계속 진행)
```

## 📈 데이터 흐름 요약

```
공개 문의
   ↓ (ProductInquiry, User, AffiliateLead 생성)
   ↓
어드민 콜 로그 기록 (선택)
   ↓ (InquiryCallLog 생성)
   ↓
구매 확정
   ↓ (Trip, Itinerary, VisitedCountry, APIS, RePurchaseTrigger 생성)
   ↓
ProductInquiry.status = "confirmed"
   ↓
사용자가 앱 사용 시작 (password = 3800)
   ↓
여행 중 피드백 수집
   ↓ (UserTripFeedback 생성)
   ↓
재구매 대상 자동 마킹 (tripCount >= 2)
```
