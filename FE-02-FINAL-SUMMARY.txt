6. UI 상태 (4가지)
================================================================================

IDLE (초기 상태):
  - 필드: gray-300 테두리
  - 버튼: 파란색 활성화
  - 메시지: 없음

ERROR (검증 실패):
  - 필드: red-500 테두리 + red-50 배경
  - 에러 메시지: "⚠️ [내용]"
  - Toast: "입력값을 확인해주세요"
  - 버튼: 회색 비활성화

SUCCESS (전송 성공):
  - 화면: "✓ 신청이 완료되었습니다"
  - 하위: "24시간 내에 매니저가 연락 드리겠습니다"
  - 5초 후 초기화

LOADING (API 요청 중):
  - 필드: gray-300 (비활성화)
  - 버튼: "신청 중..." + 스피너
  - 입력 불가

9. 테스트 계획 (6가지 시나리오)
================================================================================

1. 필드 비워두고 제출
   → Toast + 필드 강조 + 에러 메시지 ✅

2. 폰번호 형식 오류 (예: "123-456-7890")
   → Toast + phone 필드 red-500 ✅

3. 에러 수정 후 재시도
   → 필드 에러 자동 제거 ✅

4. 유효한 데이터 제출
   → 로딩 → 성공 메시지 → 5초 후 초기화 ✅

5. 중복 가입 (409 에러)
   → Toast "이미 신청하신 번호입니다" ✅

6. 서버 오류 (500 에러)
   → Toast "서버 오류가 발생했습니다" ✅
