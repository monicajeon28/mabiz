# P2 UX 검증 — 빠른 참고 가이드

## 한눈에 보는 P2 변화

### 기술 변화
```
이전 (P0-5):
페이지 로드 → /api/auth/me 호출 → authChecked 대기 → 데이터 로드

P2:
페이지 로드 → Layout 권한 검증 → 데이터 즉시 로드 (authChecked 제거)
```

### 영향받는 7개 페이지

| # | 페이지 | 제거되는 호출 | 예상 개선 | 위험도 |
|---|------|-----------|--------|------|
| 1 | `/pnr/[id]` | `/api/auth/me` + isAdminMode | 200ms | 🔴 HIGH |
| 2 | `/admin/partner-applications` | `/api/auth/me` + authChecked | 150ms | 🟠 MEDIUM |
| 3 | `/admin/affiliate-sales-by-partner` | `/api/auth/me` + authChecked | 150ms | 🟡 LOW |
| 4 | `/team/affiliate` | 3x `/api/auth/me` + authChecked | 300ms | 🟠 MEDIUM |
| 5 | `/payments` | `/api/auth/me` + authChecked | 150ms | 🟡 LOW |
| 6-7 | 기타 대시보드 | `/api/auth/me` | 50-100ms | 🟡 LOW |

## 급속 체크 (5분)

### 개발자 테스트 (자신이 하기)

```bash
# 각 페이지를 개발 서버에서 열고:
1. DevTools Network 탭 열기
2. "/api/auth/me" 검색
   - 이전: 여러 번 표시
   - P2: 0번 표시
3. Lighthouse 실행
   - LCP 개선? (< 2초)
   - FCP 개선? (< 300ms)
4. Performance 탭에서 로딩 시간 확인
   - 이전: 600-1,200ms
   - P2: 450-950ms
```

### 빠른 정성 평가

| 항목 | 체크 | 합격선 |
|-----|------|------|
| 권한 검증 작동 | `/admin/*` → AGENT 접근 → 리다이렉트? | ✅ 신속 리다이렉트 (< 200ms) |
| 다중 탭 동기화 | 탭 A 로그아웃 → 탭 B 새로고침 | ✅ 즉시 /sign-in (< 200ms) |
| 에러 메시지 | 공개 고객 + 잘못된 폰번호 | ✅ "전화번호 미일치" 표시 |
| 보안 | 비로그인 사용자 /payments 접근 | ✅ 민감정보 0% 노출 |
| 성능 | 페이지 로딩 시간 | ✅ 이전 vs P2 20% 이상 개선 |

## 오류 신호 (이것이 보이면 배포 금지)

### 🔴 CRITICAL (즉시 롤백)

```
❌ 비로그인 사용자가 /payments 접근 → 정산금 정보 표시됨
❌ 로그아웃 후 /dashboard 새로고침 → 낡은 데이터 표시
❌ /api/auth/me 호출이 여전히 3회 이상 표시됨
❌ 페이지 로딩이 더 느려짐 (이전보다 느림)
❌ 무한 리다이렉트 루프 (탭 1→탭 2→탭 1...)
```

### 🟠 MODERATE (배포 연기, 수정 후 재시도)

```
⚠️ 권한 변경이 30초 이상 지연됨
⚠️ 다중 조직 계정에서 조직 전환 후 권한 미반영
⚠️ 스크린 리더에서 aria-label 읽지 못함
⚠️ 모바일 버튼이 44px 미만
⚠️ 명도 대비가 WCAG AA 미만
```

## 다중 탭 시나리오 체크

### 시나리오 1: 로그아웃 후 새로고침 (가장 중요)

```
탭 A: /admin/partner-applications (GLOBAL_ADMIN)
탭 B: /dashboard (같은 계정)

[실행]
1. 탭 A에서 로그아웃
2. 탭 B로 전환
3. F5 (새로고침)

[예상]
✅ 즉시 /sign-in으로 리다이렉트 (< 200ms)
❌ 로딩 화면 표시 후 리다이렉트 (느림)
❌ 낡은 데이터 표시 (보안 문제)
```

### 시나리오 2: 권한 변경 (비동기, 허용 범위)

```
탭 A: AGENT (초기 권한)
탭 B: /admin/organizations (관리자)

[실행]
1. 탭 B에서 사용자 권한 AGENT → OWNER로 변경
2. 탭 A에서 /team/affiliate 접근 시도
3. F5 (새로고침)

[예상]
✅ 접근 거부 리다이렉트 (< 1초, 캐시 때문에 지연 허용)
⚠️ 권한 변경이 최대 50초 지연 (다음 단계에서 개선)
```
