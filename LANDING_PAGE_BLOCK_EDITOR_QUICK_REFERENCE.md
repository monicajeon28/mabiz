## 🎮 상호작용 가이드

### 블록 추가
```
1. 좌측 팔레트에서 블록 카드 선택
2. 마우스로 드래그
3. 중앙 캔버스에 드롭
4. ✅ 블록 추가 완료!

또는 클릭으로 추가:
1. 좌측 팔레트에서 블록 카드 클릭
2. ✅ 캔버스 끝에 추가!
```

### 블록 선택 & 설정
```
1. 캔버스의 블록 클릭
2. 블록 테두리가 파란색으로 변함
3. 우측 설정 패널에 해당 필드들 표시
4. 필드 값 수정
5. 변경사항이 캔버스에 즉시 반영!
6. [저장] 클릭 (자동 저장도 가능)
```

### 블록 순서 변경
```
방법 A - 드래그:
1. 블록의 GripVertical 아이콘 잡기 (🖱️ 마우스 포인터 변함)
2. 위/아래로 드래그
3. 마우스 떼기
4. ✅ 순서 변경!

방법 B - 버튼:
1. 블록의 [↑] 또는 [↓] 버튼 클릭
2. ✅ 인접한 블록과 위치 교환!
```

### 블록 복제
```
1. 블록의 [⎘] (복제) 버튼 클릭
2. ✅ 현재 블록 바로 아래에 복사본 생성!
3. 필요시 설정값 수정
```

### 블록 삭제
```
1. 블록의 [✕] (삭제) 버튼 클릭
2. 확인 모달 표시: "정말 삭제하시겠어요?"
3. [삭제] 클릭
4. ✅ 블록 제거!
5. "실행 취소" 토스트 3초간 표시
   → 클릭하면 복구 가능
```

### 실행 취소 & 반복
```
키보드 단축키:
Ctrl+Z (또는 Cmd+Z): 이전 상태로
Ctrl+Y (또는 Cmd+Shift+Z): 다음 상태로

또는 토스트의 "실행 취소" 링크 클릭

무제한 히스토리 지원!
```

### 반응형 미리보기
```
캔버스 상단의 탭 선택:

[💻 PC]: 100% 너비 (전체 화면 보기)
[⌨️ 태블릿]: 768px 너비 (태블릿 크기)
[📱 모바일]: 375px 너비 (스마트폰 크기)

각 탭에서 블록이 반응형으로 재배치됨
```

### 도움말 보기
```
1. 필드 옆의 (?) 버튼 클릭
2. 팝오버 열림:
   - 필드의 의미 설명
   - 💡 팁 (권장사항)
   - ❌ 피하기 (주의사항)
3. [확인] 버튼으로 닫기
```

## 🎨 블록 타입별 상세 가이드

### 🎯 Hero 섹션
```
목적: 페이지의 가장 눈에 띄는 상단 영역
구성: 큰 제목 + 부제목 + 배경 + CTA 버튼

설정 예시:
제목: "마비즈와 함께하는 파일럿"
부제목: "크루즈여행 고객 관리 쉽게 시작하세요"
버튼: "파일럿 신청" → /pilot
배경: 파란색 또는 크루즈 사진
높이: 500px (기본)

💡 팁:
- 제목은 40글자 이내
- 배경 이미지는 고화질 (1200x600px 이상)
- 텍스트가 잘 보이려면 어두운 배경 또는 오버레이 사용

❌ 피하기:
- 긴 문장 (사용자가 다 읽지 않음)
- 너무 밝은 배경 (텍스트 가독성 저하)
- 자동 음악 재생 (UX 저해)
```

### 📝 제목
```
목적: 섹션별 제목 표시
크기 옵션: H1, H2, H3, H4, H5, H6

설정 예시:
제목: "마비즈의 3가지 특징"
크기: H2 (H1은 페이지당 1개만)
색상: #1f2937 (어두운 회색)
정렬: 중앙

💡 팁:
- H1은 페이지 제목 1개만 사용
- H2는 섹션 제목 (여러 개 OK)
- 크기로 제목 만들지 말 것 (시맨틱 HTML)
```

### 📋 신청 폼
```
목적: 사용자 정보 수집
기본 필드: 이름(필수), 전화(필수), 이메일(선택)

설정 예시:
제목: "파일럿 신청"
필드:
  - 이름 (필수)
  - 전화 (필수)
  - 이메일 (선택)
제출 버튼: "신청하기"
성공 페이지: /thank-you

💡 팁:
- 필드는 3-5개 이상 넣지 말 것
- 필수 필드 최소화 (이탈율 증가)
- 전화번호는 형식 검증 필수
```

### 🖼️ 이미지
```
목적: 시각적 요소 추가
권장 포맷: JPG, PNG, WebP

설정 예시:
URL: /images/cruise-1.jpg
대체텍스트: "이탈리아 크루즈 여행 이미지"
너비: 100% (반응형)
테두리반지름: 8px (동그란 모서리)

💡 팁:
- 최대 파일 크기: 2MB
- 권장 해상도: 800x600px 이상
- 대체 텍스트는 사진 설명으로 (SEO)

❌ 피하기:
- 텍스트가 많은 사진 (읽기 어려움)
- 저화질 사진 (전문성 떨어짐)
```

### ✅ FAQ (아코디언)
```
목적: 자주 묻는 질문 답변
구성: 질문 (클릭) → 답변 펼침

설정 예시:
Q1: "마비즈는 뭔가요?"
A1: "크루즈 여행사를 위한 고객관리 시스템입니다..."

Q2: "가격은 얼마인가요?"
A2: "무료 체험 후 월 29,900원부터 시작합니다..."

💡 팁:
- 3-10개 정도가 적당
- 가장 중요한 질문을 위에
- 짧고 명확한 답변 (긴 답변은 링크로)
```

### 📊 통계 카드
```
목적: 신뢰도 강화용 숫자 표시
구성: 큰 숫자 + 아이콘 + 설명

설정 예시:
카드 1: "1000+" 고객
카드 2: "99%" 만족도
카드 3: "24/7" 지원

💡 팁:
- 실제 데이터 사용 (거짓 숫자는 신뢰 저하)
- 3-4개 카드가 이상적
- 숫자는 크게, 설명은 작게
```

## 🚨 에러 처리

### 이미지 업로드 실패
```
❌ "이미지 크기가 2MB를 초과했습니다"
→ [자동 압축] 버튼으로 크기 줄이기

❌ "지원하지 않는 포맷입니다 (JPG, PNG, WebP만 가능)"

```

```
❌ "필드 '이름'은 필수입니다"
→ 입력 필드 빨간 테두리
→ 에러 메시지 표시
→ 저장 불가

✅ 해결: 필드 채우기 → 저장 가능
```

### 네트워크 오류
```
❌ "저장 실패 (네트워크 오류)"

→ 인터넷 연결 확인

✅ 해결: 인터넷 연결 후 재시도
```

## 🎯 모범 사례 (Best Practices)

### 레이아웃 구성
```
✅ Good:
1. Hero (제목 + CTA)
2. 특징 설명 (3-5개)
3. 사용 사례 또는 이미지
4. 추천 또는 통계
5. FAQ
6. 최종 CTA
7. 신청 폼

❌ Bad:
- Hero 없이 바로 텍스트
- 텍스트만 10개 이상
- CTA 버튼 없음
- 모바일에서 안 보임
```

### 콘텐츠 길이
```
✅ 좋음:
- 제목: 40자 이내
- 부제목: 100자 이내
- 단락: 200자 이내
- 리스트 아이템: 50자 이내

❌ 나쁨:
- 제목: 100자 이상 (다 읽지 않음)
- 리스트: 10개 이상 (피로)
```

### 색상 사용
```
✅ 좋음:
- 배경: 밝은 색 (흰색, 밝은 회색)
- 텍스트: 어두운 색 (검은색, 어두운 회색)
- CTA: 명확한 색 (파란색, 빨간색)
- 명도대비: 4.5:1 이상

❌ 나쁨:
- 밝은 배경 + 밝은 텍스트 (가독성 저하)
- 너무 많은 색상 (5개 이상)
```

## 📞 자주 묻는 질문 (FAQ)

**Q: 드래그가 안 되요**
A: 마우스 포인터를 블록 카드 정중앙에 맞추고, GripVertical 아이콘 근처에서 드래그하세요.

**Q: 실수로 삭제했어요**
A: 토스트의 "실행 취소" 링크를 클릭하거나 Ctrl+Z를 누르세요.

**Q: 스마트폰에서 보면 깨져요**
A: [📱 모바일] 탭에서 확인 후 설정을 수정하세요. 특히 이미지 너비를 100%로 설정하고 텍스트 크기를 조정하세요.

**Q: 데이터가 저장되지 않았어요**
A: [✓ 저장] 버튼을 명시적으로 클릭해야 합니다. 설정 패널 하단의 저장 버튼을 누르세요.

**Q: 블록을 복사한 후 설정을 바꿀 수 있나요?
A: 네! 복제한 블록을 클릭한 후 우측 설정 패널에서 값을 수정하면 됩니다.

**Q: HTML을 직접 수정할 수 있나요?
A: Phase 2에서 지원 예정입니다. MVP(Phase 1)에서는 블록 기반으로만 편집 가능합니다.
