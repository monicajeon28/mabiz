### 3️⃣ 인터랙티브 프로토타입
**파일**: `landing-page-block-editor-prototype.html`  
**기능**:
- ✅ 블록 드래그앤드롭 추가
- ✅ 캔버스에서 블록 순서 변경 (위/아래 버튼)
- ✅ 블록 선택 → 설정 패널 자동 표시
- ✅ 실시간 미리보기 (설정 변경 → 캔버스 즉시 반영)
- ✅ 반응형 탭 (PC / 태블릿 / 모바일)
- ✅ 복제 / 삭제 / 실행 취소
- ✅ 도움말 팝오버 (각 필드마다)
- ✅ 저장 토스트
- ✅ 모달 (삭제 확인, 도움말)

**사용 방법**: 브라우저에서 열기
```bash
open landing-page-block-editor-prototype.html
# 또는
# Windows: start landing-page-block-editor-prototype.html
# macOS: open landing-page-block-editor-prototype.html
```

**상호작용 테스트**:
1. 좌측 "Hero 섹션" 드래그 → 캔버스에 추가
2. Hero 블록 클릭 → 우측 설정 패널 열림
3. "제목" 입력 → 캔버스 실시간 반영
4. "배경 색상" 변경 → 즉시 반영
5. [↑] [↓] 버튼으로 블록 순서 변경
6. [✕] 버튼으로 삭제 → 확인 모달
7. [⌨️ 태블릿] 탭 → 너비 변경 확인
8. [🔍] 검색 → "버튼" 입력 → "CTA Button" 필터링

## 🎯 핵심 설계 원칙

### 1. WYSIWYG (What You See Is What You Get)
```
설정 패널의 입력 → 캔버스 즉시 반영
추가 기능/장식 절대 금지
"사용자가 지정한 것만" 정확하게
```

### 2. 3컬럼 레이아웃
```
┌────────────┬─────────────────┬─────────────┐
│   팔레트   │   캔버스         │   설정       │
│  (250px)   │ (계산 중심)      │   (250px)   │
│            │                 │            │
│ 블록 목록  │ 라이브 미리보기  │ 선택된      │
│ 드래그 가능│ 반응형 탭        │ 블록의      │
│ 검색 가능  │ 실시간 반영      │ 설정 필드   │
│ 자주 쓰는 것│                 │            │
└────────────┴─────────────────┴─────────────┘
```

### 3. 블록 타입 정의 (10가지)
```
🎯 Hero & CTA:     Hero, CTA Button, Feature List
📝 텍스트:         Heading, Paragraph
🖼️ 이미지:        Image
📋 폼:             Form
✅ 리스트:         Bullet List, FAQ
📊 데이터:         Stats Card
🎨 레이아웃:       Spacer, Divider
```

### 4. 드래그앤드롭 UX
```
팔레트 → 캔버스: 블록 추가
블록 내 순서 변경: GripVertical 또는 버튼
```

### 5. 설정 패널 필드 타입
```
text / textarea / number / select / color /
toggle / range / image / rich-text
```

## 🔗 API 엔드포인트

```typescript
// 블록 추가
POST /api/landing-pages/[id]/blocks
{ type: 'hero', settings: {...} }

// 블록 설정 수정
PUT /api/landing-pages/[id]/blocks/[blockId]
{ settings: {...} }

// 블록 삭제
DELETE /api/landing-pages/[id]/blocks/[blockId]

// 블록 순서 변경
PUT /api/landing-pages/[id]/blocks/reorder
{ blockIds: ['block_1', 'block_3', 'block_2'] }

// 전체 저장
POST /api/landing-pages/[id]/save-blocks
{ blocks: [...] }
```

## 🎓 사용자 시나리오

### 시나리오 1: 초보자가 처음 페이지 만들기
```
1. "파일럿 모집" 페이지 생성
2. Hero 블록 드래그 (제목, 부제목 입력)
3. CTA 버튼 추가 (버튼 텍스트, 링크 설정)
4. 특징 리스트 추가 (3개 아이템)
5. 공백 추가
6. 신청 폼 추가 (기본값: 이름, 전화, 이메일)
7. [저장] 클릭
8. [배포] 클릭
→ https://mabizcruise.com/p/{slug} 라이브!
```

### 시나리오 2: 중급자가 설정 커스터마이징
```
1. Hero 블록 클릭
2. 배경색을 파란색에서 보라색으로 변경
3. 배경 이미지 업로드 (cruise-hero.jpg)
4. 오버레이 투명도 조정 (0.3 → 0.5)
5. 버튼 색상 변경
6. 반응형 탭에서 [모바일] 클릭
→ 텍스트 크기 자동 조정됨
7. [저장] → 배포
```

### 시나리오 3: 고급자가 복잡한 페이지 설계
```
1. Hero + 10개 블록으로 복잡한 구성
2. 블록 순서 변경 (드래그 vs 버튼)
3. 실행 취소 (Ctrl+Z) 여러 번
4. 특정 블록 복제 (반복되는 섹션)
5. A/B 테스트 버전 생성 (Phase 2)
6. CSS override 추가 (Phase 2)
7. [배포] → 자동 버전 관리
```

## ✨ 특별한 기능들

### 도움말 시스템
```
각 필드 옆에 (?) 버튼
↓
팝오버 열림: 설명 + 팁 + 피하기 항목
↓
사용자가 필드 의미 이해
```

### 토스트 알림
```
블록 추가: "✅ 블록이 추가되었습니다"
설정 저장: "✅ 저장되었습니다" (2초)
삭제: "✅ 블록이 삭제되었습니다" + "실행 취소" 링크
```

### 실행 취소/반복
```
Ctrl+Z: 이전 상태로
Ctrl+Y: 다음 상태로
무제한 히스토리 (메모리 제약까지)
```

### 반응형 미리보기
```
[PC] 100% 너비
[태블릿] 768px 너비
[모바일] 375px 너비
실시간 전환, 스크롤 위치 유지
```
