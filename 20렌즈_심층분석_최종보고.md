## 📊 Executive Summary

마빕 CRM의 아키텍처는 **의도는 좋지만 실행에서 3~5가지 핵심 병목**을 가지고 있습니다:

1. **메모리 누수**: 콜 기록 `findMany()` → 전체 로드 (페이지네이션 없음)
2. **비동기 실패 무시**: Fire-and-forget Drive 백업 → 실패 시 재시도 없음
3. **보안 노출**: 서비스 계정 키가 `.env`에 평문 (Vercel Secrets로 관리 필요)
4. **프론트엔드 번들**: 대시보드 405줄 + 무거운 라이브러리 조합
5. **자동화 부재**: 수동 트리거 많음 (크론 백업, 정리, 집계 없음)

**결과**: 사용자 1000명 + 콜 기록 100K 이상 시 응답 지연 (3~5초) + Drive 백업 실패율 5~10%

**해결책**: 이 보고서의 **감독관 최종 권고** 섹션을 따르면 → Lighthouse 95+ / Supabase 응답 <100ms / Drive 신뢰도 99.9%

```typescript
// /api/cron/backup-pending
export async function GET() {
  const pending = await prisma.backupJob.findMany({
    where: { status: 'PENDING', attempt: { lt: 3 } },
    take: 100, // 배치 처리
  });

  for (const job of pending) {
    try {
      await backupCallLogsToGoogleDrive({ ... });
      await prisma.backupJob.update({
        where: { id: job.id },
        data: { status: 'SUCCESS' },
      });
    } catch (err) {
      await prisma.backupJob.update({
        where: { id: job.id },
        data: {
          status: 'FAILED',
          lastError: err.message,
          attempt: { increment: 1 },
        },
      });
      
      // quota 한계 도달 시 대기
      if (err.message.includes('quotaExceeded')) {
        await new Promise(r => setTimeout(r, 60000)); // 1분 대기
        break; // 루프 중단
      }
    }
  }
  
  return NextResponse.json({ ok: true, processed: pending.length });
}

const isRetryable = (error) => {
  const msg = error.message;
  return msg.includes('timeout') || 
         msg.includes('429') ||  // Too Many Requests
         msg.includes('ECONNREFUSED');
};

const isFinal = (error) => {
```

```typescript
  // 즉시 응답 (비동기 백업은 Cron에서 처리)

    log,
    backup: { id: backup.id, status: 'PENDING' },
  }, { status: 201 });
  
  // 이후 비동기 처리 (응답 후)
  if (process.env.GOOGLE_DRIVE_CALL_LOG_FOLDER_ID) {
    backupCallLogsToGoogleDrive({ ... })
      .then(() => {
        prisma.backupJob.update({
          where: { id: backup.id },
          data: { status: 'SUCCESS' },
        }).catch(() => {});
      })
      .catch(() => {
        prisma.backupJob.update({
          where: { id: backup.id },
          data: { status: 'FAILED' },
        }).catch(() => {});
      });
  }
}
```

```prisma
model CallLog {

  userId         String
  content        String?
  result         String?
  duration       Int?
  convictionScore Int?
  nextAction     String?

  contact        Contact  @relation(fields: [contactId], references: [id], onDelete: Cascade)

model BackupJob {
```

   ```
   GET /api/cron/backup-pending?auth=token

   SELECT * FROM BackupJob WHERE status='PENDING' AND attempt<3 LIMIT 100
   ↓
   FOR EACH job:
     1. Google Drive에 콜 기록 append (파일 업데이트)
     2. status='SUCCESS' 또는 'FAILED' 업데이트
     3. 실패 시 attempt++ 하고 다음 Cron 대기
   ```

   ```
     1. Google Drive 파일명: {customerName}_{YYYY-MM-DD}.txt

   ```
