## 작업 4️⃣ | 성능 최적화 - N+1 쿼리 제거 (P0 - 1일)

**파일**: `src/app/api/passport/admin/customers/route.ts`

### 현황
- ❌ 고객 조회 시 N+1 쿼리 발생 (100명 → 300+ 쿼리)
- ❌ 화면 로딩 시간 2-3초 소요
- ❌ DB 부하 증가

### 목표
- ✅ 쿼리를 1-2개로 단축
- ✅ 로딩 시간 300ms 이하
- ✅ DB 부하 90% 감소

### 현재 문제 코드

```typescript
// 나쁜 예: 각 사용자마다 3개의 관계 쿼리 발생
const users = await prisma.gmUser.findMany({
  where: { orgId },
  include: {
    trips: { orderBy: { departureDate: 'desc' }, take: 1 },
    passportSubmissions: { orderBy: { updatedAt: 'desc' }, take: 1 },
    passportRequestsSent: { orderBy: { sentAt: 'desc' }, take: 1 },
  },

// 결과: users.length * 4 쿼리 (include 3개 + 기본 1개)
```

### 개선 방식

```typescript
// 좋은 예: Raw SQL + 한 번에 조회
const records = await prisma.$queryRaw<any[]>`
  SELECT
    u.id, u."displayName", u.phone,
    t.id as "tripId", t."departureDate", t."productName",
    ps.id as "submissionId", ps."isSubmitted", ps."updatedAt",
    prl.id as "logId", prl."sentAt", prl.status
  FROM "GmUser" u
  LEFT JOIN LATERAL (
    SELECT * FROM "GmTrip" WHERE "userId" = u.id ORDER BY "departureDate" DESC LIMIT 1
  ) t ON true
  LEFT JOIN LATERAL (
    SELECT * FROM "GmPassportSubmission" WHERE "userId" = u.id ORDER BY "updatedAt" DESC LIMIT 1
  ) ps ON true
  LEFT JOIN LATERAL (
    SELECT * FROM "GmPassportRequestLog" WHERE "userId" = u.id ORDER BY "sentAt" DESC LIMIT 1
  ) prl ON true
  WHERE u."organizationId" = ${orgId}
  ORDER BY u.id
`;

// 메모리에서 집계
const result = records.reduce((acc, row) => {
  const user = acc.find((u: any) => u.id === row.id);
  if (!user) {
    acc.push({
      id: row.id,
      displayName: row.displayName,
      phone: row.phone,
      trip: row.tripId ? { id: row.tripId, ... } : null,
      submission: row.submissionId ? { ... } : null,
      lastLog: row.logId ? { ... } : null,
    });
  }
  return acc;
}, []);
```

### 결과
- ❌ 300+ 쿼리 → ✅ 1 쿼리
- ⏱️ 로딩 2-3초 → ✅ 300ms

// 매일 자정 실행되는 크론 작업
export async function runPassportSchedules() {
  const schedules = await prisma.gmPassportSchedule.findMany({
    where: { enabled: true },
  });
  
  for (const schedule of schedules) {
    if (schedule.type === 'trip_days_before') {
      // D-{daysBeforeTrip}인 여행 찾기
      const trips = await findTripsStartingIn(schedule.daysBeforeTrip);
      
      // 자동 발송
      await autoSendPassportLinks(trips, schedule.templateId);
    }
  }
}

## 작업 6️⃣ | 재시도 로직 (P1 - 1-2일)

**파일**: `src/app/api/passport/admin/send/route.ts`

### 목표
- ✅ SMS 발송 실패 시 자동 재시도 (최대 3회)
- ✅ 지수 백오프 재시도 (1초 → 2초 → 4초)
- ✅ 실패 로그 상세 기록

### 구현

```typescript
// lib/aligo-retry.ts
async function sendSmsWithRetry(
  to: string,
  message: string,
  maxRetries: number = 3
): Promise<{ success: boolean; messageId?: string; error?: string }> {
  let lastError: Error | null = null;
  
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      const response = await fetchAligo('/send/', {
        receiver: to,
        msg: message,
      });
      
      if (response.ok) {
        return { success: true, messageId: response.mid };
      }
      
      lastError = new Error(`Aligo API: ${response.message}`);
    } catch (err) {
      lastError = err as Error;
    }
    
    // 지수 백오프: 1초 * (2^시도횟수)
    if (attempt < maxRetries - 1) {
      const delayMs = 1000 * Math.pow(2, attempt);
      await new Promise(resolve => setTimeout(resolve, delayMs));
    }
  }
  
  return { success: false, error: lastError?.message };
}
```
