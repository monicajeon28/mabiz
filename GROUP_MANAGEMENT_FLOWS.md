## 1. 고객 추가 시 퍼널 자동 시작 흐름

```
┌─────────────────────────────────────────────────────────────────┐
│ 고객 추가 (POST /api/groups/[id]/members)                        │
└─────────────────────────────────────────────────────────────────┘
                           ↓
        ┌───────────────────────────────────────┐
        │ 1. 고객 소유권 검증                    │
        │    (organizationId 일치 확인)         │
        └───────────────────────────────────────┘
                           ↓
        ┌───────────────────────────────────────┐
        │ 2. ContactGroupMember upsert          │
        │    (groupId + contactId 중복 방지)   │
        └───────────────────────────────────────┘
                           ↓
        ┌───────────────────────────────────────┐
        │ 3. 리드 스코어 +10                     │
        │    (GROUP_ASSIGNED)                   │
        └───────────────────────────────────────┘
                           ↓
        ┌───────────────────────────────────────┐
        │ 4. 그룹의 funnelId 확인               │
        └───────────────────────────────────────┘
                      ↙              ↘
            funnelId 있음            funnelId 없음
                  ↓                      ↓
            triggerGroupFunnel()    (수동 발송만)
                  ↓
        ┌─────────────────────────────────┐
        │ 5. 퍼널 스테이지별로            │
        │    VipCareSequence 생성          │
        │    (scheduledAt 계산)            │
        └─────────────────────────────────┘
                      ↓
        ┌─────────────────────────────────┐
        │ 6. Cron이 scheduledAt에 SMS 발송 │
        │    (매일 밤 자동 실행)            │
        └─────────────────────────────────┘
```

**Fire-and-Forget 패턴**: 퍼널 실패해도 그룹 배정은 성공 응답

## 2. 공개 등록 폼 (그룹 토큰) 흐름

```
┌──────────────────────────────────────────────────────────┐
│ 파트너 사이트: mabiz 그룹 등록 폼 표시                  │
│ (Script: form action="/api/groups/[id]/register")      │
└──────────────────────────────────────────────────────────┘
                           ↓
   ┌───────────────────────────────────────────┐
   │ 고객이 폼 입력:                            │
   │ - name: 홍길동                            │
   │ - phone: 010-1234-5678                   │
   │ - email: hong@example.com (선택)         │
   │ - seq: 토큰 ID (숨김)                    │
   └───────────────────────────────────────────┘
                           ↓
   ┌───────────────────────────────────────────┐
   │ POST /api/groups/[id]/register            │
   │ (seq 토큰 검증)                           │
   └───────────────────────────────────────────┘
                           ↓
   ┌─────────────────────────────────────────────────┐
   │ 1. GroupToken 검증:                             │
   │    - seq 존재?                                  │
   │    - active === true?                          │
   │    - expiresAt > now?                          │
   └─────────────────────────────────────────────────┘
                           ↓
   ┌─────────────────────────────────────────────────┐
   │ 2. Contact upsert (phone + organizationId)      │
   │    (동일 번호면 수정, 신규면 생성)            │
   └─────────────────────────────────────────────────┘
                           ↓
   ┌─────────────────────────────────────────────────┐
   │ 3. ContactGroupMember 추가                       │
   └─────────────────────────────────────────────────┘
                           ↓
   ┌─────────────────────────────────────────────────┐
   │ 4. 그룹의 funnelId 확인                         │
   └─────────────────────────────────────────────────┘
                       ↙           ↘
           funnelId 있음          funnelId 없음
                 ↓                    ↓
         퍼널 자동 시작           완료
```

**시나리오**: 랜딩페이지에 등록 폼을 embed하면, 신청자가 자동으로 그룹에 배정되고 퍼널 시작


## 4. SMS 블래스트 발송 흐름

```
┌──────────────────────────────────────────────────┐
│ POST /api/groups/[id]/blast                      │
│ message="[고객명]님, 특가 소식입니다!"           │
│ dryRun=true (또는 false)                         │
└──────────────────────────────────────────────────┘
                           ↓
   ┌─────────────────────────────────────────────┐
   │ 1. 그룹의 멤버 조회 + 필터링                │
   │    - Contact.optOutAt === null              │
   │    - Contact.phone !== ''                   │
   │    - SmsOptOut 테이블 체크                  │
   │    - 최대 200명 제한                        │
   └─────────────────────────────────────────────┘
                           ↓
                  ┌─────────────────┐
                  │ dryRun 확인      │
                  └─────────────────┘
                      ↙           ↘
              dryRun=true      dryRun=false
                    ↓                ↓
        ┌──────────────────────┐  ┌──────────────────────┐
        │ 대상 인원만 반환     │  │ 실제 SMS 발송        │
        │                      │  │                      │
        │ willSend: 180명      │  │ 배치: 10건씩 병렬   │
        │ blockedCount: 15명   │  │                      │
        │ failedCount: 5명     │  │ for (0..179, i+=10)  │
        │                      │  │   Promise.allSettled │
        │ (데이터베이스        │  │     sendSms()        │
        │  수정 없음)          │  │                      │
        └──────────────────────┘  └──────────────────────┘
                                            ↓
                                  ┌──────────────────────┐
                                  │ Aligo API 호출       │
                                  │                      │
                                  │ [고객명] 자동 치환   │
                                  │ → Contact.name 적용  │
                                  └──────────────────────┘
                                            ↓
                                  ┌──────────────────────┐
                                  │ 응답 코드별 분류     │
                                  │                      │
                                  │ 1   → sentCount++    │
                                  │ -99 → blockedCount++ │
                                  │ 나머 → failedCount++ │
                                  └──────────────────────┘
```

**제약**:
- 최대 200명 (Vercel 10초 타임아웃)
- 10건씩 배치 (Aligo 동시 연결 제한)
- 메시지 치환: `[고객명]` → Contact.name

## 7. 퍼널 타입별 발송 일정 계산

```
접촉: 2026-05-16 (오늘)
출발: 2026-08-20 (D-96)

퍼널 타입: GENERAL (DAYS_AFTER)
┌───────────────────────────────────────┐
│ 스테이지별 발송 날짜                  │
├───────────────────────────────────────┤
│ 1단계: triggerOffset=0 (오늘)         │
│   scheduledAt = 2026-05-16 10:00      │
├───────────────────────────────────────┤
│ 2단계: triggerOffset=3 (3일 후)       │
│   scheduledAt = 2026-05-19 10:00      │
├───────────────────────────────────────┤
│ 3단계: triggerOffset=7 (7일 후)       │
│   scheduledAt = 2026-05-23 10:00      │
└───────────────────────────────────────┘

퍼널 타입: VIP_CARE (DDAY)
┌───────────────────────────────────────┐
│ 출발일(D-Day) 기준 발송               │
├───────────────────────────────────────┤
│ 1단계: triggerOffset=-150 (D-150)    │
│   scheduledAt = 2026-03-22 10:00      │
├───────────────────────────────────────┤
│ 2단계: triggerOffset=-90 (D-90)      │
│   scheduledAt = 2026-05-22 10:00      │
├───────────────────────────────────────┤
│ 3단계: triggerOffset=-30 (D-30)      │
│   scheduledAt = 2026-07-21 10:00      │
├───────────────────────────────────────┤
│ 4단계: triggerOffset=0 (출발일)       │
│   scheduledAt = 2026-08-20 10:00      │
└───────────────────────────────────────┘

주의:
- triggerType="DDAY"인데 departureDate 없으면
  → 퍼널 시작 차단 (RISK-01)
- VIP_CARE 중복 방지: ACTIVE/PENDING 중복 차단
```

## 9. 트랜잭션: 원자적 삭제

```
DELETE /api/groups/[id]
         ↓
   ┌─────────────────────────────────────┐
   │ 트랜잭션 시작                       │
   └─────────────────────────────────────┘
         ↓
   ┌─────────────────────────────────────┐
   │ Step 1: ContactGroupMember 모두 삭제 │
   │ DELETE FROM ContactGroupMember      │
   │ WHERE groupId = 'grp1'              │
   │ → 500명의 멤버 레코드 삭제          │
   └─────────────────────────────────────┘
         ↓
   ┌─────────────────────────────────────┐
   │ Step 2: GroupToken 모두 삭제        │
   │ DELETE FROM GroupToken              │
   │ WHERE groupId = 'grp1'              │
   │ → 5개의 토큰 레코드 삭제            │
   └─────────────────────────────────────┘
         ↓
   ┌─────────────────────────────────────┐
   │ Step 3: ContactGroup 삭제           │
   │ DELETE FROM ContactGroup            │
   │ WHERE id = 'grp1'                   │
   └─────────────────────────────────────┘
         ↓
   ┌─────────────────────────────────────┐
   │ 트랜잭션 커밋                       │
   │ (전부 성공 또는 전부 롤백)          │
   └─────────────────────────────────────┘

주의:
- Step 2 실패 → 전체 롤백
- Contact는 삭제 안 함 (FOREIGN KEY는
  ON DELETE CASCADE로 설정)
- 멤버는 삭제됨 (Contact는 유지)
```

## 10. 성능: N+1 쿼리 방지

### ❌ 나쁜 예 (N+1)
```typescript
const groups = await prisma.contactGroup.findMany({
  where: { organizationId: orgId }
});

// 각 그룹마다 퍼널 조회 (N번 쿼리)
const result = groups.map(async (g) => {
  if (g.funnelId) {
    const funnel = await prisma.funnel.findUnique({
      where: { id: g.funnelId }
    });
    return { ...g, funnelName: funnel.name };
  }
  return g;
});
// 총 쿼리 수: 1 + N = 1 + 100 = 101개
```

### ✅ 좋은 예 (배치 조회)
```typescript
const groups = await prisma.contactGroup.findMany({
  where: { organizationId: orgId }
});

// 퍼널 ID 추출
const funnelIds = groups
  .filter(g => g.funnelId)
  .map(g => g.funnelId!);

// 1번 쿼리로 모두 조회
const funnels = await prisma.funnel.findMany({
  where: { id: { in: funnelIds } }
});

// Map으로 O(1) 조회
const funnelMap = Object.fromEntries(
  funnels.map(f => [f.id, f.name])
);

// 결과 조합
const result = groups.map(g => ({
  ...g,
  funnelName: g.funnelId ? funnelMap[g.funnelId] : null
}));
// 총 쿼리 수: 1 + 1 = 2개
```

**효과**: 100명 × 2쿼리 = 50배 성능 향상

## 14. API 호출 순서 (통합 시나리오)

### 시나리오: 새 그룹으로 캠페인 시작

**Step 1**: 퍼널 생성 (별도 API)
```
POST /api/funnels
body: {
  name: "지중해 여름 캠프",
  funnelType: "GENERAL",
  stages: [
    { order: 1, triggerType: "DAYS_AFTER", triggerOffset: 0, ... },
    { order: 2, triggerType: "DAYS_AFTER", triggerOffset: 3, ... },
  ]
}
← funnel.id = "fun123"
```

**Step 2**: 그룹 생성
```
POST /api/groups
body: {
  name: "지중해 관심 고객",
  funnelId: "fun123",
  color: "#10B981"
}
← group.id = "grp456"
```

**Step 3**: 토큰 생성 (자동 또는 수동)
```
GET /api/groups/grp456/script
← token.id = "a1b2c3d4e5f6"
← script (HTML)
```

**Step 4**: 파트너 웹사이트에 폼 embed
```
<form action="/api/groups/grp456/register">
  <input name="seq" value="a1b2c3d4e5f6" />
  ...
</form>
```

**Step 5**: 고객이 폼 제출
```
POST /api/groups/grp456/register
body: {
  seq: "a1b2c3d4e5f6",
  name: "홍길동",
  phone: "010-1234-5678"
}
← Contact 생성 + 그룹 배정 + 퍼널 시작
← VipCareSequence 3개 자동 생성
```

**Step 6**: Cron 자동 실행 (매일)
```
1. VipCareSequence에서 scheduledAt <= now인 것 조회
2. SMS 발송
3. 상태 업데이트
```

**Step 7**: 관리자가 결과 모니터링
```
GET /api/groups/grp456
← 그룹 내 고객 수: 1명 증가
GET /api/funnels/fun123
← 퍼널 완료율: 0% (아직 첫 단계)
```
