# Partner Dashboard B2C - 수동 테스트 체크리스트

> **중요**: 다음 체크리스트는 로컬 개발 환경에서 npm test 및 실제 사용자 행동을 기반으로 검증합니다.
> 프로덕션 배포 전에 반드시 완료하세요.

## 1. 대리점장(Affiliate Partner) - 자신의 매출만 조회

### 1.1 대시보드 초기 로드
- [ ] 월을 선택하지 않았을 때 현재월(5월 2026) 데이터 표시
- [ ] 총 매출액이 올바른 통화 형식(원화)으로 표시됨
- [ ] 매출 건수(salesCount)가 정확한 숫자로 표시됨
- [ ] 예약 건수(reservationCount)가 정확한 숫자로 표시됨
- [ ] 트렌드 화살표(↑/↓)가 정확히 표시됨
  - 증가: 검은색 ↑
  - 감소: 빨간색 ↓
  - 변화 없음: 회색 →

### 1.2 최근 매출 목록
- [ ] 테이블에 최대 10개의 매출 기록이 표시됨
- [ ] 각 행에 다음 정보가 포함됨:
  - 상품명 (productName)
  - 매출액 (saleAmount) — 숫자 형식
  - 수수료 (commissionAmount)
  - 수수료율 (commissionRate) — % 형식
  - 상태 (status) — 텍스트
  - 날짜 (date) — YYYY-MM-DD 형식

### 1.3 권한 검증 — 자신의 데이터만 조회 가능
- [ ] 대리점A 로그인 후 대리점A의 매출만 표시됨
- [ ] 다른 대리점(B, C)의 매출은 조회 불가능 (DB 레벨 필터링 확인)
- [ ] 조직ID(organizationId)를 통한 필터링이 작동됨

## 2. 관리자(Admin) — 전체 데이터 조회

### 2.1 전체 매출 통계
- [ ] 관리자 로그인 후 전체 조직의 매출액 합계 표시
- [ ] 전체 대리점의 매출 건수 합계 표시
- [ ] 전체 예약 건수 합계 표시

### 2.2 기간별 매출 비교
- [ ] 월 선택 드롭다운에서 과거 월(1월~4월) 선택 가능
- [ ] 선택한 월의 데이터로 페이지 업데이트
- [ ] 트렌드가 올바르게 계산됨
  - 공식: ((현재월 - 전월) / 전월) * 100

### 2.3 대리점별 비교
- [ ] 최근 매출 목록에 모든 대리점의 데이터 포함
- [ ] 대리점 이름(assignedName)이 정확하게 표시됨

## 3. 상세 페이지 네비게이션

### 3.1 최근 매출에서 상세 페이지 이동
- [ ] 테이블의 행을 클릭하면 상세 페이지로 이동 (`/partner/dashboard/detail/:saleId`)
- [ ] URL에 올바른 saleId가 포함됨
- [ ] 상세 페이지 로딩 중 로딩 표시 표시

### 3.2 상세 페이지 데이터
- [ ] 매출 기본 정보 표시:
  - 상품명
  - 매출액
  - 수수료(commissionAmount)
  - 수수료율(commissionRate)
  - 상태
  - 생성일자

### 3.3 여권(Passport) 정보
- [ ] 예약의 여권 상태 표시 (ISSUED, PENDING, REJECTED 등)
- [ ] 여권 번호 표시 (있는 경우)
- [ ] 여권 발급 날짜 표시 (있는 경우)

### 3.4 PNR 정보
- [ ] 예약의 PNR 상태 표시 (CONFIRMED, PENDING, FAILED 등)
- [ ] PNR 코드 표시 (있는 경우)
- [ ] PNR 생성 날짜 표시 (있는 경우)

## 4. 월별 필터링

### 4.1 월 선택 기능
- [ ] 월 선택 드롭다운이 작동함
- [ ] 선택한 월이 하이라이트됨
- [ ] 월 선택 후 페이지가 즉시 업데이트됨 (1초 이내)

### 4.2 월별 데이터 정확성
- [ ] 2026년 5월 선택 → 2026-05-01 ~ 2026-05-31의 데이터만 표시
- [ ] 2026년 4월 선택 → 2026-04-01 ~ 2026-04-30의 데이터만 표시
- [ ] startDate와 endDate가 정확하게 계산됨

### 4.3 월별 트렌드 계산
- [ ] 현재월과 전월 비교 값이 정확함
- [ ] 계산 공식: ((현재 - 전월) / 전월) * 100
- [ ] 0% 변화 시 →(화살표) 표시

## 5. 에러 처리

### 5.1 네트워크 오류
- [ ] API 호출 실패 시 에러 메시지 표시
- [ ] 재시도 버튼이 표시됨
- [ ] 재시도 후 데이터가 정상 로드됨

### 5.2 권한 오류
- [ ] 미인증 사용자가 접근 시 로그인 페이지로 리다이렉트
- [ ] 권한이 없는 사용자가 다른 조직의 데이터 조회 불가
- [ ] 403 Forbidden 응답에 대한 적절한 UI 처리

### 5.3 세션 만료
- [ ] 세션 만료 시 로그인 페이지로 리다이렉트
- [ ] 만료 메시지가 표시됨
- [ ] 재로그인 후 정상 작동

### 5.4 데이터 없음
- [ ] 매출 기록이 없을 때 "데이터 없음" 메시지 표시
- [ ] 최근 매출 테이블이 비어있어도 레이아웃이 깨지지 않음
- [ ] 통계는 0으로 표시됨

## 6. 성능 테스트

### 6.1 로딩 속도
- [ ] 대시보드 초기 로드: 2초 이내
- [ ] 월 선택 필터링: 1초 이내
- [ ] 상세 페이지 로드: 1.5초 이내

### 6.2 데이터 양 처리
- [ ] 매출 기록 1000+개: 페이지 성능 저하 없음
- [ ] 최근 매출 목록: 최대 10개 표시 (N+1 쿼리 없음)

## 7. 응답 필드 검증 (API)
