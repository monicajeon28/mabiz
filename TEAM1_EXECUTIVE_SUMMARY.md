## 💼 비즈니스 임팩트

### 현재 상태 (문제)

```
대리점 파트너들이 ShortLink만 있고, A/B 테스트 불가능
→ SMS 카피 개선 효과 측정 불가
→ 의사결정 근거 없음
→ CPA 낭비 발생
```

### 도입 후 (기대 효과)

```
✅ SMS CTA 문구 2가지 비교 가능
  - "지금 예약하세요!" vs "35% 할인 예약하기"
  - 2주 데이터로 우승 선언 가능
  
✅ 개별 링크 성과 대시보드
  - 각 링크의 CTR (노출수 / 클릭수)
  - 통계적 유의성 자동 판정
  
✅ 신속한 최적화
  - 우승 카피 자동으로 다른 파트너에게 추천
  - A/B 테스트 이력 유지 (과거 학습 가능)
```

### 예상 ROI

```
CPA 개선: 5-15% (SMS 카피 최적화)
→ 연간 고객획득비용 1,000만 원 절감 (100명 × 100만 원 × 10% 개선)

구현 비용: 개발 시간 8시간
→ ROI: 1년 내 1,250배 회수 ✅
```

## 📊 3가지 옵션 비교

### Option 1: 별도 테이블 (ShortLinkABTest만)

**장점**:
- 기존 코드 변경 없음

**단점**:
- 리다이렉트 분산 로직 복잡
- Impression 추적 별도 관리
- API 필터링 필요 (테스트 링크 제외)

**점수**: 7/10 (안전하지만 불완전)

---

### Option 2: ShortLink에 필드 추가

**장점**:
- 필드 3개만 추가 (간단해 보임)

**단점**:
- 기존 코드 300-500줄 수정 필요 (17개 파일 영향)
- DB 마이그레이션 복잡 (10K+ 레코드 영향)
- 개발자 혼동 증가
- **롤백 불가능** (비용이 너무 커짐)

**점수**: 3/10 (이것만은 피해야 함)

---

### Option 3: 하이브리드 ⭐⭐⭐⭐⭐

**구성**:
```
ShortLink (기존, 수정 없음)
  ↓
ShortLinkABTest (신규: 링크 2개 비교)
  ↓
ShortLinkImpression (신규: 노출 추적)
```

**장점**:
- 기존 ShortLink 0줄 수정
- 마이그레이션 안전성: 최고
- 호환성: 100%
- 성능: 미리 계산된 통계 <10ms
- 확장성: C/D 링크 추가 쉬움

**점수**: 10/10 (완벽한 선택)

## 🚀 주요 기능

### 1. A/B 테스트 생성

```bash
POST /api/links/create-test
{
  "testName": "Day 0 SMS CTA 테스트",
  "variantA_id": "기존_링크_A",
  "variantB_id": "새로운_링크_B"
}

응답: 201 Created
{
  "test": {
    "id": "test_123",
    "status": "DRAFT"
  }
}
```

### 2. 테스트 시작

```bash
PATCH /api/links/tests/test_123/start
응답: { "status": "ACTIVE" }
```

### 3. 실시간 통계

```bash
GET /api/analytics/ab-tests/test_123

응답: {
  "variantA": {
    "clickCount": 145,
    "impressions": 3200,
    "ctr": 4.53%
  },
  "variantB": {
    "clickCount": 138,
    "impressions": 3150,
    "ctr": 4.38%
  },
  "pValue": 0.65,
  "winner": null,
  "recommendation": "더 많은 데이터 필요"
}
```

### 4. 우승 선언

```bash
PATCH /api/links/tests/test_123/declare-winner
{ "winner": "A" }

응답: { "status": "COMPLETED" }
```

## 🔐 리스크 관리

### Risk 1: 리다이렉트 분산 로직 버그

**영향도**: 높음 (테스트 무의미)  
**대응책**: A/A 테스트 선행 (100회 클릭 → 45-55% 분산 검증)

### Risk 2: Impression 데이터 불완전

**영향도**: 중간 (CTR 계산 오류)  
**대응책**: 신규 테스트만 Impression 추적, 기존 데이터는 "불완전" 표시

### Risk 3: 마이그레이션 실패

**영향도**: 높음 (서비스 중단)  
**대응책**: Staging 배포 사전 테스트, 롤백 플랜 준비

### Risk 4: 성능 저하

**영향도**: 중간 (사용자 불편)  
**대응책**: 인덱스 추가, N+1 문제 확인, 캐싱 고려
