📋 에러 처리 검증 (5가지)
================================================================================

1. 401 Unauthorized (인증 필요)
   - getAuthContext() 실패 시
   - try-catch로 처리

2. 403 Forbidden (권한 없음)
   - PATCH/DELETE에서 OWNER/GLOBAL_ADMIN 아닐 때
   - 명시적으로 반환: status 403

3. 404 Not Found (멤버 없음)
   - 멤버 조회 실패 시
   - 명시적으로 반환: status 404

4. 400 Bad Request (검증 실패)
   - memberId/role 필수값 없음
   - role 유효값 아님
   - 자신 수정 불가
   - 마지막 OWNER 삭제 불가
   - status 400으로 반환

5. 500 Internal Server Error (서버 오류)
   - 예상치 못한 예외
   - try-catch로 처리
   - 에러 로깅
