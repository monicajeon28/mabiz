### 🔴 Issue 3: sortOrder Race Condition (422줄)
**파일:** `src/app/(dashboard)/landing-pages/[id]/page.tsx:422`

**문제:**
```typescript
// 배치 루프 시작 (420줄)
const BATCH_SIZE = 3;
for (let i = 0; i < validFiles.length; i += BATCH_SIZE) {
  const batch = validFiles.slice(i, i + BATCH_SIZE);
  
  // 배치 결과 처리 중 (442줄)
  const sortOrder = images.length + successImages.length;  // ← 위험!
  
  // 문제: 배치 i=0 처리 중 다른 파일이 추가되면
  // images.length이 변할 수 있음!
  
  // 예시:
  // 배치1 (0-2): images.length=5 → sortOrder=[5,6,7]
  // 배치2 (3-5): 배치1 완료 후 setImages() → images.length=8
  //              이제 images.length=8 → sortOrder=[8+0, 8+1, 8+2] ✅ 올바름
  //
  // 하지만 동시성 고려하면 배치1 처리 중 다른 영역에서
  // setImages()가 호출되면 꼬임 가능
}
```

**10렌즈 분석:**
- **동시성** 🚨 → race condition 여전히 존재
- **설계** ❌ → images.length를 실시간 참조

**근본 원인:** 
- uploadFiles()에서 성공한 이미지를 즉시 setImages()로 반영
- 다음 배치에서 images.length를 사용할 때 이미 변경됨

**해결 방법:**
```typescript
// 루프 시작 전 initial 이미지 개수 고정
const initialImageCount = images.length;

// 배치 루프
for (let i = 0; i < validFiles.length; i += BATCH_SIZE) {
  const batch = validFiles.slice(i, i + BATCH_SIZE);
  
  // ... 배치 처리 ...
  
  // sortOrder 계산 (고정된 값 기반)
  const sortOrder = initialImageCount + successImages.length;
}
```

### 🔴 Issue 4-5: HTTP res.ok 누락 (231줄, 244줄)
**파일:** `src/app/(dashboard)/landing-pages/[id]/page.tsx:231, 244`

**문제:**
```typescript

  method: "PATCH",
  headers: { "Content-Type": "application/json" },
```

**10렌즈 분석:**
- **안정성** 🚨 → HTTP 5xx 응답 시 JSON parsing 실패
- **일관성** ❌ → 다른 함수들은 res.ok를 체크하는데 이 둘만 안 함
- **에러 처리** ❌ → 네트워크 오류 감지 불가

**근본 원인:** 초기 코드 검토 시 모든 fetch를 수정했지만, 이 2개가 누락됨

**해결 방법:**
```typescript

    method: "PATCH",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ ... }),
  });
  
  if (!res.ok) {  // ← 추가
    throw new Error(`HTTP ${res.status}`);
  }
  
  const data = await res.json();
  setEmailSaveMsg(data.ok ? "저장됐어요!" : "저장 실패");
  setTimeout(() => setEmailSaveMsg(""), 2500);
```
