🔧 구현 상세 (6가지 핵심 함수)
================================================================================

1️⃣ detectSegmentByAge(age?, ageRange?, preferenceType?, familyComposition?)
   └─ 우선순위: preferenceType > familyComposition > age > ageRange
   └─ 반환: Segment (A-E)
   └─ 예: preferenceType="romantic" → A / age=45, family_with_kids → B

2️⃣ detectLens(payload) → LensDetectionResult
   └─ 신호 기반 점수 시스템 (L0-L10)
   └─ 반환: { currentLens, confidence (0-100), triggers [] }
   └─ 예: healthConcerns → L9 (점수 50, 신뢰도 90%)

3️⃣ calculateRiskScore(payload, lens, segment) → RiskScoringResult
   └─ 10가지 신호 누적 계산 (의료, 가격, 준비, 경쟁사, ...)
   └─ 반환: { riskScore (0-100), riskLevel, signals [], recommendedAction }
   └─ 예: 건강우려(+25) + 가격민감(+20) = 위험도 45 (MEDIUM)

4️⃣ getRecommendedRiskAction(riskLevel, signals, lens) → string
   └─ Risk Level별 자동 액션 추천
   └─ CRITICAL → URGENT_HEALTH_SCREENING / PRICE_NEGOTIATION_CALL
   └─ HIGH → PREPARATION_GUIDANCE_SMS / FAMILY_PERSUASION_CALL
   └─ MEDIUM → PAYMENT_REMINDER_SMS / INTEREST_CLARIFICATION_SMS

5️⃣ createOrUpdateContact(organizationId, payload) → ContactAutoCreateResult
   └─ 통합 함수: Segment + Lens + Risk + Tags 한 번에 처리
   └─ Contact 생성/업데이트 + lensMetadata + riskScore 저장
   └─ 반환: { contactId, segment, lens, lensConfidence, riskScore, riskLevel, tags }

6️⃣ generateTags(segment, lens, riskLevel, source) → string[]
   └─ 기본 태그 + 복합 태그 + 렌즈 특수 태그 자동 생성
   └─ 예: ["source:form_submission", "segment:B", "lens:L1", "risk:HIGH",
          "B_L1_HIGH_PRIORITY", "price_sensitive"]
