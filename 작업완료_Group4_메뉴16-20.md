### 3. 사용자 경험 개선 (P1)

#### 에러 메시지 명시화
```tsx
// 이전: console.error만 출력
.catch((e) => { if (e.name !== "AbortError") console.error("[payslips]", e); })

// 개선: 사용자 피드백
.catch((e) => {
  if (e.name !== "AbortError") {
    console.error("[payslips]", e);
    setError(e.message || "데이터를 불러올 수 없습니다.");
  }
})
```
**효과**: 네트워크 오류, 권한 오류 등을 사용자에게 명확히 전달

#### Optimistic Update (담당자 지정)
```tsx
// 이전: 서버 응답 후 모달 다시 로드 (500ms 지연)
setAssignResult("✅ 담당자가 지정되었습니다.");
setTimeout(() => openDetailModal(selectedMember), 500);

// 개선: 즉시 UI 업데이트
const selectedStaffData = staffList.find((s) => s.id === selectedStaff);
setAssignResult(`✅ 담당자: ${selectedStaffData.displayName}로 지정 중...`);
```
**효과**: 사용자가 즉시 완료 확인, 체감 성능 향상

#### 태그 입력 필터
```tsx
// 이전: SUGGEST_TAGS 전체 표시
{SUGGEST_TAGS.map((tag) => <button>{tag}</button>)}

// 개선: 입력값으로 필터링
{SUGGEST_TAGS.filter((tag) =>
  tag.toLowerCase().includes(tagInput.toLowerCase())
).map((tag) => <button>{tag}</button>)}
```
**효과**: 사용자가 관련 태그만 선택 가능, 자동완성 경험

### 4. 데이터 정확성 (P1)

#### KST 타임존 명시화
```tsx
// 이전: UTC 기준
function formatDate(iso: string) {
  const d = new Date(iso);
  return `${d.getUTCFullYear()}-${String(d.getUTCMonth() + 1).padStart(2, "0")}-${String(d.getUTCDate()).padStart(2, "0")}`;
}

// 개선: KST(UTC+9) 변환
function formatDate(iso: string) {
  const d = new Date(iso);
  const kst = new Date(d.getTime() + 9 * 60 * 60 * 1000);
  return `${kst.getUTCFullYear()}-${String(kst.getUTCMonth() + 1).padStart(2, "0")}-${String(kst.getUTCDate()).padStart(2, "0")}`;
}
```
**효과**: 한국 출발 상품의 출발일 기준 정확성, D-day 계산 오류 제거

#### refundRate 소수점 통일
```tsx
// 이전: API와 UI에서 각각 처리
// API: Math.round((totalRefund / totalSaleAmount) * 1000) / 10
// UI: agent.refundRate.toFixed(1)

// 개선: API에서 한 번만 처리
// API: Number((totalRefund / totalSaleAmount * 100).toFixed(1))
// UI: {agent.refundRate}% (소수점 이미 포함)
```
**효과**: 중복 처리 제거, 정확도 유지
