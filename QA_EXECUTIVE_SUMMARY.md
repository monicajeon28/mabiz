## 🔴 P0 긴급 수정 5가지 (필수)

### 1. limit/page 검증 (30분)
```
문제: limit=-1, page=0 같은 이상한 값 처리 안 함
해결: 1-100 범위로 강제 설정
코드 위치: route.ts 라인 23-24
```

### 2. category + tone 필터 (1시간)
```
문제: 톤 필터가 OR 조건으로 작동 (AND여야 함)
해결: 쿼리 로직 재구성
코드 위치: route.ts 라인 45
```

### 3. isActive 기본값 (10분)
```
문제: 업로드 후 데이터가 보이지 않음 (false 기본값)
해결: true로 변경 + migration
코드 위치: schema.prisma 라인 2186
```

### 4. 요청 크기 제한 (20분)
```
문제: 10MB+ 파일 업로드 시 서버 부하
해결: 5MB 제한 + 413 에러 반환
코드 위치: route.ts POST 핸들러
```

### 5. 빈 검색어 처리 (20분)
```
문제: 초기 로드 시 전체 데이터 조회
해결: 빈 상태 또는 안내 메시지
코드 위치: QaLibrary.tsx 라인 100
```

## 💡 주요 통찰 (Key Insights)

### 발견 1: 입력값 검증 부족
- limit/page/검색어 길이 등 경계값 검증 없음
- → **결론**: 모든 외부 입력에 대한 화이트리스트 검증 필수

### 발견 2: 필터 로직 오류
- category (AND) + tone (OR) 혼재
- → **결론**: 복잡한 쿼리는 Raw SQL로 명시적으로 작성

### 발견 3: 에러 처리 미흡
- 네트워크/타임아웃/서버 에러 구분 없음
- → **결론**: 에러 타입별 사용자 메시지 사전 정의

### 발견 4: 초기 상태 불명확
- 검색어 없을 때 전체 데이터 조회
- → **결론**: 초기 상태 명시 (빈 상태 또는 안내)
